## 四、课题开题报告 ✏️（核心，约 3200 字）

> 字体小四宋体，行间距 20 磅。可加页填写。

### 4.1 课题的目的和意义

随着大语言模型（LLM）与视觉-语言模型（VLM）的飞速发展，视觉-语言-动作模型（Vision-Language-Action, VLA）作为机器人通用控制范式正在快速崛起。π₀（Black et al., 2024）、π₀.₅（Physical Intelligence, 2025）、OpenVLA（Kim et al., 2024）等典型 VLA 已在仿真和真机操作任务上展现强大能力。然而当前 VLA 研究普遍面临两个根本约束：

**约束 1（效率）**：主流 VLA 参数量在 3-7 B 级别，全参数微调显存需求达 46 GB 以上，远超消费级 GPU 容量；推理延迟 ≥ 70 ms，控制频率限制在 15 Hz 左右，难以匹配现代机器人 ≥ 30 Hz 的实时控制需求。

**约束 2（鲁棒）**：通过模仿学习训练的 VLA 在多模态扰动（视觉、动作、指令、环境四类共 17 种典型扰动）下表现脆弱。RobustVLA（Guo et al., 2026）报告 π₀ 在 0.05 幅度动作噪声下成功率从 96% 暴跌至 52.4%，0.1 幅度下完全失败。

现有工作要么追求效率（如 SmolVLA 0.45B，但鲁棒性未验证），要么追求鲁棒（如 RobustVLA 在 π₀ 3.3B 上 +12.6% 鲁棒增益，但训练显存超出消费级 GPU）。**「高效 × 鲁棒」交叉文献几近空白**（本项目调研 421 篇候选中无实质交集），构成本项目的研究空间。

本项目意义在于：(1) 学术层面，回答「鲁棒训练增益是否依赖大模型容量」这一与 Madry 等（2018）经典结论可能存在张力的关键问题；(2) 工程层面，提供首个可在消费级 24 GB GPU 复现的高效鲁棒 VLA pipeline；(3) 应用层面，为家庭服务、医疗辅助等对实时性与鲁棒性双高要求的具身智能场景提供方法学基础。

### 4.2 课题研究状况（国内外主要发展现状与趋势）

**(1) VLA 模型的演化**。π₀（[1]）首次将 flow matching 与 VLM 联合用于通用机器人控制，3.3B 参数；π₀.₅（[2]）通过 co-training 扩展到开放世界场景。OpenVLA（Kim 2024）以 7B 参数推动 VLA 标准化。轻量化方向 SmolVLA（[3]）以 0.45B 参数在 LIBERO 平均成功率 87.3%，与 π₀ 持平（86.0%），证明小模型可达可信基线，是本项目的骨干选择。

**(2) 自适应推理与动作分块**。ACT（Zhao 2023, [4]）引入 action chunking 范式，固定 chunk size = 100。AAC（[5]）首次提出基于动作熵的自适应 chunk size，在 LIBERO 平均 +0.9 pp。RTC（Black, Galliker, Levine, NeurIPS 2025, [6]）和 TT-RTC（[7]）解决了 chunk 边界跳变问题，已在 π₀.₆ 上 drop-in 验证，与本项目 AAC（chunk 长度自适应）正交可叠加。1-step 推理方向上，A2A（[8]）通过本体感觉初始化把 flow matching 压缩到单步（0.56 ms），是 stretch goal。

**(3) 对抗训练与鲁棒微调**。Madry 等（[9]）的 PGD 对抗训练奠基；TRADES（Zhang 2019）、AWP（Wu 2020）、Free-AT（Shafahi et al., NeurIPS 2019, [10]）相继提出显存优化变种。机器人侧 BYOVLA（Hancock et al., NeurIPS 2024, [11]）通过运行时 VLM 干预实现视觉鲁棒，但每步调用外部 LLM 慢 50.6 ×。RobustVLA（[12]）首次系统覆盖 17 种多模态扰动，提出 worst-case action noise + flow matching mismatch 双正则化，在 π₀ 上 +12.6%、FR5 真机 25-demo 下 +65.6%。

**(4) PEFT 与鲁棒训练交叉**。AdvLoRA（Lee 2024, [13]）首次在 VLM 上验证 LoRA + adversarial。Few-Shot Adversarial LoRA（[14]）指出少样本场景下 r=16 不足，建议升级到 r=32。DAC-LoRA（2025）提出动态对抗课程学习。但**未有工作在 VLA + LoRA + 多模态 worst-case 范式下做端到端验证**，构成本项目的明确空白。

**(5) 理论：capacity-robustness 关系**。Yin et al. ICML 2019（[15]）的 Rademacher complexity 分析表明 robust generalization gap 与 width/√n 相关；Schmidt 2018（[16]）指出对抗鲁棒泛化需要更多数据。本项目据此推测：**少样本 + worst-case 数据增广范式下，小模型的相对鲁棒增益与大模型相当甚至更优**，并将通过 capacity scan（SmolVLA 0.45B / π₀ 3.3B / OpenVLA 7B 三点曲线）实证验证。

### 4.3 主要内容、基本思路、难点和创新点

**研究内容**包含三个紧密耦合的模块：

**模块 A（推理时高效）**：在 SmolVLA 异步推理栈基础上实现 AAC。核心算法：每帧动作经 N=5/20 次采样估计高斯熵 $E_t = \frac{1}{2}\log[(2\pi e)^d \det(\Sigma_t)]$，按 $h^* = \arg\max_h(\bar E_{h+1} - \bar E_h)$ 决定 chunk 截断点。创新点是把 $h^*$ 同时作为 chunk 边界与异步队列的 prefetch 触发器，形成统一的"不确定性 → 调度信号"映射。

**模块 B（训练时鲁棒）**：在 SmolVLA 上集成 RobustVLA 双正则化。Output-robust 目标：$\min_\theta \mathcal{L}_{\pi_0}^\tau + \lambda_{out} \max_\delta \|v_\theta - u - \delta\|^2$；Input-robust 目标：$\min_\theta + \lambda_{in} \max_{\omega^*} \|v_\theta(\omega^*(o)) - u\|^2$；外层用 UCB 多臂老虎机自动选择 17 种扰动中的最坏者。**LoRA r=32 包裹**（采纳 Few-Shot Adv LoRA 建议升级自 r=16）控制显存。

**模块 C（理论联系）**：探索 worst-case $\delta^*$（RobustVLA）与 epistemic uncertainty $\Sigma_t$（AAC）的对偶关系。弱版本：在 LIBERO-Plus 上给出实证 correlation 曲线；强版本：基于 PAC-Bayes / Yin 2019 Rademacher 框架推导 generalization gap bound。

**基本思路**：以 SmolVLA 为统一骨干，模块 A 在推理阶段独立运作（训练时不开 N 次采样），模块 B 在训练阶段独立运作（推理时无显存增量），模块 C 在 Phase 3 联合实验阶段呈现。时序错开使两机制显存峰值不叠加。

**主要难点**：(1) RobustVLA 是 JAX/Flax 实现，需 1 周翻译到 PyTorch；(2) RobustVLA 训练后的 SmolVLA 上 AAC 熵估计可能因 velocity field 被"拉平"而系统性偏小，需消融验证；(3) LoRA r=32 在少样本下的 catastrophic overfitting 风险；(4) 联合训练 vs 顺序集成的协同效应判断。

**创新点**（系统/经验/理论三层）：详见 §3.4。

- 训练显存逐项账本（覆盖 24/48/80 GB 三档 GPU，全参 / LoRA r=16 / LoRA r=32 / Free-AT fallback 四种路径）
- 推理延迟逐项叠加分析（SmolVLA 同步 50 ms + AAC N=20 +25 ms + 异步摊销，目标 30+ Hz）
- 训练-推理一致性问题清单（3 个子问题 + 必做消融）
- 风险与缓解的二次细化（4 个高风险、4 个中风险均有具体备选方案）

**(3) 硬件与软件条件**。SmolVLA、π₀、AAC、RobustVLA、LIBERO 等开源代码均已确认可获取；项目主线在 24 GB 4090 / 48 GB A6000 / 80 GB A100 任一档 GPU 上均可执行（详见可行性账本）；导师课题组提供算力支持。

**(4) 团队准备**。团队成员已系统学习 PyTorch、HuggingFace Transformers / PEFT 库、深度学习对抗训练理论；已完成 SmolVLA 论文与 lerobot 仓库的代码精读。
