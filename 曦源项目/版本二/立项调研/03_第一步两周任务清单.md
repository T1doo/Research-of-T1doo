## W1 · Day 3-5（06-03 ~ 06-05）：方向词 GT 标注脚本

### A 同学任务
- [ ] **D3**：读 LIBERO 源码理解 demo 数据结构
  - 关键路径：`libero/libero/datasets/` 下的 `.hdf5` 文件
  - 每个 demo 含：`observations/robot0_eef_pos`, `observations/object`, `actions`
- [ ] **D4**：写 6 方向 + grasped 自动标注脚本 `scripts/annotate_directions.py`
  - 输入：单条 demo trajectory（含 robot eef pose 与 object pose 时序）
  - 输出：每步对应方向词标签 ∈ {right, left, up, down, front, back, grasped}
  - 算法：
    1. 计算 `dir = object_pos - robot_eef_pos`（3D 向量）
    2. 找绝对值最大的轴 → 主方向
    3. 检查 grasped 状态（距离 < 阈值 + gripper closed）
- [ ] **D5**：在 5 个 LIBERO-Spatial demo 上跑标注脚本

### B 同学任务
- [ ] **D3-D5**：目视检查 50 个标注样本（每个 demo 抽 10 步），记录错误率
  - 目标：错误率 < 20%
  - 常见 corner case：物体被遮挡、初始状态距离过远、抓取瞬间方向词跳变

### 同步检查点
- [ ] **D5·末**：方向词标注错误率 < 30%？
  - 是 → 进入 W2
  - 否 → 修改算法（如增加滑动平均、引入 robot frame 坐标变换）

## W2 · Day 1-3（06-08 ~ 06-10）：InSpire plugin 实现

### A 同学任务
- [ ] **D1**：fork OpenVLA-OFT 仓库到团队 GitHub
- [ ] **D1**：找到 prompt template 构造点
  - 定位：`prismatic/util/data_utils.py` 或 `prismatic/preprocessing/` 下的 prompt builder
  - 改造：在 task instruction 前加 `"In which direction is the {target} relative to the robot? Options: right, left, up, down, front, back, grasped. Answer: "`
- [ ] **D2**：加 7-class 方向词 cls head
  - 在 `prismatic/models/vlms/prismatic.py` 的模型类中加 `self.direction_head = nn.Linear(hidden_dim, 7)`
  - 在 forward 中取 prompt 末尾的 hidden state，过 `direction_head` 得 logits
- [ ] **D3**：改 loss
  - 在训练 loop 中加 `loss = action_loss + 0.1 * direction_ce_loss`
  - 准备 dataloader：把 W1 生成的方向词 GT 加进 sample dict

### B 同学任务
- [ ] **D1-D3**：写评测脚本骨架 `eval/eval_libero.py`
  - 输入：checkpoint path + LIBERO suite name
  - 输出：每任务 SR + 方向词预测准确率
  - 这个脚本 P2 评测会反复用，提前打磨

## W2 · Day 4-5（06-11 ~ 06-12）：sanity check 微调

### A 同学任务
- [ ] **D4·上午**：LoRA(r=16) 配置 + LIBERO-Spatial 100 episode demo 数据
- [ ] **D4·下午**：开训（预计 4-8 小时）
  ```bash
  python scripts/finetune_libero.py \
    --base_model openvla/openvla-7b-oft \
    --lora_r 16 \
    --task_suite libero_spatial \
    --max_steps 1000 \
    --direction_loss_weight 0.1 \
    --output_dir ./outputs/inspire_sanity_v0
  ```
- [ ] **D5·上午**：监控 loss 曲线
  - action loss 应在 1000 step 内下降 50%+
  - direction CE loss 应从 ~1.95 (random) 降到 < 1.0

### B 同学任务
- [ ] **D4-D5**：跑 vanilla OpenVLA-OFT 在 LIBERO-Spatial clean 上的 baseline
  - 取 50 episode 测 SR，作为后续对比基线
- [ ] **D5·下午**：跑 A 同学训出的 InSpire 版本，对比方向词预测准确率
  - 预期：方向词准确率 > 50%（高于 random 14.3%）
  - SR 预期：≥ vanilla baseline - 10pp（先求"不退化"）

### W2 末 Go/No-Go
- [ ] **D5·晚**：召开团队会议，回答 3 个问题：
  1. 方向词 cls head 是否在学（准确率 > 50%）？
  2. action SR 是否没有大退化（≥ vanilla - 10pp）？
  3. 训练显存是否 < 40GB（单卡 A100 可承受）？

| 三问回答 | 决策 |
|---|---|
| 全是 → | ✅ Plan A 进入 P1·M2，继续推进 |
| 1-2 否 → | ⚠️ 给自己 2 周修 bug，最晚 M2 末决策 |
| 全否 → | 🛑 启动 [[02_备选方案_FocusVLA鲁棒性诊断]] 切换 |
