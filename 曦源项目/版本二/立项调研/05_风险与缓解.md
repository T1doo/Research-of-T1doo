# 05 · 风险与缓解

## 1. Top 3 风险（按严重度排序）

### 风险 1 · InSpire 代码未释 + FocusVLA 代码未释（双重未释）

**严重度**：🔴 高 · **概率**：60%（截至 2026-05-26 两者都只承诺 "will be public"）

**影响**：
- 没法直接复现 InSpire / FocusVLA 的训练设置
- 实验结果的"对照公平性"会被审稿人质疑

**缓解策略**：
1. **InSpire 自己手写** — 方法极简：prompt 改造 + 7-class cls head + cross-entropy，约 200 行 PyTorch 代码可手写，已在 [[2025-05_InSpire_Zhang]] §方法论 部分梳理完整
2. **FocusVLA 用 OpenVLA-OFT 代理** — 不强依赖 FocusVLA 原版。叙事调整为「显式 grounding (InSpire) vs 省略视觉特征架构 (OpenVLA-OFT)」——OpenVLA-OFT 是 FocusVLA 论文 §2 明确批判的「shortcut 架构」典型，作为对照样本完全合法
3. **持续主动追踪** — 每周一查 InSpire / FocusVLA GitHub 页 + Lianli Gao / Yichi Zhang 个人主页；若 P2 期间释放代码立即切换

**触发条件**：W2 末两个代码都未释 → 进入"自己手写 InSpire + OpenVLA-OFT 代理"路线（默认路线，不慌）

### 风险 2 · 方向词 GT 推导有噪声

**严重度**：🟡 中 · **概率**：40%

**影响**：
- 如果方向词标签噪声 > 30%，cls head 学不到，InSpire plugin 在 OpenVLA-OFT 上不生效
- H1 假设的验证基础被破坏

**缓解策略**：
1. **数据源选择**：用 LIBERO 自带的 ground-truth object/robot pose（误差 < 1mm），而不是从视觉估计
2. **小样本质检前置**：W1 D3-D5 先目视检查 50 样本，错误率 < 30% 才进入 W2
3. **粒度容错**：6 方向粒度足够粗（每方向覆盖 60° sector），小角度抖动不引发标签翻转
4. **滑动平均**：抓取 / 释放瞬间方向词跳变可用 3-step 滑动多数表决
5. **难样本剔除**：物体被遮挡时标记 `uncertain`，训练时跳过该步
6. **联系 InSpire 作者** — UESTC Lianli Gao 组，邮件请教 corner case 处理

**触发条件**：W1 D5 末标注错误率 > 30% → 启动滑动平均 / corner case 处理；再不行 W2 末决策切 Plan B

### 风险 3 · 团队工程能力跟不上（最隐性风险）

**严重度**：🟡 中 · **概率**：30%

**影响**：
- 2 名大二本科生，PyTorch / transformer / LoRA 经验可能不足
- 改 OpenVLA-OFT 内部模块涉及 hidden state 提取、cls head 注入、loss 联合等多个工程坑
- W2 末若仍调不通 prompt 改造，整个 Plan A 推迟

**缓解策略**：
1. **工程前置学习**（建议在 2026-05 末完成）：
   - PyTorch 官方 tutorial：transformer block 内部结构（especially attention output 与 hidden state 区别）
   - HuggingFace `peft` 库 doc：LoRA 配置参数
   - OpenVLA-OFT 仓库 README + 至少 1 个 demo notebook 走完
2. **AI 辅助开发**：W1-W2 期间 Claude/GPT API 预算 ¥150（约 30-50 次代码 review 对话），用于解决 PyTorch 工程坑
3. **社区求助**：加入 HuggingFace LeRobot Discord、知乎 LLM 板块、reddit r/MachineLearning，碰到坑 24h 内求助
4. **每周复盘**：每周日晚团队会 30 分钟，识别工程瓶颈
5. **导师 office hour**：每月 1 次（约 30-60 分钟）请导师过工程关键决策

**触发条件**：W2 末 prompt 改造 / cls head / LoRA 任一环节没跑通 → 给自己 2 周 buffer 修 bug；M2 末仍未通 → 切 Plan B

## 2. 次要风险（影响有限但需注意）

### 风险 4 · GPU 资源不足或价格上涨

**严重度**：🟢 低 · **概率**：50%

**缓解**：
- 优先用学校共享 GPU（如有）
- 国内云（AutoDL / 矩池云）有学生优惠
- P2 期间评测可分批跑（每天跑 1-2 维度），避免一次性占大量 GPU-h

### 风险 5 · LIBERO-Plus 评测结果与论文数字差异大

**严重度**：🟢 低 · **概率**：30%

**缓解**：
- 评测时严格按 LIBERO-Plus 官方协议（seed、episode 数、step 限制）
- 每个数字至少 3 seed 取均值 ± std
- 复现 [[2025-10_LIBERO-Plus]] 中的 OpenVLA baseline 作为校准

### 风险 6 · VLA-Risk 代码 / 数据始终不释

**严重度**：🟢 低（仅是 stretch goal） · **概率**：50%

**缓解**：
- VLA-Risk 在主路径中是 stretch goal，不释也不影响 H1/H2 验证
- 用 LIBERO-Para（已开源，专做语言改写鲁棒性）作为 instruction 维度的替代

### 风险 7 · H1（互补假设）被实证否定

**严重度**：🟢 低 · **概率**：35%

**缓解**：
- 哪怕"完全重叠"也是有价值的 finding（"两条路 → 同一种 grounding"，对学界是反直觉的结论）
- 改叙事重点为 H2（叠加增益），如 H2 也不显著，再退到 H3（failure predictor）
- 最差情况：故事改为「负面 finding workshop short paper」，仍可投出

## 3. Plan A → Plan B 切换决策树

```
W2 末 (M0.5)
├── A. 方向词准确率 > 50% & action SR ≥ vanilla - 10pp
│   └── ✅ 继续 Plan A，进入 P1 后期
│
├── B. 方向词准确率 30-50% & action SR > vanilla - 15pp
│   └── ⚠️ 给 2 周 buffer 调参（learning rate / loss weight / batch size）
│       └── M2 末（W6）再决策
│           ├── 通过 → 继续 Plan A
│           └── 仍不行 → 切 Plan B
│
├── C. 方向词准确率 < 30%（cls head 不学）
│   └── 🛑 检查方向词 GT 标注脚本
│       ├── 找到 bug → 修复后回 A/B
│       └── 标注本身就有 > 30% 噪声 → 直接切 Plan B
│
└── D. action SR 退化 > 15pp（vanilla baseline 被打穿）
    └── 🛑 InSpire loss weight 调小（0.1 → 0.05 → 0.02）
        ├── 通过 → 继续 Plan A（但弱化 InSpire 贡献）
        └── 仍退化 → 切 Plan B
```

```
P2 末 (M5)
├── H1 强成立（≥ 2 维度强项不重叠）
│   └── ✅ 继续 Plan A P3-P4 推进
│
├── H1 弱成立（仅 1 维度，p ∈ [0.05, 0.1]）
│   └── ⚠️ 增加 instance 数 / 加 seed → 重跑关键维度
│       └── 仍弱 → 改叙事：「主要差异在 X 维度」
│
└── H1 完全否定（任何维度都不显著）
    └── 改叙事：「两路 grounding 等价」
        └── 仍可投 workshop 短稿（这本身是有价值的 finding）
```

## 4. 切换 Plan B 的具体操作（48 小时内可完成）

| 操作 | 时间 | 备注 |
|---|---|---|
| 团队会议确认切换 | 1h | A + B + 导师（如可联系） |
| 关闭 InSpire 模块（prompt / cls head / loss） | 4h | git revert 或新 branch |
| 切换到 attention map 提取（PyTorch hook） | 8h | 复用 [[02_备选方案_FocusVLA鲁棒性诊断]] 的设计 |
| 准备 failure predictor 输入特征 | 8h | 与 Plan A P4 工作重叠 |
| 跑首批诊断实验（1 维度 × 2 模型） | 24h | 验证 pipeline 通 |
| 更新立项文档（备注切换原因） | 2h | 写入 `notes/plan_b_switch_log.md` |
| **合计** | **47h** | 2 天内完成 |

## 5. 重要：不要做的事

- ❌ **不要在 W2 末 ROI 还没出之前就大幅扩展 scope**（如同时尝试 VLA-Risk + SmolVLA + 4 个变体）
- ❌ **不要追求 100% 完美的方向词 GT**——70-80% 就够 plug-in 起作用
- ❌ **不要为了"工程美感"重写 OpenVLA-OFT 的 fork**——只做必要 patch，留可读性给原仓库 maintainer
- ❌ **不要省略 baseline**——所有 4 变体都必须有 3-seed 均值，否则审稿人会问
- ❌ **不要在 P1 期间研究 P3 的 attention map**——按 phase 顺序推进，避免 context switch
