# 00 · 总览与立项决定

## 1. 为什么要切换到版本二

T1doo 团队原版本一定位为 **Efficient VLA**——在 SmolVLA-0.45B 上同时跑：

1. **AAC**（自适应推理）— 推理时根据场景熵动态调 chunk 长度
2. **RobustVLA**（对抗训练）— worst-case δ 多模态扰动训练
3. **可选 RC-NF**（异常检测）— 失败检测 + 主动恢复

导师反馈的核心问题（[[版本一/立项/调研/00_总览与立项建议]] 也记录过此判断）：

> "**AAC × 鲁棒性的耦合实现难度高、一年期产出风险大**，建议**收窄到鲁棒性单一主线**深耕。"

同时导师抛出**导航星论文 FocusVLA**（[arXiv:2603.28740](https://arxiv.org/abs/2603.28740)，2026-03，HIT-Shenzhen × DaiMon Robotics × NJU × RUC），明确指引"视觉利用率 → 隐性鲁棒性"这条工程可控的路线。

## 2. FocusVLA 的关键定位（导师推荐的根本原因）

FocusVLA 的核心论断：**VLA 鲁棒性差的根因不是 encoder，而是架构 shortcut**——action query 在 mixed attention 中绕过了视觉细节。它提出两个机制：

- **Modality Cascaded Attention**：把并行 mixed attention 拆成串行的 3 次独立 cross-attention，切断 shortcut
- **Focus Attention**：patch-level top-K (256) + channel-level element-wise gate

效果：0.5B 参数在 LIBERO 多权重 98.7%，**打败 7B 的 OpenVLA-OFT (98.5%)**；训练 1.5× 加速。

**但 FocusVLA 完全没在 robustness benchmark 上系统评测**（论文只报 LIBERO clean 98.7%）。作者自述 4 大局限：

1. 「鲁棒性对**背景 + 纹理同时变化**仍有限」
2. 「对**初始状态**敏感」← 立项关键切入点
3. 「VLM 内部的视觉利用没涉及」
4. 「**未在 robustness benchmark 上系统评测**」

详见精读笔记：[[2026-03_FocusVLA_Zhang]]。

## 3. 现实约束（必须降级方案的依据）

| 维度 | 数值 |
|---|---|
| 团队 | **2 名大二本科生**（综述报告原假设 4-6 人） |
| 预算 | **¥10,000**（约 $1400，能租 100-200 GPU-hours 云端 A100） |
| 时间 | 1 年（2026-06 → 2027-06） |
| GPU | 学校共享 / 云端租用，单卡 ≤ 80GB |
| 目标 | **能投个 workshop 论文**（不强求顶会 main） |

这个约束意味着：综述报告 [[论文探索/01_课题排序与方向推荐]] 中给出的 6 个候选方案排序需要**重新加权**——novelty 与可行性的权重要倒过来。

## 4. 选定方案（Plan agent 三方对比 + 用户确认）

### Plan A 主推 · 候选 C · FocusVLA × InSpire 互补研究

用 **InSpire**（[arXiv 2505.13888](https://arxiv.org/abs/2505.13888)，详见 [[2025-05_InSpire_Zhang]]）的 zero-data plugin（强制 VLA 先输出方向词再输出 action）作为**显式 spatial grounding**，对照 FocusVLA / OpenVLA-OFT 代表的**隐式 attention 聚焦**，在 LIBERO-Plus 7 维 × VLA-Risk 6 维扰动上系统验证两路 grounding 的鲁棒性差异、互补性、可叠加性。

**详细设计**：[[01_主方案_FocusVLA_InSpire互补]]

### Plan B 兜底 · 候选 A · FocusVLA-Probe 鲁棒性诊断

完全不训练 VLA，把 FocusVLA（或其代理 OpenVLA-OFT/SmolVLA）当黑盒，在 LIBERO-Plus × VLA-Risk × LIBERO-Para 三大 benchmark 上做系统化鲁棒性诊断 + 失败模式画像。

与 Plan A **共用 OpenVLA-OFT + LIBERO-Plus 评测 pipeline**，若 Plan A 在 M2 末复现卡壳，48 小时内可切换，损失 < 1 个月。

**详细设计**：[[02_备选方案_FocusVLA鲁棒性诊断]]

### 排除方案 · 候选 B · FocusVLA-Robust-Mini

手写 Modality Cascaded Attention 接入 SmolVLA + 加 TRADES-mini 损失。**对 2 名大二本科生工程风险过高**——论文公式简洁但 attention 模块的实际实现涉及 KV cache / masking / 并行优化等多个 PyTorch 工程坑。详见 [[07_为什么不选其他方案]]。

## 5. 项目命名（双轨）

老师推荐的论文 FocusVLA 直接指向了一条工程可控的路线，但项目实际定位是"双谱系互补"研究，命名需要兼顾两种受众：

### 申请书学术版（突出理论意义）

> **「具身智能模型中显式与隐式视觉接地信号的鲁棒性互补机制研究」**

适合在**曦源项目申请书**的标题、摘要、研究意义部分使用。突出了：
- 「显式 vs 隐式」的理论对立
- 「视觉接地（visual grounding）」的学术术语
- 「互补机制」的现象学定位
- "具身智能" 而非具体模型名，扩大受众

### 立项书工程版（突出鲁棒性关键词）

> **「基于显式空间推理插件的视觉-语言-动作模型鲁棒性实证研究——FocusVLA 谱系 × InSpire 谱系的交叉验证」**

适合在**立项信息表**、组会汇报、技术细节描述中使用。突出了：
- 「插件」的轻量工程定位
- 「鲁棒性」的直接关键词
- 「FocusVLA × InSpire」具体锚定，方便检索
- 「实证研究」体现评测/诊断而非纯理论

实际申请时建议**两个版本根据材料类型混合使用**——学术版做大标题，工程版做副标题或摘要扩展。

## 6. 核心研究问题（H 假设）

| 假设 | 命题 | 验证方式 |
|---|---|---|
| **H1（互补）** | 显式 grounding（InSpire）在初始状态扰动 / 视角变换上鲁棒增益显著；隐式 attention 聚焦类（OpenVLA-OFT / SmolVLA + Focus-style）在背景纹理 / distractor 上鲁棒增益显著。**两类方法的强项维度不重叠** | 4 模型变体 × 7 维 = 28 实验，统计每维度显著性 |
| **H2（叠加）** | InSpire plugin + Focus-style 架构的简单叠加在 LIBERO-Plus 7 维平均 SR 上**优于任一单独使用** ≥ 3pp | 4 变体直接对比平均 SR |
| **H3（诊断）** | 方向词预测准确率与最终 action 成功率的相关系数 r ≥ 0.6 → InSpire 可作为运行时 failure predictor | 收集成败 episode 配对，训分类器算 AUC |

详见 [[01_主方案_FocusVLA_InSpire互补]] §核心假设。
