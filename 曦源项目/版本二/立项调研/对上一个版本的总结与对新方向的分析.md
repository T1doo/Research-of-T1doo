# 老师推荐的 FocusVLA 论文，我的解读

## 它到底在做什么

**它不是一篇"鲁棒性"论文**，它是一篇"视觉信息利用"论文。这一点必须先讲清楚，否则你们容易被关键词带偏。

它的核心叙事是：现有 auto-regressive VLA（OpenVLA-OFT、VLA-Adapter 这条线）有三个 bottleneck：

1. **架构 bias**：mixed attention 让动作 query 走"捷径"，绕过视觉细节
2. **信息过载**：视觉 token 太多（512 个），注意力被稀释
3. **任务无关噪声**：背景信息淹没了有用信号

它的解法是两件事：

1. **Modality Cascaded Attention**：把视觉/动作 query/动作 latent 三路从"混在一起"改成"串行依次 cross-attention"，强迫动作 latent 必须经过视觉信息
2. **Focus Attention** = patch-level（TopK 选 256/512 个 token）+ channel-level（element-wise gate）

主要数字：LIBERO 平均 98.7%（0.5B 参数打 7B），RoboTwin Hard 15% vs π0 6%，训练 1.5× 加速。

**关键事实**：它在 LIBERO 标准 split 上跟 OpenVLA-OFT、VLA-Adapter 比，**完全没做扰动鲁棒性评测**。它说自己 robust，但那是论文修辞——意思是"对视觉细节更准"，不是"对扰动更鲁棒"。

## 它和"鲁棒性"的真实关系

**有间接关系**：如果模型只看任务相关 patch，那理论上对背景/光照变化应该更鲁棒（因为它根本就不看这些 token）。论文 figure 8(a) 显示 wrist camera 视图只剩 gripper 和目标物体——这跟 LIBERO-Plus 论文的 Finding 6（"wrist camera 的近距离视角提供光照不变的几何线索，因此光照鲁棒性强"）是**同一个机制**。

**但论文自己没把这件事做出来**：它没在 LIBERO-Plus / 任何扰动 benchmark 上跑过；它没分析"focus 后哪些扰动会变得更鲁棒、哪些不会"；它没跟鲁棒训练方法（RobustVLA、对抗训练）比较过。

**这是一个真正的 gap**——而且是个老师推荐的论文自然引出的 gap，不是你们硬找的。

# 评估你们提出的两条思路

## 思路 1：把 focus 的 token 数做成可变长（按任务复杂度智能调整）

**直白讲：我不太建议把这个作为核心。** 理由有几条：

**(a) 你们之前的方案就是栽在"自适应"上的**。AAC 是把 chunk 长度自适应，老师否决了。把"focus token 数自适应"作为新主线，本质上是同一种思路换了个对象——chunk 自适应做不了，token 数自适应同样会卡在：怎么定义"复杂程度"信号、怎么避免训练-推理一致性问题、怎么消融。老师再看一遍可能还是同样反应。

**(b) 它在工程上比 AAC 更难**。AAC 至少有现成 entropy 信号可以借鉴；"任务复杂度"是什么完全没定义——你要么用 entropy（重新做 AAC 那一套），要么用别的信号（比如 attention 熵），但这些都得自己设计 + 验证 + 消融，比想象的复杂得多。

**(c) FocusVLA 论文自己已经做过 token 数 sweep**：128 / 256 / 384 / 512，结论是 **256 最好**，128 太少（93.8% Long）、512 太多（95.0% Long）。它已经报告"过少会损失信息、过多会稀释注意力"——这是个固定值的 sweet spot，不是任务级别变化的事情。你要论证"动态选优于固定 256"，得在 LIBERO Long/Spatial/Object/Goal 四个 split 上都能拿到 +1pp 以上，且消融能讲清楚信号是什么——这非常难。

**(d) Token 数从来不是 LIBERO/RoboTwin 上的性能瓶颈**。FocusVLA 已经 98.7% 了，再调 token 数能挤出来的空间小于 1pp。

如果硬要做，我建议**把"自适应"换成更具体的、视觉端可解释的东西**——例如"按 wrist camera vs primary camera 的不同性质分别选不同数量的 token"，或者"按场景中检测到的物体数量分配 token 预算"——但这又回到工程复杂度的问题。

## 思路 2：聚焦"光线鲁棒性"

这条思路方向**对**，但**直接做"光线"这一个维度是不够的**。我刚搜了一下，这是当前的真实状态：

**已经有的工作**（你们要 framing 时必须知道）：

|论文|做了什么|跟你们的关系|
|---|---|---|
|**LIBERO-Plus** (2510.13626, 2025-10)|7 维扰动 benchmark，包括 light conditions。**Finding 6**：当前 VLA 对光照变化已经有一定鲁棒性，因为 wrist camera 提供光照不变的几何线索|这是你们必须正面回应的发现——它直接说光照不是当前 VLA 的主要短板|
|**RobustVLA** (2510.00037)|你们之前调研过。17 扰动里包括 lighting variation|已经把光照覆盖了|

3. **但有一个大风险**：LIBERO-Plus 已经发现 wrist camera 已经把光照问题缓解了——你们必须先验证这点是不是真的，否则做完发现"光照本来就没那么糟"，故事讲不下去

# 我的核心判断与建议

老师否决你们方案的真正原因，我猜不只是"工程量大"，更是**"主线讲不清楚"** —— AAC + RobustVLA + LoRA + capacity scan 四件事，每件都是合理的，但合起来听不出来"你们到底在解什么问题"。

老师推荐 FocusVLA，我猜传达的信号是：**"你看，人家用一个非常清晰的视角（视觉信息利用）+ 两个具体组件（cascaded attention + focus attention），讲了一个完整故事。你们也找这种清晰角度。"**

那么"只做鲁棒性"的真正约束是：**找一个角度 + 一个具体机制 + 一个清晰故事**，而不是把方向三那一摊东西全做完。

**我想提出第三条思路供你们考虑**，然后我们再一起聊。这条思路是上面两条的合体改良版：

### 思路 3 雏形：**"Focus-driven Robustness"——用 visual focusing 作为鲁棒性的诱因**

核心 hypothesis：**"VLA 对扰动脆弱的根本原因不是模型不够大，而是它在看不该看的东西。"** 这是 FocusVLA 已经埋下的伏笔，但他们没挖。

具体可能的切法（任选其一或组合）：

- **A. 只做"视觉端扰动 + Focus 机制"的因果分析**：把 FocusVLA 的 cascaded attention + focus attention 跑出来后，在 LIBERO-Plus 的 7 维扰动上系统评测，回答："哪些扰动 Focus 天然能解决（背景/distractor）？哪些不能（光照/相机视角/物体外观）？为什么？" → 这一篇可以是 **empirical + analysis paper**，工作量可控。
    
- **B. 给 Focus 机制加上"鲁棒性导向"的训练**：在 cascaded attention 训练时显式注入扰动 → 验证 focus pattern 在扰动下是否还能保持稳定 → 如果不能，怎么用一个轻量损失把它拉回来。这是方法学论文，但工作量明显比之前的方案小。
    
- **C. 聚焦"光照 + 物体外观"两个维度**（而非所有扰动）：因为这两个是 wrist camera 帮不了你的扰动（外观变化在 wrist view 一样存在）。这是上面思路 2 的优化版——不是只做光照，而是做"appearance-level perturbation"这一类。
