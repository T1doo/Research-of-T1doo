# Efficient VLA 立项 — 总览与立项建议

> [!info] 一句话推荐
> **以 SmolVLA (0.45 B) 为统一骨干，主线做"方向一 AAC 集成"，次线做"方向三 RobustVLA 移植"，并把"方向二 RC-NF 轻量监控"作为可选扩展。三个方向技术上正交、显存上无冲突、复现风险递增，能在 6-8 个月内产出 1 篇主论文 + 1 篇 workshop / 短文。**

## 1. 三方向对比矩阵

| 维度 | 方向一<br>动态自适应推理 | 方向二<br>轻量异常检测 | 方向三<br>数据高效鲁棒微调 |
|------|----------------------|--------------------|---------------------|
| 核心论文 | AAC (arxiv 2604.04161) | RC-NF (arxiv 2603.11106) | RobustVLA (arxiv 2510.00037) |
| 技术家族 | inference-time scheduling | side-car monitoring | training-time regularization |
| 是否动主干 | ❌（推理时算法） | ❌（旁路） | ✅（修改训练目标） |
| 训练显存增量 | 0 | 旁路单独训练（< 8 GB） | 中（worst-case 外内双循环） |
| 推理延迟增量 | +23 ms（N=20 采样） | +50-100 ms（含 SAM2） | 0 |
| 数据需求 | 复用既有 demo | 仅正样本 | 仅 25 demo 即可大涨 |
| 与 SmolVLA 耦合度 | **高**（async + chunk 天然耦合） | 中（需 SAM2 等外部模块） | **高**（直接替换损失函数） |
| 创新性空间 | 高（异步下的 AAC 触发） | **极高**（轻+轻文献空白） | 高（移植到小模型未做） |
| 复现风险 | 中（AAC 代码已开源，但靠 GR00T） | **高**（RC-NF GitHub 不明、SAM2 重量级） | **低**（RobustVLA 已开源完整 demo） |
| benchmark 对位 | LIBERO / CALVIN / RoboTwin | LIBERO-Anomaly-10 / LIBERO-Plus | LIBERO 17 扰动 / FR5 真机 |
| 与导师方向匹配 | 系统优化、real-time AI | 鲁棒性、安全 AI | adversarial robustness、PEFT |
| **本项目综合分** | **A**（最佳保底） | **B+**（高新颖但风险大） | **A-**（最易出活的方向） |

## 2. 推荐主线：**SmolVLA + AAC + RobustVLA** 的双方向贯通

### 2.1 为什么选这条主线

三个 agent 独立得出的最优切入点都指向 SmolVLA：

> - Agent 1（方向一）：**切入点 A — SmolVLA + AAC 集成**（1-2 周可跑 LIBERO 结果，保底 + 异步耦合的新颖性）
> - Agent 2（方向二）：**切入点 A — SmolVLA + RC-NF 轻+轻组合**（文献空白，单卡 < 24 GB）
> - Agent 3（方向三）：**切入点 E — π0 + LoRA + RobustVLA 双方向贯通**（方向一轻量化 + 方向三鲁棒）

把三个建议交叉后得到一个连贯的主线：

```
  [SmolVLA 0.45 B 骨干 (锁定)]
       │
       ├── 方向一注入：AAC entropy-driven chunk size adaptation（推理时）
       │     └── 配合 SmolVLA 异步推理，AAC 截断作为提前预测的额外触发条件
       │
       └── 方向三注入：RobustVLA input/output 双正则 + 多臂老虎机（训练时）
             └── 把 LoRA(r=16) 应用到主干，让 worst-case 外内双循环显存可控
```

**为什么把方向二降为可选扩展**：
- RC-NF 依赖 SAM2 (50 ms)，推理延迟会冲击方向一压低延迟的目标；
- RC-NF 的 GitHub 链接论文未明确给出，复现存在实质性风险；
- 但 RC-NF 与方向一/三的接口非常干净（side-car），如果立项中期有时间，可以作为 add-on 模块插入。

### 2.2 主线产出

**主论文**（目标会议：CoRL 2027 / IROS 2027）：
> *"Efficient and Robust Lightweight VLA: Adaptive Inference Scheduling Meets Worst-Case Regularization in 0.45 B Parameters"*
- 核心贡献 1：把 AAC 的 entropy-driven chunk 截断与 SmolVLA 异步推理队列耦合，形成新的推理调度算法；
- 核心贡献 2：在 SmolVLA + LoRA 上首次复现 RobustVLA 双正则化，验证鲁棒训练增益不依赖大模型容量（agent 3 假设）；
- 核心结果：LIBERO + LIBERO-Plus 上同时拿到推理延迟 ↓ 30% + 鲁棒扰动平均成功率 ↑ 10%+ 的双重提升。

**workshop / 短文**（方向二可选扩展）：
> *"RC-NF on a Diet: Plugging Robot-Conditioned Anomaly Detection into a 0.45 B VLA"*
- 把 RC-NF 挂到 SmolVLA，并把 SAM2 换成 MobileSAM/EfficientSAM；
- 端到端延迟从 100 ms 压到 40 ms 以内；
- 在 LIBERO-Anomaly-10 + LIBERO-Plus 双 benchmark 上评测。

## 3. 备选主线

### 备选 A：方向三独立深做（最稳）
- 选项：仅做 *SmolVLA + RobustVLA* 移植，省略方向一。
- 优点：复现风险最低（RobustVLA 完全开源 + SmolVLA 完全开源），4 个月内可见结果；
- 缺点：缺乏"系统创新"故事，论文 novelty 偏单点。

### 备选 B：方向一独立深做（最有系统感）
- 选项：仅做 *SmolVLA + AAC + A2A 三合一*（agent 1 切入点 E）。
- 优点：极致 efficiency 故事完整；
- 缺点：AAC 与 A2A 代码均偏新，工程量较大。

### 备选 C：方向二独立深做（最有差异化）
- 选项：仅做 *SmolVLA + RC-NF + 双臂 RoboTwin*（agent 2 切入点 A+C）。
- 优点：文献完全空白，差异化最强；
- 缺点：RC-NF 复现风险 + RoboTwin 双臂注入异常的工程量都很大，本科生周期内有失败风险。

### 备选 D：三方向贯通（最雄心）
- 选项：方向一 + 二 + 三 全部做。
- 优点：故事完整、可发顶会；
- 缺点：本科曦源 6-12 个月周期不太够，建议作为博士起点。

## 5. 风险与依赖

### 高风险
- **AAC 在 SmolVLA 异步推理下的触发耦合**：原 AAC 是同步采样，与 async queue 集成可能出现死锁或采样冲突。**缓解**：先做同步 AAC@SmolVLA baseline，再做异步耦合。
- **RC-NF GitHub 不可获取**：项目页有，但代码未明确开源。**缓解**：立项第一周邮件作者；若无回应，备选 A/B 立即启动。
- **LIBERO-Plus 官方代码与协议**：本次调研未拉到具体地址。**缓解**：通过 Allen AI VLA Harness 仓库获取或邮件作者。

### 中风险
- **SmolVLA LIBERO-Long 仅 71% 的能力上限**：可能即使集成 AAC 也只能拉到 80%，不如 GR00T+AAC 的 92.8%。**缓解**：报告"在小模型上的相对增益"，而非追求绝对 SOTA。
- **真机硬件依赖**：方向二/三的部分实验需要 SO100 / Franka / FR5。**缓解**：先用仿真做主结果，真机作为 stretch goal。

### 低风险
- **CALVIN/RoboTwin 集成成本**：仿真器有学习曲线，但社区文档充分。
- **RobustVLA 复现**：开源完整 + demo 视频充分，最稳。

## 6. 与导师方向匹配的建议

如果导师关注：
- **效率 / 实时系统 / 边缘部署** → 主线 + 方向一权重加重，强调 0.45 B + 1-step + AAC 异步组合；
- **鲁棒性 / 安全 / adversarial** → 主线 + 方向三权重加重，强调多臂老虎机 + worst-case + 真机 25-demo；
- **机器人本体 / 双臂 / 灵巧操作** → 备选 C 或 D，强调 RoboTwin 双臂场景的方法学创新；
- **大模型 / 基础模型 / 多模态** → 备选 D 或者直接换骨干为 π0.5 + LoRA，强调跨方向通用框架。

## 7. 评审委员会可能提的三个问题与回应

> **Q1**：为什么不直接用 π0 而是 SmolVLA？π0 是 SOTA，结果更有说服力吧？
>
> **A**：(1) π0 全参微调 46 GB 显存恰好是项目硬件上限，方向三 RobustVLA 的外内双循环优化会显著推高显存，本项目实际不可行；(2) SmolVLA 0.45 B 在 LIBERO 平均上已经与 π0 3.3 B 持平（87.3% vs 86.0%）、SO100 真机超越 π0（78.3% vs 61.7%），说明 SmolVLA 已经达到可信的 baseline 质量；(3) 立项主题是 "Efficient VLA"，SmolVLA 在精神和数据上都更契合，π0 留作对照实验组。

> **Q2**：你们方向三的鲁棒训练在 SmolVLA 上是否仍能拿到 RobustVLA 在 π0 上 +12.6% 的增益？这是关键假设。
>
> **A**：这是论文的核心假设之一，分三步回答：(1) RobustVLA 在 π0 (3.3 B) 和 OpenVLA (7 B) 上分别 +12.6% 和 +10.4%，**增益不随模型规模线性扩张**，说明鲁棒训练利用的是 loss landscape 几何而非容量；(2) Madry et al. 2018 等经典对抗训练文献显示小模型的鲁棒-精度 trade-off 反而更好；(3) 即使绝对增益略低，相对增益（25-demo 设置下）应仍显著。我们会在论文中报告完整的"增益 vs 模型规模"曲线作为关键消融。

> **Q3**：方向二的 RC-NF 复现风险这么高，为什么还放进调研？
>
> **A**：(1) RC-NF 是当前唯一一个把 "VLA + 异常检测" 做出 SOTA 的工作（AUC 0.9309 vs 最佳基线 0.8500），不放是失职；(2) 我们把它定位为"可选扩展"而非主线，主线不依赖 RC-NF；(3) 即使 RC-NF 代码不可获取，方向二的核心思想（轻量旁路 + normalizing flow + LIBERO-Anomaly benchmark）已经公开，本科生从零实现 RC-NF 也是合理的 stretch goal。
