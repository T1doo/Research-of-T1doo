# 方向二 · 基于轻量级模块的异常检测

> [!info] 立项导向
> 在大模型 VLA 主干（π0/π0.5，3.3B）之外挂一个轻量级旁路监控模块，用 Normalizing Flow 在正样本上做无监督密度估计，实时输出 OOD 分数，触发 state-level rollback 或 task-level replanning。该方向不动 VLA 主干、不需重训练大模型，参数量与显存增量极低（推理延迟 < 100 ms），是在 46 GB 显存约束下"为大模型补能力"的高性价比切入点。核心风险是 RC-NF 在 LIBERO 仿真之外的真实任务/双臂场景泛化性、以及阈值 α 选择带来的误报-漏报权衡。

## 1. 立项导向摘要

**方向定义**。本方向不试图改 VLA 主干，而是在已经部署好的 VLA（例如 π0、π0.5、SmolVLA）之外，再训练一个轻量旁路网络，用来在每个控制 tick 判断"现在 robot/object 的状态轨迹是否还在分布内"。一旦判定异常，向上层控制循环回报 OOD 信号，触发：(a) state-level rollback —— 让机器人执行 homing 回到初始姿态后等异常分数回落；(b) task-level replanning —— 通知 LLM 高层规划器或人类操作员重启任务。这是一种"插件式补丁"思路：把大模型当成一个不完美的策略黑盒，靠外挂监控器在线兜底。

**为什么对 Efficient VLA 立项有价值**。立项书的硬件约束是 batchsize=2 全参数微调 46 GB 显存，对于 3.3B 的 π0 这条线意味着只能用 80 GB A100/A6000 训练，几乎没有余量去同时跑两个 3B 级模型。轻量异常检测的优雅之处在于：(1) RC-NF 整体响应延迟 < 100 ms（SAM2 取 mask 50 ms + flow 推理 30 ms + 其他 ≤ 20 ms），相对 π0 自身 73 ms 的推理延迟，叠加后仍可支撑 ≥ 10 Hz 实时控制；(2) 训练只需正样本，无需重训 VLA，规避了 46 GB 显存做 VLA 反向传播的瓶颈；(3) RC-NF 在 LIBERO-Anomaly-10 上平均 AUC 0.9309 / AP 0.9494，比 GPT-5/Gemini 2.5 Pro/Claude 4.5/FailDetect 等基线最高 0.8500 高出 8 个百分点，方法本身已被验证 SOTA。

**RC-NF 与 π0/π0.5 baseline 的关系**。原论文明确把 π0 作为被监控的 VLA 之一（pi-zero 实际部署），说明 RC-NF 已经验证可挂在 π0 上。π0/π0.5 提供 action chunk（H=50，控制频率 20–50 Hz），而 RC-NF 在每个 tick 用 robot state（关节/位姿）+ SAM2 抽取的 object state（点云/mask）做异常评分。π0.5 引入 hierarchical 子任务预测（先吐 high-level subtask 文本再吐 action），RC-NF 的任务嵌入 τ 正好可以替换/复用 π0.5 的子任务表示，二者在数据接口上是天然契合的。

**关键风险**。(1) **真实任务覆盖**：LIBERO-Anomaly-10 的三类异常（gripper open / gripper slippage / spatial misalignment）相对工程化，真实任务的异常分布可能更长尾，复现时可能出现 AUC 在 in-domain 高、跨任务跳水的情况；(2) **依赖 SAM2**：object 分支用 SAM2 抽 mask，意味着推理时仍要跑一个分割大模型（50 ms），轻量化"轻"在哪里需要重新审视；(3) **阈值漂移**：α=0.05 分位数阈值要 task-by-task 标定，迁移到 RoboTwin/CALVIN 时需要重新收集正样本统计；(4) **开源不确定**：项目页 `heikaishuizz.github.io/RC-NF/` 已公开，但 GitHub 仓库链接尚未在论文正文中给出，复现成本未明。

## 2. 核心论文深度解析

### 2.1 RC-NF：机器人条件归一化流（核心论文）

**问题定义**。VLA 通过模仿学习训练，本质上只能在训练分布上工作。一旦在部署阶段出现：执行器物理故障（夹爪打滑）、动力学异常（夹爪没合拢）、空间扰动（任务从右后方搬到左前方），VLA 没有可靠机制感知"我执行错了"。直接让 VLA 自己回报置信度不靠谱——它本身是 OOD 的受害者。所以需要一个独立的、外挂的、概率密度可解释的监控器。

**方法**。RC-NF 用条件归一化流建模 `p(x | c)`：
- `x` 是 object state（SAM2 抽出的物体点集 / 形变向量）；
- `c = (s, τ)` 是条件，s 是机器人本体状态（关节、末端位姿），τ 是任务嵌入；
- 核心映射 `z = f_c(x)`，把 x 转到一个 task-conditioned 高斯潜空间 `z ~ N(μ_task, I)`；
- 对数似然：`log p_{X|C}(x|c) = log p_{Z|C}(z|c) + Σ log|det ∂f_{i,c}(y_{i-1})/∂y_{i-1}|`；
- 高斯先验项：`log p_{Z|C}(z|c) = Const − 1/2 · ||z − μ_task||²₂`。

训练阶段最大化对数似然，**只用正样本**，用 BalancedHardSampler 做数据去偏采样，共训 100 epochs。推理阶段：每个 tick 算 `log p(x_t | c_t)`，取负号当作异常分数；阈值 `Upper_T = μ_T + Q_{1−α}(D_T)`，其中 α = 0.05 是分位数阈值，需要 per-task 在正样本验证集上标定。架构上**关键的解耦**是：robot 分支和 object 分支分别参数化两个流，再交叉条件化，这样 robot state 提供"任务该走到哪"的先验，object state 用来检验"物体是否被正确操控"。

**LIBERO-Anomaly-10 benchmark**。覆盖原 LIBERO 全部 10 个任务，注入三类异常：
1. **Gripper Open（10 个任务全覆盖）**：夹爪保持张开，机器人继续按 VLA 输出执行，物体不会被抓起，期望 RC-NF 在抓取动作"应该闭合"时立刻报警；
2. **Gripper Slippage（10 个任务全覆盖）**：夹爪摩擦系数置零，抓起后立刻打滑，object 轨迹偏离任务参考；
3. **Spatial Misalignment（3 个派生任务）**：把任务的目标隔间从"后方"改为"左/前/右"，机器人朝错误方向走，object 不会进正确容器。

**实验与基线**。VLM 直接判定基线：GPT-5、Gemini 2.5 Pro、Claude 4.5（让多模态 LLM 看图像直接打 anomaly 分）；流模型基线：FailDetect（基于 flow matching 的失败检测器）。结果在所有异常类型上 RC-NF 全胜，且 LLM 类基线表现不稳定（成本高、延迟高）。

**关键数值**。
- 平均 **AUC 0.9309**（最佳基线 0.8500，差距 +0.0809）；
- 平均 **AP 0.9494**（最佳基线 0.8507，差距 +0.0987）；
- 端到端响应延迟 **< 100 ms**，其中 SAM2 取 mask 50 ms、RC-NF 推理 30 ms；
- 训练 epoch 数 100；
- 阈值 α = 0.05。

**消融**（在平均 AUC 上度量贡献度）：
| 消融 | 平均 AUC |
| --- | --- |
| 完整 RC-NF | **0.9309** |
| 去掉任务嵌入 τ | 0.8643（−0.0666） |
| 去掉机器人状态 s | 0.7152（−0.2157） |
| 去掉位置残差分支 | 0.8947（−0.0362） |
| 去掉动态形状分支 | 0.6841（−0.2468） |

可以看到：**动态形状分支（object 流）和机器人状态条件 s 是两个最重要的设计**，缺一个直接掉 22–25 个百分点；任务嵌入次重要；位置残差锦上添花。

**代码/数据**。项目主页 `https://heikaishuizz.github.io/RC-NF/`，GitHub 仓库链接未在论文中给出。LIBERO-Anomaly-10 作为新 benchmark 由作者一并发布。

**与 π0 集成**。论文在真实世界实验中明确把 RC-NF 作为 plug-and-play 模块挂在 π0 上，运行时 π0 正常生成 action chunk 控制机器人，RC-NF 并行读取 robot state + 摄像头帧（→ SAM2 → object state）算异常分数。一旦超阈值：
- **State-level rollback**（短期、便宜）：暂停 π0 输出，触发 homing procedure 让机器人回到初始姿态，等异常分数回落后把控制权交回 π0；
- **Task-level replanning**（重型、需要高层）：向 LLM 规划器或人类回报"任务失败"，要求重置环境、重新规划。

### 2.2 π0：Flow Matching VLA backbone（baseline）

**架构**。π0 = PaliGemma 3B VLM backbone + 300M action expert，采用 mixture-of-experts 设计，通过 blockwise causal mask 把 token 切成三段：images/language（图文）、state（本体状态）、actions（动作 token）。action token 用 flow matching 损失监督连续输出。π0-small 是 470M 的对照组（没有 VLM 初始化），用来证明大 VLM 的重要性。

**Flow matching 关键公式**：
- 噪声化动作 `A_t^τ = τ · A_t + (1 − τ) · ε`；
- 训练损失 `L^τ(θ) = E ‖v_θ(A_t^τ, o_t) − u(A_t^τ | A_t)‖²`；
- timestep 采样用 Beta 分布偏向低 τ（高噪声），cutoff `s = 0.999`；
- 推理用 forward Euler，**10 步**，δ = 0.1。
- action chunk 长度 **H = 50**，对应 50 Hz 控制可吐 1 秒动作。

**训练数据**。预训练混合约 **10,000 小时**：
- Physical Intelligence 内部数据 **903M timesteps**：单臂 106M、双臂 797M，覆盖 68 个任务 / 7 种机器人配置；
- 开源数据占 9.1%：OXE "Magic Soup" 子集 + Bridge v2 + DROID；
- 权重按 `n^0.43` 缩放对抗任务-机器人组合的样本不均衡。

**参数量与算力**。3.3B 总参（3B VLM + 300M action expert）。**LIBERO 表现在原论文未直接给出**——π0 的评测主打 Physical Intelligence 内部任务（laundry folding、bussing、box assembly），不是公开 benchmark。可参考的对比：在 shirt folding 上 π0 ≈ 0.95、π0-small ≈ 0.70、Octo ≈ 0.40、OpenVLA ≈ 0.30；bussing easy 上 π0 ≈ 0.80、π0-small ≈ 0.50、Octo ≈ 0.30、OpenVLA ≈ 0.20。

**推理延迟**（RTX 4090）：image encoders 14 ms + observation forward pass 32 ms + 10 步 action forward pass 27 ms = **总计 ~73 ms**，对应 UR5e/Franka 20 Hz、移动平台 50 Hz 控制。

**本项目可用性**。论文未承诺官方开源 checkpoint，但 Physical Intelligence 后续以 openpi 仓库形式放出了 π0/π0.5 的权重和训练代码（社区已大量复现）。3.3B 参数在 80 GB A100 上做 batchsize=2 全参数微调需约 46 GB，与项目硬件约束完全吻合。LoRA / Q-LoRA 路径可进一步压到 20 GB 以下。

### 2.3 π0.5：开放世界泛化（baseline）

**与 π0 的差异**。
- 两阶段训练：预训练用 **FAST 离散 token**（效率高），post-training 再接 flow matching action expert（轨迹平滑）；
- **Hierarchical 推理**：先吐 high-level semantic subtask 文本（如"pick up the plate"、"adjust the blanket"），再 condition 在子任务上吐低层动作；
- 显式预测 object 检测框，把视觉 grounding 显式化。

**Co-training 策略**。第一阶段训练样本中 **97.6% 不来自移动操作机器人**，混合 6 大类数据：
- MM（Mobile Manipulator）：~400 小时、~100 个家庭采集；
- ME（Multi-Environment）：非移动机器人，多场景；
- CE（Cross-Embodiment）：实验室数据，含 OXE；
- HL（High-Level）：人工标注的 semantic subtask；
- WD（Web Data）：图像描述、VQA、object localization；
- VI（Verbal Instructions）：post-training 阶段加，专家示范。

预训练 280k 步 + post-training 80k 步。

**长程任务能力**。论文展示 **10–15 分钟级**任务：完整清理一个厨房 / 卧室，在 **3 个完全未训练过的真实家庭**测试，是 end-to-end 学习系统首次达到该量级的开放世界泛化。语言遵从分别用 in-distribution 与 OOD 物体测试。模型在 3/12/22/53/82/104 个采集地点的 scaling 曲线显示位置数显著决定泛化能力。

**参数量**。论文未明确披露，但延续 π0 架构（VLM 主干 + 较小 action expert），整体仍在 3B+ 量级。openpi 实际放出的 π0.5 checkpoint 与 π0 同量级。

**推理延迟**。flow matching 同样 10 步去噪，50 Hz 控制 target pose + base velocity。具体毫秒数未单独标。

**与本项目硬件契合度**。参数量与 π0 同量级，因此 46 GB 显存的微调约束同样适用。π0.5 多出的 hierarchical 输出对方向二有额外价值：它显式吐 subtask 文本，正好可以作为 RC-NF 任务嵌入 τ 的来源，省去额外的 task encoder。

## 3. RC-NF 与 π0/π0.5 集成路径

RC-NF 在原论文实测中已经挂在 π0 真实部署上，集成方式是 **side-car**（旁路）而非 in-line：π0 的 forward 路径完全不动，RC-NF 只读 π0 看见的同一份观测（图像、机器人状态）和任务规范，并行计算异常分数。

**数据接口**。
- robot state s：直接复用 π0 输入的 `state` block（关节角、末端位姿、夹爪状态）；
- object state x：从相机帧出发，过 **SAM2**（50 ms）抽出目标 mask / 点集，再过 RC-NF 的 object 分支；
- task embedding τ：在 π0 上可以用语言指令的 PaliGemma text embedding，在 π0.5 上可以直接拿 hierarchical 输出的 subtask token。

**训练数据需求**。只要 π0 在某个任务上的 **成功 rollout 数据**。原论文 100 epoch 训练，BalancedHardSampler 平衡样本。对项目级复现来说，LIBERO 每个任务收 50–100 条成功轨迹就够起步，无需额外人工标注异常。

**推理时延叠加**。π0 自身 73 ms（一次 observation + 10 步 action），RC-NF 旁路 30 ms（实际可与 π0 forward 并行）+ SAM2 50 ms。最坏情况串行 ≈ 153 ms，对应 ~6.5 Hz；最优并行 ≈ 100 ms 对应 10 Hz。**关键工程问题**：SAM2 能不能换成更轻的分割器（如 EfficientSAM / FastSAM / MobileSAM）把延迟压到 20 ms 以下。

**是否需要修改 π0 主干**。完全不需要。这是该方向最大的工程优势 —— 兼容任意已部署的 VLA（π0、π0.5、SmolVLA、甚至 ACT/DP）。

```
            ┌───────────────────────────┐
            │ VLA 主干 (π0 / π0.5 3.3B) │ ──── action chunk (H=50) ───► 机器人
            └───────────────────────────┘
                  │ (共享观测)
                  ▼
            ┌──────────────────────────────────────┐
            │ RC-NF 旁路监控                       │
            │   • SAM2 抽 mask (50 ms)             │
            │   • Robot 分支 流(s)                 │
            │   • Object 分支 流(x | s, τ)         │
            │   • 算 log p(x|c) → anomaly score    │
            └──────────────────────────────────────┘
                  │
        ┌─────────┴─────────┐
        ▼                   ▼
  score ≤ 阈值        score > 阈值
  (正常,VLA 继续)     ├─► State-level rollback (homing,回正等回落)
                     └─► Task-level replanning (通知 LLM/人重启)
```

## 4. 在 Benchmark 上的评测路径

| Benchmark | 在方向二中的角色 | 备注 |
| --- | --- | --- |
| **LIBERO-Anomaly-10** | 直接对位评测，复现 RC-NF AUC 0.9309 / AP 0.9494 | 三类异常（gripper open / slippage / spatial misalignment），10 任务 |
| **LIBERO** | 评测被监控的 π0 / SmolVLA baseline 自身成功率，作为正样本来源 | SOTA 普遍 ≥ 98%，用来收集正样本训 RC-NF |
| **LIBERO-Plus** | 评测 RC-NF 在视觉/物理扰动下是否仍然鲁棒 | 用来检验异常分数是否对合理扰动稳定（避免误报）|
| **CALVIN** | 长程任务下 RC-NF 在任务边界处的表现 | 子任务切换是天然 OOD 高发期 |
| **RoboTwin** | 双臂场景扩展 | 论文未覆盖，是项目可挖的新颖点 |

推荐复现路径：先在 LIBERO + LIBERO-Anomaly-10 上对齐论文数字（验证训练 pipeline），再在 LIBERO-Plus 上做鲁棒性扩展实验，最后把 benchmark 自身扩到 RoboTwin。

## 5. 本项目可切入的研究点

### 切入点 A：在 SmolVLA (0.5B) 上集成 RC-NF —— 「轻 + 轻」组合（强推荐）
- **想法**：RC-NF 原论文只跨度过 π0 这种 3.3B 主干。把它挂到 0.5B 的 SmolVLA 上，看异常检测能不能补偿小模型的能力短板，形成"小 VLA + 监控器 → 接近大 VLA"。
- **复现成本**：低。SmolVLA 已开源，全参数微调显存 ~6 GB，配合 RC-NF（参数量 < 50M）总体可在 24 GB 单卡完成。
- **新颖性**：高。原 RC-NF 与 SmolVLA 文献中均未交叉评估，"轻量主干 + 轻量监控"是 Efficient VLA 主题最纯粹的实例。
- **硬件可行性**：完全可行，远低于 46 GB 上限。

### 切入点 B：把 RC-NF 的 Normalizing Flow 替换为更轻量的密度估计器
- **想法**：消融显示 dynamic shape branch 和 robot state branch 是性能主因，flow 本身可能 over-engineered。可尝试 VAE / Energy-based Model / Score-based 替换，看能否在 AUC 不掉 > 2 个百分点的前提下把推理压到 10 ms。
- **复现成本**：中。需要重写 object 分支，重训 100 epoch。
- **新颖性**：中等。属于"压缩论文核心模块"，可作为本科生工作合理范围。
- **硬件可行性**：训练显存 < 8 GB。

### 切入点 C：扩展 LIBERO-Anomaly-10 到 RoboTwin 双臂场景
- **想法**：原 benchmark 只覆盖单臂 + 3 类异常。双臂任务自然包含"协同失败"（一只手放开过早 / 另一只手撞上对方）新型异常，是 RC-NF 没碰过的领域。新建 RoboTwin-Anomaly-N 作为副产品发布。
- **复现成本**：中高。需要在 RoboTwin 中注入异常（参考 LIBERO-Anomaly-10 的三类思路）、跑通 SAM2、可能要重训 RC-NF 适配双臂 robot state。
- **新颖性**：高。双臂异常检测目前文献几乎空白。
- **硬件可行性**：训练显存 < 16 GB，推理 GPU 需求与 RC-NF 一致。

### 切入点 D：把 SAM2 换成 MobileSAM/EfficientSAM，压低 RC-NF 端到端延迟
- **想法**：当前 100 ms 延迟里 SAM2 占 50 ms。验证用 5–10 ms 级的轻量分割器能否把端到端做到 ≤ 40 ms（25 Hz），让 RC-NF 真正适配 50 Hz 移动操作场景。
- **复现成本**：低。只换分割模块，重新评测 AUC。
- **新颖性**：中。属于工程优化但对部署价值大。
- **硬件可行性**：完全可行，推理显存 < 4 GB。

### 切入点 E：用 RC-NF 异常分数做主动数据收集
- **想法**：RC-NF 不只是 fail-safe，反过来高异常分数的轨迹片段是 VLA 微调的"难样本"。把 RC-NF 当作主动学习的难度筛选器，仅用 OOD 片段重新微调 π0，看能否用 10% 数据量达到全量微调效果。这直接把方向二与方向三（鲁棒性微调）打通。
- **复现成本**：中。需要一轮 π0 微调，46 GB 上限内可行（LoRA 后更宽松）。
- **新颖性**：高。把"异常检测"从被动监控升级为主动数据策略。
- **硬件可行性**：用 LoRA + 80 GB A100 可跑通。
