# 完整科研技术路线

> [!info] 总规划
> 一次调研给的是季度粒度时间表（Phase 0-5，约 8 个月）；本文细化到**代码模块、实验配置、消融设计、go/no-go 决策点**。所有 file:function 级修改位置参见 [[03_SmolVLA代码可注入点分析]]，所有可借鉴方法参见 [[01_AAC相关工作扩展]] [[02_RobustVLA相关工作扩展]]。

## Phase 0 · 启动（2026-06，2-3 周）

### 0.1 输入

- 主线已批准（[[00_主线深度可行性分析]] §6 Go decision）
- 硬件：单卡 80 GB A100 / 48 GB A6000 / 24 GB 4090 任一（实验室确认）

### 0.2 具体步骤

```bash
# 1. 环境准备
conda create -n efvla python=3.11 -y
conda activate efvla
pip install torch==2.4 torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# 2. clone lerobot
git clone https://github.com/huggingface/lerobot.git
cd lerobot
git checkout c0a2e981  # Agent 3 验证过的 SHA，避免上游变动
pip install -e ".[smolvla]"

# 3. clone RobustVLA 参考（JAX/Flax 实现）
cd ..
git clone https://github.com/gakakulicc/RobustVLA.git

# 4. clone LIBERO
git clone https://github.com/Lifelong-Robot-Learning/LIBERO.git
cd LIBERO && pip install -e .

# 5. 下 SmolVLA 预训权重
huggingface-cli download lerobot/smolvla_base --local-dir ckpts/smolvla_base

# 6. 跑通 baseline
cd lerobot
python lerobot/scripts/eval.py \
  policy=smolvla \
  policy.path=../ckpts/smolvla_base \
  eval.env=libero \
  eval.task_suite=libero_spatial

# 7. 配 Allen AI VLA Eval Harness
git clone https://github.com/allenai/vla-evaluation-harness.git
# 按其 README 配 SmolVLA adapter
```

### 0.3 输出

- 跑通 SmolVLA 在 LIBERO-Spatial 上的复现，**目标成功率 ≥ 88%**（论文报告 90.0%，允许 ±2 pp）
- Allen AI Harness 接入，47× throughput 单卡 H100 / A100 上 18 分钟跑完 2000 episodes

### 0.4 Go/No-Go criteria

- ✅ Go：LIBERO-Spatial 复现 ≥ 88% & 显存 ≤ 实际 GPU 上限 80%
- ❌ No-Go：复现 < 80%（说明环境出问题，需排查）→ 退一步用 ACT/DP 简化 baseline 启动

### 0.5 风险

- huggingface-cli 下权重在国内可能慢，准备 mirror 或预先下载到本地
- lerobot 主分支可能 breaking，已 pin 到 SHA `c0a2e981`

## Phase 1 · AAC@SmolVLA（2026-07 中 ~ 08 中，6 周）

### 1.1 输入

- Phase 0 跑通的 SmolVLA baseline
- 一次调研 + Agent 1 + Agent 3 整理的 8 个 AAC 注入点（详见 [[03_SmolVLA代码可注入点分析]]）

### 1.2 代码模块

```
lerobot/policies/smolvla/
├── aac_sampler.py            ← 新建：N 次采样估计 entropy
├── aac_scheduler.py          ← 新建：h*=argmax 触发逻辑 + ξ 阈值
└── modeling_smolvla.py       ← Edit:
                              │  - SmolVLAPolicy.predict_action 加 N 采样调用
                              │  - VLAFlowMatching.sample_actions 暴露 sample stream
lerobot/scripts/
├── policy_server.py          ← Edit: receive_actions 加 must_go AAC hook
└── robot_client.py           ← Edit: control_loop_action 加 entropy 截断 prefetch
```

预计代码增量：~300 行（含单测）

### 1.3 实验设计

**1.3.1 主实验**

| 配置 | LIBERO-Spatial | LIBERO-Object | LIBERO-Goal | LIBERO-Long | 平均控制 Hz |
|------|---------------|---------------|-------------|-------------|------------|
| SmolVLA vanilla（baseline） | 90.0% | 96.0% | 92.0% | 71.0% | 20 |
| SmolVLA + RTC（参考社区） | ≥ 91 | ≥ 96 | ≥ 92 | ≥ 73 | 25 |
| **SmolVLA + AAC N=5（ours）** | ≥ 92 | ≥ 96 | ≥ 93 | ≥ 75 | 18 |
| **SmolVLA + AAC N=20（ours）** | ≥ 93 | ≥ 97 | ≥ 94 | **≥ 78** | 15 |
| **SmolVLA + RTC + AAC（最终方案）** | ≥ 94 | ≥ 97 | ≥ 94 | **≥ 80** | 30+ |

LIBERO-Long 的 71%→80% 提升是关键卖点（SmolVLA 论文承认的最大短板）。

**1.3.2 必做消融**

- AAC N 值 sweep：N ∈ {1, 5, 10, 20, 40}，找 cost-benefit 拐点
- AAC ξ 阈值 sweep：ξ ∈ {0.01, 0.05, 0.1, 0.2}（参考 AAC 原论文）
- AAC vs AQC（Agent 1 发现的 advantage-baseline 替代方案 arxiv 2605.05544）：替换 entropy → advantage 看是否消除 ξ 调参敏感性
- 异步触发模式：纯 AAC 截断 / 纯 must_go queue / 二者耦合

### 1.4 输出

- `01_AAC_SmolVLA_report.pdf`：5 页内部技术报告
- 代码合并到 lerobot fork `efvla-smolvla-aac` 分支
- 4 张关键 figure：N-sweep / ξ-sweep / 控制 Hz vs 成功率 trade-off / RTC × AAC 消融

### 1.5 Go/No-Go criteria

- ✅ Go：LIBERO-Long 成功率 ≥ 75%（比 SmolVLA 71% 至少 +4 pp）
- ⚠️ Conditional：LIBERO-Long 增量 < 4 pp 但 ≥ 2 pp → 重新审视 ξ / entropy 估计方法
- ❌ No-Go：所有 LIBERO 子集成功率均无显著提升 → 切换到 RTC + TT-RTC 路径（备选 B），跳过 AAC 创新

## Phase 2 · RobustVLA@SmolVLA（2026-09 ~ 10，8 周）

### 2.1 输入

- Phase 1 完成的 SmolVLA + AAC baseline
- 8 个 RobustVLA 注入点（参见 [[03_SmolVLA代码可注入点分析]]）
- RobustVLA JAX/Flax 参考实现 `RobustVLA/openpi/models/pi0.py`

### 2.2 关键工程：JAX→PyTorch 翻译（预留 1 周）

```
RobustVLA/openpi/models/
├── pi0.py:_compute_adversarial_action_loss  ← JAX，~50 行 PGD L∞
└── prompt_augmentation.py, ucb_augmentation_balancer.py  ← JAX/numpy，混合

目标产物（PyTorch）：
lerobot/policies/smolvla/
├── robust_loss.py             ← PGD inner max + outer min
├── ucb_balancer.py            ← 多臂老虎机选 17 arms
└── perturbation_ops.py        ← 17 种扰动算子（输入侧）
```

策略：
1. 翻译 `_compute_adversarial_action_loss` 为 PyTorch（最关键）
2. 单元测试：固定 input/seed，JAX vs PyTorch 输出差 < 1e-5
3. UCB balancer：参考 numpy 实现重写，加 z-score 归一化 + EMA 0.9
4. 17 种扰动算子：input-side 6 视觉 + 3 语言 + action-side 5 噪声 + env 3，对齐原论文

**Fallback**（若 JAX 翻译卡住超过 2 周）：用 RobustVLA 论文复现版（社区已有 1 个 PyTorch 端口在 arxiv:2510.00037 issue 区，立项时确认状态）。

### 2.3 代码模块（在 Phase 1 基础上）

```
lerobot/policies/smolvla/
├── robust_loss.py             ← 新建（见上）
├── ucb_balancer.py            ← 新建
├── perturbation_ops.py        ← 新建
└── modeling_smolvla.py        ← Edit: VLAFlowMatching.forward 加 worst-case δ
lerobot/configs/
└── policy/smolvla_robust.yaml ← 新建：双正则 + LoRA r=32 + Free-AT 备选

scripts/
└── train_robust.py            ← 新建：外内双循环 wrap（含 Free-AT 单循环 flag）
```

预计代码增量：~600 行

### 2.4 实验设计

**2.4.1 主实验：LIBERO 17 扰动**

复现 RobustVLA 原论文表 1 的扰动评测协议，把 backbone 从 π0 (3.3B) 换成 SmolVLA (0.45B)：

| 配置 | 干净 | 17 扰动平均 | clean-robust gap |
|------|------|-----------|----------------|
| SmolVLA vanilla | 87.3% | ~50%（估计） | -37 pp |
| SmolVLA + RobustVLA full FT（reference） | 86.5% | **≥ 75%** | -11 pp |
| **SmolVLA + RobustVLA + LoRA r=32（ours）** | 86% | **≥ 73%** | -13 pp |
| SmolVLA + RobustVLA + LoRA r=16 | 85% | ~70% | -15 pp |
| SmolVLA + Free-AT 单循环（fallback） | 86% | ~68% | -18 pp |

**核心 hypothesis**：LoRA r=32 RobustVLA 增益 ≥ full FT 的 90%（即 +14 pp / +15 pp）

**2.4.2 必做消融**

- **LoRA rank sweep**：r ∈ {8, 16, 32, 64}（Agent 2 强烈建议覆盖此范围）
- **Capacity scan**（Agent 4 建议）：SmolVLA 0.45B / π0 3.3B / OpenVLA 7B 三点曲线 — 验证"小模型增益相对值"假设
- **Data scarcity scan**（Agent 4 建议）：25 / 50 / 100 / 250 demo 上跑 RobustVLA，看少样本时小模型是否优势更大
- **input vs output robust 单独消融**：分别只开一个
- **UCB vs uniform vs curriculum**：3 种扰动采样调度

### 2.5 输出

- 代码合并到 `efvla-smolvla-robust` 分支
- 主 figure：3×3 grid（capacity × data × method）
- 关键 finding：robust-clean gap 在 SmolVLA 上是否更小

### 2.6 Go/No-Go criteria

- ✅ Go：LoRA r=32 RobustVLA 17 扰动平均 ≥ 73%（达到 full FT 90%）
- ⚠️ Conditional：60% ≤ 平均 < 73% → 启用 Free-AT 单循环 + LEAF 文本扰动替代
- ❌ No-Go：< 60% → 重审主线，可能需切换到 ACT/DP backbone

## Phase 3 · 联合实验（2026-11，4 周）

### 3.1 输入

- Phase 1 SmolVLA + AAC checkpoint
- Phase 2 SmolVLA + RobustVLA-LoRA-r32 checkpoint

### 3.2 关键实验

**3.2.1 联合训练 vs 顺序集成**

| 训练顺序 | LIBERO clean | LIBERO-Plus 17 扰动 | 控制 Hz | 训练耗时 |
|---------|-------------|-------------------|---------|---------|
| AAC 推理 only + Vanilla 训练 | 88 | 50 | 30 | 1× |
| RobustVLA 训练 + AAC 推理（独立）| 86 | 73 | 30 | 3× |
| **AAC 推理 + RobustVLA 联合训练** | 87 | **75** | 30 | 3× |
| 联合训练 + Free-AT fallback | 86 | 70 | 30 | 1.2× |

**3.2.2 训练-推理一致性消融（详见 [[00_主线深度可行性分析]] §3）**

- AAC@vanilla vs AAC@RobustVLA 的 chunk 长度分布对比

- δ 与 Σ 的 correlation 曲线（理论 contribution 实证）

### 3.5 Go/No-Go
