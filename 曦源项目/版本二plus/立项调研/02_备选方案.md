# 02 · 备选方案 · v2 only（FSF 退化）+ 多 Fallback 场景

> **核心姿态**：备选方案 = v2plus 主路触发严重失败时的回退路径。**所有 fallback 都共享 v2 的完整代码框架**（OpenVLA-OFT + LoRA + InSpire plugin + LIBERO-Plus 评测脚本），切换成本 < 1 周。

## § 2 · 主备选方案 · v2 only（FSF 退化）

### 2.1 触发条件

以下任一情况触发，立即退到 v2 only：

1. **N0 末**：4 个核心仓库中 ≥ 2 个装不通（FastVGGT、SAM2、GroundedSAM、OpenVLA-OFT）
2. **N1 末**：V3 clean SR 比 V1 低 > 10pp，且 β 调到 0.05 仍未恢复
3. **磁盘资源不足**：FastVGGT 缓存即使 fp8 量化仍 > 学校配额，且学校不允许外接存储

### 2.2 v2 only 方案细节

退到 v2 only 意味着：
- 关掉 V3 (+InSpire+FSF) 变体
- 仅保留 V1 (vanilla) 与 V2 (+InSpire) 2 变体
- 评测协议、H1/H3 假设完全同 v2

**与 v2 的区别**：经过 v2plus 的 P0-P1 P0-P1 投入后退回，已经多走了 ~3 个月，需要重新规划时间：

| Phase | 原 v2 时间 | v2plus 退到 v2 only 后 |
|---|---|---|
| P1 复现 | M1-M2 | M1-M3（已多花 1 个月） |
| P2 主评测 | M3-M5 | M4-M6 |
| P3 ablation | M6-M7 | M7-M8 |
| P4 failure predictor | M8-M9 | M9-M10 |
| P5 写作 | M10-M12 | M11-M12（1.5 月）|

**优势**：v2 只 2 变体，GPU-h 需求降低到 ~120h；可顺势把节省的预算补到真机或写作

### 2.3 v2 only 时的师哥沟通

如果发生 FSF 退化，需要主动与师哥沟通：
- "我们尝试了 FSF 但在 OpenVLA-OFT 上不收敛"
- "已确认 SF 论文方法在原 backbone (OpenVLA-OFT) 上的差异"
- "fallback 到 v2 + 重点跑真机评测，仍能交workshop short paper"

## § 3 · 子备选方案

### 3.1 Plan B-VGGT 原版（FastVGGT 失败时）

**触发条件**：FastVGGT 在 LIBERO 渲染图上 mIoU < 0.5，或与 VGGT 原版输出差异 > 10%

**应对**：
- 切到 VGGT 原版（github.com/facebookresearch/vggt）
- 离线缓存时间从 11 GPU-h 增到 35 GPU-h（VGGT 比 FastVGGT 慢 3-4×）
- 缓存大小不变（650 GB / fp8 325 GB）
- 训练侧无差异

**预算影响**：GPU-h +24h；其他不变

### 3.2 Plan B-VGGT-DP（VGGT 在合成图上质量差时）

**触发条件**：VGGT/FastVGGT 在 LIBERO 上的 depth 预测 mIoU < 0.5（即合成图 vs 真实图 domain gap 太大）

**应对**：
- 切到 VGGT-DP（arXiv:2509.18778）——专为 robot manipulation 训练的 VGGT 变体
- 或在 LIBERO 上 fine-tune VGGT teacher（小心梯度爆炸）

**预算影响**：可能需要在 LIBERO 上小规模 fine-tune VGGT，+10 GPU-h

### 3.3 Plan B-Focus Mask 单源（H5 失败）

**触发条件**：A3 ablation 显示 M-ABC 与最佳单源（M-A/M-B/M-C）差异 < 0.5pp

**应对**：
- 主提交改为单源 M-A（方向词 GT，最便宜与 v2 工具复用度最高）
- M-B（SAM2）与 M-C（DINOv2）进 discussion 章节作为 ablation 报告
- H5 改为 "focus（任何来源）> uniform"

**论文影响**：novelty 略降级；但仍有"focus > uniform"贡献

### 3.4 Plan B-真机降级（H4 失败）

**触发条件**：真机 V3 zero-shot SR < 30%，或扰动评测 V3 vs V1 反退化

**应对**：
- 真机改为 qualitative case study（10 episode/任务 × 3 任务 = 30 episode）
- 主投稿叙事仍以仿真为主，真机作为 "preliminary real-world results"
- H4 降级为 "qualitative real-world demonstration"

**论文影响**：失去 H4 这一核心贡献，但仍 viable workshop short

### 3.5 Plan B-真机数据不够（V3 fine-tune 退化）

**触发条件**：M8 真机 V3 在 50 demo 上 fine-tune 后 zero-shot SR < V1 - 10pp

**应对**：
- 方案 1：扩 demo 到每任务 100（×2 人工，约 5 天）
- 方案 2：纯 zero-shot 评测，不做 fine-tune（LIBERO 训好直接迁移）
- 方案 3：减少任务数（3→2 任务），但每任务 100 demo

## § 4 · 与 v2 备选方案的差异

| 维度 | v2 备选方案 | v2plus 备选方案 |
|---|---|---|
| **主备选** | FocusVLA 鲁棒性诊断（不训练） | v2 only（关 FSF） |
| **次备选数量** | 1（FocusVLA 诊断） | 4（VGGT原版/VGGT-DP/Mask单源/真机降级） |
| **切换成本** | < 1 周 | < 1 周（fallback 路径都共享 v2 代码） |
| **触发节点数** | 2（M2、M5） | 5（N0/N1/N2/N3/N4） |

**关键设计**：v2plus 的 fallback 路径**层级化**——任一组件失败都有独立 fallback，**最坏情况退到 v2 only**，再坏退到 v2 的备选（FocusVLA 诊断）。

## § 5 · 切换决策矩阵

| 节点 | Go 条件 | No-Go 应对 | 决策时间 |
|---|---|---|---|
| **N0** | 4 仓库装通 + FastVGGT mIoU ≥ 0.5 | 缺 ≥ 2 → v2 only; FastVGGT 差 → 原版 | 当天 |
| **N1** | V3 clean SR ≥ V1 - 3pp & VGGT 缓存完整 | 退化 > 10pp → v2 only | M3 末 |
| **N2** | H1 ≥ 2 维度显著 & H5 M-ABC > M-None | 完全无增益 → negative finding | M6 末 |
| **N3** | H4 ≥ 5pp | 退化 → case study 降级 | M9 末 |
| **N4** | AUC ≥ 0.65 & 论文数据齐 | 不齐 → 转 ICLR/CCC | M11 末 |

每次 No-Go 后给 1-2 周 buffer 重试，仍不通则切 fallback。
