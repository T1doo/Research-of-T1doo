# 00 · 版本二plus 总览与立项决定

> **项目代号**：**FocusedSpatialForcing for VLA Robustness (FSF-VLA)**
> **主题名**：**基于聚焦式 3D 表征对齐的视觉-语言-动作模型扰动鲁棒性实证研究**
> **一句话定位**：在版本二的"OpenVLA-OFT + InSpire 显式方向词提示"之上，新增 **Focused Spatial Forcing (FSF) 通道**——以冻结的 FastVGGT 为 teacher，对 VLA Layer 24 视觉 token 做余弦相似度对齐；对齐损失的 per-token 权重由 **方向词 GT × DINOv2 CLS attention 双源融合的 focus mask** 调制；前景 1.0、上下文 0.5、背景 0.1；在 LIBERO-Plus 7 维仿真扰动 + 真机 3 维扰动两个轴系统验证 5 个核心假设。
> **设计原则（v2 纯净版，2026-05-27 设计 review 确认）**：focus mask 来源仅使用 v2 体系内工具——方向词 GT 推导（v2 复用）与 DINOv2 CLS attention（已在 OpenVLA-OFT vision encoder 中，免费）。**不引入 SAM2 / GroundedSAM**（属 v3 工具链），保持 v2 → v2plus 是最简增量升级路径。

## 1. 为什么要切换到版本二plus

### 1.1 项目立项的三次迭代

T1doo 团队的立项方向已经历三次迭代：

| 版本 | 方向 | 状态 | 备注 |
|---|---|---|---|
| **版本一** | Efficient VLA · AAC + RobustVLA + RC-NF 三方向耦合 | ❌ 已舍弃 | 导师反馈"耦合实现难度高、一年期产出风险大" |
| **版本二** | FocusVLA × InSpire 互补研究（2 变体 + 视觉利用 ablation） | ✅ 师哥确认"方向可行" | 详见 `版本二/立项调研/08_最终方案_FocusVLA_InSpire互补研究.md` |

| **版本二plus**（**本文档**） | 版本二 + Focused Spatial Forcing（3D 表征对齐） | ⏳ 立项中 | 本目录 |

### 1.2 师哥 2026-05-27 讨论的核心建议

师哥在与团队 2026-05-27 的讨论中，对版本二给出了**明确的方向认可**与**3 条改进建议**：

> **师哥反馈摘要**（原话）：
> 1. "感觉加 3d 是个好想法，但是这个会一定程度上增大工作量，**按照 inspire 的会，特别是要在真机上验证的话**，因为真机数据需要自己来标注了。不过我们可以接受，所以你要探索一下怎么加 3d。"
> 2. "https://arxiv.org/abs/2510.12276 ，可以参考一下这篇，**隐式的 3d 表征对齐**。版本二的参考论文 **FocusVLA 中也用了 vggt**，可以看下他们做法是不是有区别。"
> 3. "总结来说，一个大概的思路是：**spatial forcing 提供 3d 表征监督，但是需要 focus 在重要的位置。一些不相关的背景提供的监督可以更弱点。**"
> 4. "**vggt 的话是 cvpr best paper**，后面出了几个续作 也可以大致看下，对我们这个新版本的思路有没有启发和借鉴意义。"

### 1.3 团队的回应

基于师哥建议，团队做了如下三件事，形成版本二plus：

1. **深度精读 [Spatial Forcing](https://arxiv.org/abs/2510.12276)（Li et al. 2025-10, ICLR 2026 投稿）**：弄清楚"隐式 3D 表征对齐"的具体技术——VGGT 作为冻结 teacher，对 VLA Layer 24 视觉 token 做余弦相似度对齐，仅 ~30 行代码，训练加速 3.8×、数据效率提升 5.9×。
2. **对比 FocusVLA 中 VGGT 的用法**：发现 FocusVLA 把 VGGT 作为**与 SigLIP/DINOv2 并行的第三个 encoder**，**集成在 policy 阶段**——作者自述"VGGT 梯度弱且不稳定，性能上限受限"。这是与 Spatial Forcing 截然不同的（劣等）用法。
3. **调研 VGGT 续作生态**：VGGT (CVPR 2025 Best Paper, arXiv:2503.11651) 后已有 **12+ 个续作**——FastVGGT、StreamVGGT、π³、HD-VGGT、VGGT-Long、InfiniteVGGT、VGGT-DP、3D-Mix for VLA、AugVLA-3D、SceneVGGT 等。其中 **FastVGGT (arXiv:2509.02560, ICLR 2026)** 提供训练无关 4× 加速 + 无损精度，是最适合的 teacher 实现。

**核心融合点**：师哥的"focus 在重要位置 + 背景弱监督"思路，与 Spatial Forcing 的"均匀监督"形成自然增量——这是版本二plus 命名 **FocusedSpatialForcing (FSF)** 的由来。

## 2. 版本二的现状与版本二plus 的增量

### 2.1 版本二的核心方法学

版本二（v2 修订版）以 OpenVLA-OFT-7B 为统一主干，做 2 个模型变体（vanilla / +InSpire）的 LIBERO-Plus 7 维扰动评测：

- **变体 1**：vanilla OpenVLA-OFT + LoRA(r=16)，原 LIBERO instruction
- **变体 2**：同上 + 前置方向词 prompt + 7-class direction head + 联合 loss $\mathcal{L}_{action} + 0.1 \mathcal{L}_{direction}$
- **方向词 GT** 从 demo 的 `robot0_eef_pos` 与 `object_pos` 自动推导（6 方向 + grasped = 7 类）
- **视觉利用 ablation**：token dropout / patch mask / attention 可视化（推理时进行，非训练）
- **3 个假设**：H1（鲁棒性增益）/ H2 与 FocusVLA 叠加（v2 降级为 discussion）/ H3（failure predictor）

### 2.2 版本二plus 的 4 项增量

| # | 增量 | 师哥提示的对应 |
|---|---|---|
| **△1: FastVGGT teacher** | 引入冻结 FastVGGT 作为 3D 表征 teacher | "VGGT 是 CVPR best paper" + "FocusVLA 也用了 VGGT 但做法不同" |
| **△2: Focused Spatial Forcing loss** | 在 OpenVLA-OFT Layer 24 加 FSF loss（继承 Spatial Forcing 论文范式） | "参考 arXiv:2510.12276 隐式 3D 表征对齐" |
| **△3: 双源 focus mask** | 方向词 GT × DINOv2 CLS attention 双源融合，背景弱监督 0.1（v2 纯净路径，**不引入 SAM2**） | "spatial forcing 需 focus 在重要位置，背景可以弱点" |
| **△4: 真机评测** | 3 任务 × 50 demo × 3 扰动维度 = 360 episode | "按 InSpire 的会，要在真机上验证" |

**变体设计调整**：从 v2 的 2 变体扩到 **3 变体** —— V1 (vanilla) / V2 (+InSpire) / **V3 (+InSpire+FSF, v2plus 主创新)**。

### 2.3 版本二plus 的核心 loss

$$\mathcal{L}_{total} = \mathcal{L}_{action} + \alpha \cdot \mathcal{L}_{direction} + \beta \cdot \mathcal{L}_{FSF}$$

$$\mathcal{L}_{FSF} = -\frac{1}{\sum_{i=1}^{N} m_i} \sum_{i=1}^{N} m_i \cdot \cos\Big( \text{MLP}(\text{BN}(x_i^{V,L24})) + E_i, \; f_i^{3D,L23} \Big)$$

其中：
- $\alpha = 0.1$（继承 v2 InSpire）
- $\beta = 0.5$（继承 Spatial Forcing 原论文最优值）
- $m_i \in [0.1, 1.0]$（双源融合后的 token 焦点权重；背景 0.1 / 上下文 0.5 / 前景 1.0）
- $x_i^{V,L24}$：OpenVLA-OFT Llama2-7B Layer 24 第 $i$ 个视觉 token hidden state（4096-dim）
- $f_i^{3D,L23}$：FastVGGT aggregator Layer 23（penultimate）输出第 $i$ 个 token feature（1024-dim）
- BN + 2-layer MLP 投影对齐维度；2D sinusoidal PE 保留 spatial 顺序

详见 [[01_主方案_FocusedSpatialForcing]] § 1 与 [[08_最终方案_FSF详细技术报告]] § 3。

## 4. 选定方案（双轨）

### 4.1 主方案 · FocusedSpatialForcing (FSF) on OpenVLA-OFT

**核心姿态**：v2 框架完全保留 + 增量 3 个组件（FastVGGT teacher、FSF loss、双源 focus mask）+ 增量 1 个评测轴（真机）。**不替换 v2 任何模块，只新增**。

**研究问题**：在 LIBERO clean + LIBERO-Plus 7 维扰动 + 真机 3 维扰动下，**隐式 3D 表征对齐（FSF）与显式方向词提示（InSpire）的叠加，是否能进一步提升 VLA 模型的扰动鲁棒性？双源融合 focus mask 是否优于均匀监督？**

**详细设计**：[[01_主方案_FocusedSpatialForcing]]、[[08_最终方案_FSF详细技术报告]]

### 4.2 备选方案 · v2 only（FSF 退化）

仅在 **N1（M3 末）V3 (+InSpire+FSF) 相对 V1 退化 > 10pp** 或 **N0（M0 末）FastVGGT 在 LIBERO 上 mIoU < 0.5** 时启动。技术上把 V3 关掉 FSF 通道，恢复 v2 的 2 变体方案。

**关键**：与主方案**共用 v2 的全部代码框架**（OpenVLA-OFT + LoRA + InSpire plugin + LIBERO-Plus 评测脚本），切换成本 < 1 周。

**详细设计**：[[02_备选方案]]

### 4.3 排除方案

- **真改 FocusVLA 架构**（重写 Modality Cascaded Attention + Focus Attention） → 工程超纲，与 v2 已舍弃同理由
- **复现 PointVLA / SpatialVLA 等显式 3D 输入路线** → 需深度传感器或显式 3D 监督数据集，本科一年期承担不起
- **预训练新 3D-aware backbone**（如 VIPA-VLA 路线） → 算力门槛 > 100 GPU-day，超出 ¥10K 预算
- **把 FSF 改为对 VLA 多层都做对齐**（multi-layer SF） → 工程量翻倍，Spatial Forcing 论文已实证 Layer 24 单层最优

详见 [[07_为什么不选其他方案]]。

## 6. 核心研究问题（5 个 H 假设速览）

| 假设 | 命题 | 证伪条件 | 验证方式 |
|---|---|---|---|
| **H1（仿真鲁棒性增益）** | V3 (+InSpire+FSF) 在 LIBERO-Plus 至少 2 个扰动维度上 SR drop 比 V1 (vanilla) 小 ≥ 5pp（p < 0.05），预期在 viewpoint 与 state 显著 | 0 维度显著 or 全维度 p ≥ 0.1 | 21 cell × 3 seed × 300 inst 配对 t-test (Bonferroni) |
| **H2（叠加效应）** | V3 平均 SR > max(V2, +FSF only via M-None) + 1.5pp | 完全无叠加 | A3 ablation 中 M-None 作为 +FSF only 代理 |
| **H3（failure predictor，继承 v2）** | 方向词时序训练 LightGBM 二分类器预测最终成败 AUC ≥ 0.65 | AUC < 0.6 | 5-fold CV，基于 2k 配对 episode |
| **H4（真机鲁棒性增益）** | V3 在真机 9 cell 平均 SR drop 比 V1 小 ≥ 5pp（p < 0.10，真机 sample 少 → p 放宽） | drop 差 < 3pp or V3 反退化 | 360 episode 真机评测，配对 t-test |
| **H5（双源 focus 优于单源/均匀，v2plus 核心创新）** | M-AC（双源融合）> max(M-A, M-C) + 1pp，且 M-AC > M-None（SF 原版均匀）+ 1.5pp（p < 0.05） | M-AC 与最佳单源差 < 0.5pp | A3 mask 来源 ablation × 3 维子集 × 3 seed × 200 inst |

**关键工程同构**：5 个假设的验证都建立在 (i) OpenVLA-OFT + LoRA 训练（v2 已通），(ii) FastVGGT teacher 离线缓存（v2plus 新增），(iii) Focus mask 双源管线（方向词 GT 投影 + DINOv2 attention，**均在 v2 工具体系内**），(iv) LIBERO-Plus 评测脚本（v2 已通），(v) 真机推理与人工评测协议（v2plus 新增）。

详见 [[01_主方案_FocusedSpatialForcing]] § 2 与 [[08_最终方案_FSF详细技术报告]] § 4。
