# 05 · 风险与缓解

> **整体姿态**：v2plus 比 v2 多了 4 个新增组件（FastVGGT teacher + FSF projector + 多源 Focus mask + 真机数据），新增风险约 4 个。每个风险都有明确的触发条件、缓解策略、与 fallback 路径。**最坏情况退到 v2 only**，再坏退到 v2 的备选（FocusVLA 纯诊断），切换成本 < 1 周。

## § 1 · Top 7 风险（按严重度排序）

### 风险 R1 · VGGT 离线缓存超磁盘 / 缓存失败

**严重度**：🔴 高 · **概率**：30%

**影响**：650 GB 是 v2 原本不需要的存储；学校存储配额可能不够；离线缓存失败则训练 GPU-h × 3-5

**触发条件**：
- P0 末发现学校共享存储无 650 GB 配额
- WebDataset 在多进程 DataLoader 上 IO 瓶颈
- FastVGGT 在 LIBERO 4 suite 全量 pre-extract 失败（OOM/超时）

**缓解（按优先级）**：
1. **fp8 量化**：650 GB → 325 GB（精度损失 < 1%，P0 末验证）
2. **仅缓存 LIBERO-Spatial 1 个 suite**：325 → 81 GB；先单 suite 验证 FSF
3. **学校大存储节点 / 阿里云 OSS 冷存储**：约 ¥30/月 / 100GB；1 年 ¥360 < ¥400 实验耗材预算
4. **streaming VGGT 推理**：训练时即时跑 FastVGGT；GPU-h × 3-5 但占用磁盘 0
5. **fallback**：完全放弃 FSF，退到 v2 only

**决策树**：

```
P0 末检查磁盘 → 
  ≥ 650 GB：直接缓存 bf16
  300-650 GB：fp8 量化（先验证精度）
  100-300 GB：仅缓存 1 suite 试跑
  < 100 GB：streaming 或退 v2 only
```

### 风险 R2 · FSF Loss 在 OpenVLA-OFT 上不收敛或退化 SR

**严重度**：🔴 高 · **概率**：30%

**影响**：核心创新方法不 work，v2plus 退化到 v2

**触发条件**：
- M3 末 V3 vs V1 clean SR 退化 > 10pp
- FSF loss 在 1000 step 后未下降 50%
- 训练 loss 出现 NaN

**缓解（按优先级）**：
1. **预先验证（P0 末）**：在 5 个 LIBERO-Spatial demo 上跑 100 step FSF 训练，loss 应下降 ≥ 30%
2. **β 扫描**：β 从 0.05 起步，2× 增到 0.5（额外 1-2 天训练）
3. **逐层冻结**：FSF 训练时先冻结所有 LoRA，仅训 projector；再解冻 LoRA
4. **warmup**：FSF loss 在前 1000 step 用 β=0（仅训 LoRA），再线性 ramp 到 β=0.5
5. **fallback**：若 M3 末 V3 退化 > 10pp，切到 V2 only（成 v2 路线）

**决策树**：

```
M3 末 V3 vs V1 clean SR diff →
  ≥ -3pp（≤3pp 退化）：进 P2
  -5 to -10pp：调 β + buffer 2 周
  > -10pp：切 v2 only
```

### 风险 R3 · Focus Mask 多源融合无显著差异（H5 失败）

**严重度**：🟡 中 · **概率**：40%

**影响**：H5 是 v2plus 的方法学贡献核心，失败则方法学 novelty 降级

**触发条件**：
- A3 ablation 显示 M-ABC 与最佳单源（M-A/M-B/M-C）差异 < 0.5pp
- M-ABC 与 M-None（SF 原版均匀）差异 < 0.5pp

**缓解（按优先级）**：
1. **多组 ablation 找 sweet spot**：F1/F2/F3/F4 4 种融合，bg_w 4 个值，fg/ctx/bg 阈值 3 组
2. **改叙事**：哪怕 M-ABC 与 M-A 等价，仍可论证"focus 比 uniform 好"，仍有 contribution
3. **fallback**：主提交改单源（M-A 最便宜，与 v2 工具复用度 100%），M-B/M-C 进 discussion

**预期改叙事**：
- H5 原版："多源 > 单源"
- H5 降级："任何 focus 都比 uniform 好"
- H5 完全失败："focus mask 与 uniform 等价" → 改回 SF 原版做主路

### 风险 R4 · 真机数据 50 demo/任务不够（fine-tune 退化）

**严重度**：🟡 中 · **概率**：45%

**影响**：V3 在真机上 zero-shot 直接评测 SR < 30%，无法验证 H4

**触发条件**：
- M8 真机 V1 zero-shot SR < 30%
- M8 真机 V3 fine-tune 后 zero-shot SR < V1 - 10pp

**缓解（按优先级）**：
1. **主路**：LIBERO 训好 LoRA + 真机 50 demo 仅做 calibration fine-tune（10-20 step），不全调
2. **fallback 1**：扩到每任务 100 demo（×2 人工 = 10 天）
3. **fallback 2**：仅 zero-shot 真机评测（不 fine-tune），LIBERO LoRA 直接迁移
4. **fallback 3**：真机改为 qualitative case study（10 episode/任务），不做统计
5. **极端 fallback**：减少任务数（3→1 任务），仅 1 任务上做 360 episode 评测

### 风险 R5 · FastVGGT 在 LIBERO 合成图上质量差

**严重度**：🟡 中 · **概率**：25%

**影响**：teacher feature 不可靠，FSF 监督失效

**触发条件**：
- P0 末 5 张 LIBERO 渲染图上 FastVGGT depth map 与 GT depth mIoU < 0.5
- VGGT 原版 vs FastVGGT 输出差异 > 10%

**缓解（按优先级）**：
1. **P0 严格验证**：5 张 LIBERO 渲染图 + 5 张真机图，目视 + 数值（depth map quality）对比
2. **fallback 1**：换 VGGT 原版（不要 FastVGGT 加速；GPU-h +24h）
3. **fallback 2**：换 VGGT-DP（arXiv:2509.18778，专为机器人训练的 VGGT 变体）
4. **fallback 3**：在 LIBERO 上小规模 fine-tune VGGT teacher（小心梯度爆炸，+10 GPU-h）
5. **极端 fallback**：用 DINOv2 self-supervised feature 替代 VGGT（但失去 3D 含义，FSF 降级为 self-distillation）

### 风险 R6 · 团队工程能力跟不上（2 大二本科生）

**严重度**：🟡 中 · **概率**：30%

**影响**：P1 训练 pipeline 不通；FSF 集成卡壳；进度延迟 1-2 月

**触发条件**：
- W2 末（M0 D14）OpenVLA-OFT 还没 forward 跑通
- M1 末 V1 LoRA 没在 LIBERO-Spatial 上跑出 ≥ 90% clean SR
- M2 末 FSF projector 代码不能 training

**缓解（按优先级）**：
1. **前置学习**（2026-05 末完成）：
   - PyTorch transformer block 基础
   - HuggingFace peft 库 LoRA 使用
   - OpenVLA-OFT README 完整走完
   - 阅读 Spatial Forcing 官方代码
2. **AI 辅助开发**：API 预算 ¥800 用于 P0-P1 期间代码 review（Claude/GPT）
3. **导师 office hour**：每月 1 次过工程关键决策
4. **社区求助**：HuggingFace LeRobot Discord、知乎、Reddit r/MachineLearning
5. **每周复盘**：30 分钟团队会同步进展与阻塞

### 风险 R7 · LIBERO-Plus 评测耗时超预算

**严重度**：🟢 低 · **概率**：50%

**影响**：GPU-h 超出 230h 预算

**触发条件**：
- P2 末 GPU-h 已用 200h+
- 主评测 21 cell 跑完后 ablation 预算不够

**缓解（按优先级）**：
1. **分批跑**：每天跑 1-2 维度，避免一次性占大量 GPU-h
2. **优先维度**：viewpoint/state（最可能验证 H1）先跑，texture/light 后
3. **降 instance**：500 → 300（已采纳；power 仍 0.7+）
4. **再降 instance**：300 → 200（power 仍 0.6+）
5. **stretch ablation 砍**：A7/A8 推到论文 stretch 章节，不进主对比

## § 2 · 立项评审风险（评委可能问什么）

| Q | 答案要点 |
|---|---|
| **Q1: 为什么加 3D？v2 不够吗？** | (1) v2 暴露 viewpoint/state 维度是 VLA 最弱链；(2) 方向词是 7 类离散，无法表达细粒度 3D；(3) SF 原论文已证 +1.4pp、训练加速 3.8×、数据效率 5.9×；(4) 师哥讨论后明确认为这是 promising direction |
| **Q2: 为什么用 VGGT 而不直接用深度图？** | (1) SF 论文实证：显式 depth estimator 引入误差；(2) LIBERO 仿真 vs 真机 depth 分布不一致；(3) VGGT 隐式表征对齐更鲁棒、无额外深度估计开销 |
| **Q3: 为什么是 focus 而不是 dense alignment？** | (1) SF 原文均匀监督在 LIBERO 上有效，但**对 viewpoint 扰动并未充分提升**；(2) 师哥根据物理直觉建议；(3) AttentionVoxel 等已证 task-relevant 区域的 3D 监督最有效；(4) H5 ablation 直接验证 |
| **Q4: 真机评测会不会太重？** | (1) 按 InSpire 论文真机协议，3 任务 × 50 demo 在 1 周内可完成；(2) 实验室已有真机硬件；(3) 仅 1.5 天人工执行评测；(4) 预算上调 ¥0（实验室硬件免费） |
| **Q5: 2 人本科生能搞定？** | (1) 渐进升级（v2 全部组件保留）；(2) FSF 集成是 ~30 行 SF 原论文证明的修改；(3) Source A focus mask 与 v2 工具 100% 复用；(4) 最大新工作量在真机数据采集（约 1 个月） |
| **Q6: 与 FocusVLA 中的 VGGT 有何不同？** | (1) FocusVLA 用 VGGT 在 policy 阶段并行 encoder，**自述"梯度弱且不稳定"**；(2) Spatial Forcing 是中间层对齐（Layer 24）；(3) v2plus 继承 SF 的位置，再加 focus mask 解决师哥指出的"不相关背景监督要弱"问题；(4) H5 ablation 直接验证融合策略 |
| **Q7: focus mask 的 ground truth 怎么定义？** | (1) 多源融合（方向词 GT × SAM2 × DINOv2 CLS attention）；(2) 不是单一真值，是任务相关性的 soft signal；(3) H5 ablation 验证多源 > 单源 |
| **Q8: 跟现有 3D-aware VLA 工作有何区别？** | (1) 不需要显式 3D 输入（vs PointVLA, SpatialVLA 等）；(2) 不需要重训 VLA backbone（vs GP3, VIPA-VLA 等）；(3) 隐式对齐 + 多源 focus 在文献中是空白 |
| **Q9: H1-H5 假设会不会过多？** | (1) H1/H3 完全继承 v2，无额外验证压力；(2) H2 是 discussion-level；(3) H4 与 H5 是 v2plus 的核心新假设；(4) 不会全失败（最差 H4 H5 失败仍有 H1 H3） |

## § 3 · Plan A → Plan B 切换决策树（5 个 fallback 场景）

```
场景 1: VGGT 离线缓存失败/超磁盘
  → 改用 streaming 推理（每 batch 即时跑 FastVGGT），GPU-h ×3 但少占磁盘
  → 或仅缓存 LIBERO-Spatial 1 个 suite，先单 suite 验证 FSF

场景 2: FSF 在 OpenVLA-OFT 上 SR 退化 > 10pp（N1 触发）
  → β 调小 0.5 → 0.1 → 0.05
  → 仍退化：切 Plan B = 仅 V1+V2（回 v2 完整方案）

场景 3: Focus mask 多源融合无差异（H5 失败）
  → 改叙事："任意 focus mask 都比均匀好" → 仍是有意义结论
  → 主提交改为单源 M-A（最便宜的）

场景 4: 真机评测时间不够（M8 末仅完成 60% episode）
  → 真机扰动维度从 3 降到 2（去掉初始位置）
  → 或单任务 × 3 扰动（去 T2/T3）

场景 5: H1 仿真无显著增益（N2 触发）
  → 改叙事：FSF 是"训练效率工具"而非"鲁棒性工具"，主推 SF 原论文的 3.8× 加速 + 5.9× 数据效率
  → 投稿 venue 改为 Robot Learning workshop 而非 main
```
