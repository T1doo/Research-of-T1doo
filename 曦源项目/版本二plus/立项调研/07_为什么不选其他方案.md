# 07 · 为什么不选其他方案

> **本文档对照所有可能的 v2plus 替代方案**，逐一说明排除理由。便于在与师哥/导师讨论或评审答辩时回答"为什么不选 X？"的问题。

## § 1 · 总览：排除方案的 5 大类

| 类别 | 替代方案 | 排除理由 |
|---|---|---|
| **A. 真改架构类** | A1: 真复现 FocusVLA Cascaded + Focus Attention | 工程超纲（4 周架构改造），与 v2 相同理由排除 |
| | A2: 真改 VLA backbone 加 3D head | 算力门槛 > 100 GPU-day，超 ¥10K |
| **B. 显式 3D 输入类** | B1: PointVLA / SpatialVLA 路线 | 需要显式深度或点云传感器；LIBERO 仿真有但真机难统一 |
| | B2: DepthVLA / 3D-CAVLA 路线 | 显式 depth head + 联合训练；增加架构复杂度 |
| **C. VGGT 用法替代类** | C1: 仿 FocusVLA 把 VGGT 作 policy 并行 encoder | 作者自述"梯度弱不稳定"；与 SF 优等做法对比劣 |
| | C2: 把 VGGT 作初始化预训练 | 算力门槛太高 |
| **D. SF 范式替代类** | D1: SF 原版均匀监督（不加 mask） | v2plus 的核心创新点就是 focus mask；不加 mask 等价 SF 复现 |
| | D2: Multi-layer SF（多层都对齐） | 工程量翻倍，SF 论文已证 Layer 24 单层最优 |
| **E. 评测/scope 替代类** | E1: 不做真机评测 | 师哥明确要求"按 InSpire 的会，要在真机上验证" |
| | E2: 5 任务 × 100 demo 大规模真机 | 超 2 人 ¥10K 预算 |
| | E3: 4 变体（v1/v2/v3/v4=单独+FSF） | 紧约束下 GPU-h 不够，用 ablation 代替 |

## § 2 · A 类 · 真改架构方案的排除

### 2.1 A1：真复现 FocusVLA Cascaded + Focus Attention

**方案**：放弃 FSF，转而真实复现 FocusVLA 的两个机制（Modality Cascaded Attention + Focus Attention）

**优势**：
- 与师哥推荐的"FocusVLA 导航星"完全对齐
- 复现自身就有研究价值
- 0.5B 参数打败 7B 的工程美感

**致命缺点**：
1. **代码未释**：FocusVLA 论文（arXiv:2603.28740）2026-03-30 发布，目前 GitHub 不可见。若不自己复现，根本无法跑
2. **架构改造工程量极大**：Cascaded Attention 需要重写 transformer attention block，不是 plug-and-play；patch-level top-K 的可微化需要 careful 处理
3. **训练侧风险高**：Focus Attention 的 channel-level gate 需要从头训练，难以与 LoRA 兼容
4. **VGGT 在 FocusVLA 中是"梯度弱不稳定"** —— 作者自述局限。我们想用 VGGT 的优等做法（SF）而不是劣等做法（FocusVLA）

**v2plus 的回应**：本项目继承 FocusVLA 的"视觉利用"诊断视角（视觉 ablation 在 P3），但**不复现其架构改造**；采纳 SF 的 VGGT 优等用法 + 增加 focus mask 创新。

### 2.2 A2：真改 VLA backbone 加 3D head

**方案**：在 OpenVLA-OFT 上加一个独立的 depth/point map prediction head，joint train action loss + 3D loss

**优势**：显式 3D output 可解释；可视化便利

**缺点**：
1. **需要 dense 3D GT**：LIBERO 仿真自带但真机需要 RGB-D 相机 + 标定
2. **增加 architecture 复杂度**：新 head 需要从头训练；与 LoRA 兼容差
3. **SF 论文已证明**：显式 depth 监督引入 depth estimator 误差，**不如隐式表征对齐**
4. **算力门槛高**：joint train action + depth 需要更多 GPU-h

**v2plus 的回应**：采纳 SF 论文实证的"隐式表征对齐 > 显式 depth output"结论。

## § 3 · B 类 · 显式 3D 输入方案的排除

### 3.1 B1：PointVLA / SpatialVLA 路线

**方案**：在输入端加入 3D 信息——点云、Ego3D PE 等

**SpatialVLA 具体方案**：每个 patch 不仅有 2D position emb，还附加 3D world-coord（从深度+相机外参反投影）。Adaptive Action Grids 把工作空间离散化为 grid cells。

**优势**：显式 3D prior；跨 embodiment 迁移友好

**缺点**：
1. **依赖深度传感器**：在真机部署时（如 SO-100 标配只有 RGB 相机），退化为 monocular depth estimation
2. **预训练需求**：SpatialVLA 在 1.1M episode 上预训练，本科生 1 年期完全做不到
3. **跨 LIBERO 和真机数据**：需要重新设计 dataset interface 处理 3D 输入
4. **与 SF 相比**：SpatialVLA 是 input 端注入 3D（feature-aware），SF 是 supervision 端注入 3D（label-aware）——后者工程更轻

**v2plus 的回应**：仅在 §2 Related 引用 SpatialVLA 作为对比工作；不复现。

### 3.2 B2：DepthVLA / 3D-CAVLA 路线

**方案**：在 VLA 输出端加 depth prediction head，与 action head 联合训练

**DepthVLA 具体方案**：显式 depth-aware spatial reasoning；depth head 与 action head 共享 backbone

**优势**：显式可视化 depth

**缺点**：
1. **depth GT**：LIBERO 仿真有但真机需 RGB-D
2. **增加输出端复杂度**：depth head + action head + 联合训练；与 LoRA 兼容差
3. **SF 已证明 depth 估计器引入误差**：DepthVLA 的 depth output 不可避免有估计误差，污染 action 学习

**v2plus 的回应**：在 §2 Related 引用 DepthVLA 作为"显式 vs 隐式"3D 监督的对比工作。

## § 4 · C 类 · VGGT 用法替代方案的排除

### 4.1 C1：仿 FocusVLA 把 VGGT 作 policy 并行 encoder

**方案**：放弃 SF 的"中间层对齐"，转而像 FocusVLA 那样把 VGGT 作为第 3 个 encoder（与 SigLIP/DINOv2 并行）

**优势**：和 FocusVLA 用法一致，与师哥推荐的"FocusVLA 导航星"对齐

**致命缺点**：
1. **FocusVLA 作者自述**："VGGT features can only be injected at the policy stage to avoid disrupting the pretrained VLM, and their gradients are relatively weaker and less stable, which limits their performance upper bound." —— 这是作者明确承认的局限
2. **性能上限低**：FocusVLA 用 VGGT 性能受限；SF 用 VGGT teacher 性能充分（+1.4pp, 3.8× 加速）
3. **代码复杂度高**：需要在 policy 阶段集成第 3 个 encoder

**v2plus 的回应**：采纳 SF 的"中间层对齐 + Teacher"模式；明确拒绝 FocusVLA 的"policy 并行"模式。这正是师哥提示"FocusVLA 也用了 VGGT，看做法是否有区别"的答案——**做法不同，SF 是优等**。

### 4.2 C2：把 VGGT 作初始化预训练

**方案**：用 VGGT 的 transformer 权重初始化 OpenVLA-OFT 的某些层

**优势**：3D prior 在初始化就注入

**缺点**：
1. **维度不匹配**：VGGT 是 1024-dim，Llama2-7B 是 4096-dim，需要复杂的权重迁移
2. **预训练数据未知**：VGGT 在 3D 数据上预训练，与 LIBERO action 数据分布差异大
3. **算力门槛**：完整 fine-tune 而非 LoRA，超 ¥10K
4. **VIPA-VLA 已尝试**：相关工作 VIPA-VLA（arXiv:2512.13080）做了类似事情，需要大规模 human video 预训练；本科生做不到

**v2plus 的回应**：拒绝。SF 的"冻结 teacher + 训练 student LoRA"模式工程友好得多。

## § 5 · D 类 · SF 范式替代方案的排除

### 5.1 D1：SF 原版均匀监督（不加 focus mask）

**方案**：完全复现 SF 论文，不引入 focus mask 创新

**优势**：直接复用 SF 代码；最低风险

**致命缺点**：
1. **失去 v2plus 的方法学创新**：v2plus = v2 + FSF，FSF = SF + focus mask。如果不加 mask，v2plus 就是 v2 + SF 复现，**没有原创贡献**
2. **师哥提示明确**："spatial forcing 提供 3D 监督，但需要 focus 在重要位置。一些不相关的背景提供的监督可以更弱点。" —— focus mask 是师哥明确要求的
3. **AttentionVoxel 等已证**：task-relevant 区域的 3D 监督最有效，均匀监督有 noise

**v2plus 的回应**：拒绝。focus mask 是 v2plus 区别于 SF 复现的核心创新点。如果在 A3 ablation 中 M-ABC 与 M-None 等价，则 H5 失败，改叙事 "focus 与 uniform 等价" —— 这仍是有意义的 finding，但**主路必须包含 focus mask**。

### 5.2 D2：Multi-layer SF（多层都对齐）

**方案**：不仅在 Layer 24 对齐，在 Layer 8/16/24/32 都做对齐

**优势**：监督密度更高

**致命缺点**：
1. **工程量翻倍**：4 个 projector + 4 个 VGGT 取层 + 4 个 loss + 联合调权重
2. **SF 论文已证明**：Layer 24 单层最优（Layer 1: 94.6%, Layer 8: 95.7%, Layer 16: 93.8%, Layer 24: 96.9%, Layer 32: 94.8%）；多层对齐可能让 student 过拟合 teacher
3. **GPU-h 翻倍**：cache 4 层 feature 而非 1 层，磁盘从 650GB → 2.6TB

**v2plus 的回应**：拒绝。继承 SF 的单层 Layer 24。但在 A2 ablation 中扫描 {8, 16, 24, 32}（共 4 组）验证单层最优。

## § 6 · E 类 · 评测/scope 替代方案的排除

### 6.1 E1：不做真机评测

**方案**：v2plus 仅做仿真（LIBERO-Plus 7 维），不做真机

**优势**：紧约束下最容易完成

**致命缺点**：
1. **师哥明确要求**："按 inspire 的会，特别是要在真机上验证的话，因为真机数据需要自己来标注了。**不过我们可以接受**" —— 师哥已明确接受真机标注的工作量
2. **失去 H4**：真机扰动假设是 v2plus 的核心新假设；不做真机 = v2plus 只有 3 个新假设（H2/H5）+ 2 个继承假设（H1/H3）
3. **投稿适配度低**：CoRL workshop 等机器人会议偏好有真机的工作；仿真-only paper 的接受率低
4. **实验室已有真机**：硬件成本 = 0，主要是 1 周采集 + 1 周评测的人工

**v2plus 的回应**：拒绝。真机是 v2plus 的核心增量轴。

### 6.2 E2：5 任务 × 100 demo 大规模真机

**方案**：真机 scope 升级到 5 任务 × 100 demo × 3 扰动维度

**优势**：覆盖更广，统计 power 更强

**致命缺点**：
1. **超人工预算**：5 × 100 demo × 5 分钟/demo = 41 小时纯采集 + 30 小时 reset/调试 = 9 天人工
2. **超 GPU-h 预算**：5 任务 × LoRA fine-tune × 3 变体 = 60 GPU-h
3. **超时间预算**：P2-P3 已紧；扩 scope 会挤压 ablation 时间

**v2plus 的回应**：拒绝。中等 scope（3 任务 × 50 demo × 3 扰动维度 = 360 episode）是 InSpire 论文标准协议，1 周可完成。

### 6.3 E3：4 变体（V1/V2/V3/V4=单独+FSF）

**方案**：增加 V4 = OpenVLA-OFT + FSF only（不含 InSpire 方向词）作为单独 +FSF 变体

**优势**：直接验证 H2（InSpire 与 FSF 的叠加效应）

**致命缺点**：
1. **GPU-h 增 25%**：V4 单独训练 × 3 seed × LIBERO 4 suite ≈ 30 GPU-h；评测 × 3 seed × 7 维 ≈ 50 GPU-h
2. **超 ¥10K 预算**：原 230 GPU-h → 310 GPU-h，按 ¥26/h 算多花 ¥2080
3. **可用 ablation 代替**：A3 中 M-None（=SF 原版均匀监督，等价 +FSF only 的简化版）作为 +FSF only 代理

**v2plus 的回应**：拒绝 4 变体；用 3 变体 + A3 ablation 中 M-None 作 V4 代理（**用户已确认**）。这是紧约束下的合理折衷。

## § 7 · 排除方案的对比表汇总

| 维度 | v2plus 主路 | 排除方案 A1（真改 FocusVLA） | 排除方案 B1（SpatialVLA） | 排除方案 D1（SF 原版） |
|---|---|---|---|---|
| 工程量 | 中（v2 +30%） | 高（4 周架构） | 极高（预训练 + 跨 embodiment） | 低（仅复现） |
| 算力需求 | 230 GPU-h | 200+ GPU-h | 大量 | 150 GPU-h |
| 原创贡献 | 多源 focus + 真机 | 复现 | 无 | 无 |
| 与师哥提示对齐 | 全部对齐 | 部分对齐 | 不对齐 | 不对齐 focus mask |
| 真机验证 | 是 | 否（无代码） | 难 | 可以 |
| 与 SF 论文区别 | focus mask + 真机 + InSpire 叠加 | — | 不同 paradigm | 无差异 |

## § 8 · 排除决策的最终原则

v2plus 的所有方案选择遵循以下 5 个原则：

1. **必须直接响应师哥 4 条提示**（加 3D / Spatial Forcing / focus mask / VGGT 续作借鉴）
2. **必须在 2 人 + ¥10K 紧约束下可行**（实验室真机硬件免费）
3. **必须有原创方法学贡献**（focus mask 多源融合是 v2plus 的核心创新）
4. **必须有清晰 fallback 路径**（最坏退到 v2 only，切换成本 < 1 周）
5. **必须能在 1 年期内完成**（P0-P5 共 12 月，每 phase 都有 Go/No-Go）

所有 5 原则同时满足的方案就是 v2plus 主路：**FastVGGT teacher + FSF on Layer 24 + 多源 focus mask + V1/V2/V3 + LIBERO-Plus 7 维 + 真机 9 cell**。
