# 09 · 与版本二差异化说明

> **本文档专为与师哥/导师讨论时单独使用**——浓缩版本二 vs 版本二plus 的所有关键差异，1-2 页内说清楚。便于在 30 分钟会议中协助快速判断。

## 一句话差异化

| | 版本二 v2 | **版本二plus** |
|---|---|---|
| **核心姿态** | 显式方向词提示插件（output 层 grounding） | **显式方向词 + 隐式 3D 表征对齐（output + encoder 双层 grounding）** |
| **核心方法** | InSpire 方向词 plugin + LoRA 微调 + 视觉利用 ablation | **+ FastVGGT teacher 对 Layer 24 做余弦相似度对齐 + 双源融合 focus mask 调制** |
| **研究问题** | "显式提示能否提升 VLA 鲁棒性" | **"显式提示与隐式 3D 对齐叠加能否在仿真+真机两轴系统提升 VLA 鲁棒性？双源融合 focus 是否优于均匀监督？"** |
| **与 FocusVLA 故事的紧密度** | 间接（InSpire 是 alternative path） | **更直接（替代 FocusVLA 用 VGGT 的劣等做法，采纳 Spatial Forcing 的优等做法）** |
| **评测轴** | 仅仿真（LIBERO-Plus 7 维） | **仿真 + 真机（3 任务 × 3 扰动 = 360 episode）** |

## 6 维详细对比

### 1. 工程量

| | 版本二 v2 | 版本二plus |
|---|---|---|
| 训练 VLA | ✅（LoRA + cls head + loss 联合） | ✅（LoRA + cls head + **FSF projector**） |
| 写方向词 GT 标注脚本 | ✅（5 天） | ✅（复用 v2） |
| 写 cls head + loss 联合 | ✅（3 天） | ✅（复用 v2） |
| LoRA 微调调参 | ✅（7 天） | ✅（复用 v2，3 变体） |
| attention hook | ✅（2 天，可选） | ✅（复用 v2） |
| **FastVGGT 装通 + sanity** | ❌ | ✅（**新增 3 天**） |
| **FastVGGT 离线缓存 pipeline** | ❌ | ✅（**新增 5 天，11 GPU-h 一次性**） |
| **FSF projector 代码** | ❌ | ✅（**新增 2 天，~30 行**） |
| **Focus mask 双源融合管线** | ❌ | ✅（**新增 4 天**，Source A 复用 v2、Source C 复用 DINOv2） |
| **真机平台 + 数据采集** | ❌ | ✅（**新增 14 天**，实验室已有平台） |
| **真机评测** | ❌ | ✅（**新增 10 天，360 episode**） |
| 评测 + 统计 + 论文 | 25 天 | 30 天（+3 变体 + 真机数据） |
| **总工程时间** | **~54 天** | **~95 天**（+76% 工程量） |

**关键差异**：v2plus 工程量增长 76%，但 ~41 天的增量集中在 **(a) 新增组件（FSF projector / VGGT 缓存 / Focus mask）** 与 **(b) 真机数据采集与评测**。两者都是师哥明确要求的"加 3d"与"按 InSpire 的会，要在真机上验证"。

### 2. Novelty 来源

**版本二 v2**：
- "InSpire 在 LIBERO-Plus 多维扰动下的系统验证"——InSpire 原论文只在 clean 或自家 OOD 评测
- "视觉利用 ablation（token dropout / patch mask / attention 可视化）"作为机制证据

**版本二plus**：
- **"显式方向词 + 隐式 3D 表征对齐的双层 grounding 互补效应"**——文献中**完全空白**：
  - InSpire（显式 output）+ Spatial Forcing（隐式 encoder）从未在同一项目中叠加验证
  - FocusVLA 也用了 VGGT 但作法不同（policy 阶段并行 encoder，作者自述"梯度弱不稳定"）
- **"双源融合 focus mask（方向词GT × DINOv2 CLS attention，v2 纯净路径不引入 SAM2）"**——首次提出，对应师哥"focus 在重要位置，背景弱"
- **"focus mask 来源消融（H5）"**——验证双源 > 单源 > 均匀
- **"真机扰动评测协议"**——按 InSpire 真机协议（3 任务 × 50 demo × 3 扰动 = 360 episode）
- 继承 v2 的"InSpire 在 LIBERO-Plus 系统验证"与"视觉利用 ablation"两项 novelty

**v2** novelty 集中在 **方法验证**（InSpire 真的鲁棒吗）；**v2plus** novelty 集中在 **方法叠加 + 双源 focus + 真机** 三轴。后者贡献更多面。

### 3. 与 FocusVLA / Spatial Forcing 故事的对应

师哥推荐：「**沿 FocusVLA 方向深挖鲁棒性**，加 3D；**参考 arXiv:2510.12276 Spatial Forcing 的隐式 3D 对齐**；**FocusVLA 中也用了 VGGT，看是否有区别**」

**版本二 v2**：
- 不涉及 3D；InSpire 是显式 alternative
- 对 FocusVLA 4 大局限只回应 #1 (背景纹理) 和 #2 (初始状态)，不直接回应 #3 (VLM 视觉利用) 与 #4 (robustness benchmark 评测)

**版本二plus**：
- **直接采纳师哥三条建议**：
  - "加 3D" → FSF 通道（隐式 3D 表征对齐）
  - "参考 Spatial Forcing" → loss 公式直接复用 SF + 加 mask 调制
  - "FocusVLA 中 VGGT 做法是否有区别" → §1.3 在 [[08_最终方案_FSF详细技术报告]] 中详述对比表
- **回应 FocusVLA 4 大局限**：
  - **#1（背景纹理）**：FSF 在 viewpoint/state 维度对齐 3D 表征 + focus mask 弱化背景 → 减少背景纹理变化的影响
  - **#2（初始状态）**：3D 表征对齐让 VLA 对几何位置变化更鲁棒
  - **#3（VLM 视觉利用）**：FSF 直接在 VLM Layer 24 注入 3D 监督，显式改善视觉利用 + 继承 v2 视觉 ablation 提供证据
  - **#4（robustness benchmark）**：LIBERO-Plus 7 维 + 真机 3 维 = **两轴系统评测**

**v2plus 与师哥指引的对齐度比 v2 高一档**。

### 4. 主要风险

| | 版本二 v2 | 版本二plus |
|---|---|---|
| **风险 1** | InSpire 代码未释（60%）→ 自己手写 200 行 | **VGGT 离线缓存超磁盘**（30%）→ fp8 量化 / 学校大存储 / streaming fallback |
| **风险 2** | 方向词 GT 推导噪声 > 30%（40%）→ 滑动平均 + corner case | **FSF Loss 在 OpenVLA-OFT 上退化 SR**（30%）→ β warmup 0→0.5 / fallback 切 V2 only |
| **风险 3** | cls head 不学（30%）→ 调 loss weight | **Focus mask 双源无差异（H5 失败）**（35%）→ 改叙事 "focus > uniform"；P3 stretch 引入 SAM2 |
| **风险 4** | LoRA 工程坑（30%） | **真机 50 demo 不够 fine-tune**（45%）→ zero-shot / case study fallback |
| **风险 5** | FocusVLA 代码未释（60%）→ OpenVLA-OFT 代理 | **FastVGGT 在 LIBERO 合成图上质量差**（25%）→ VGGT 原版 / VGGT-DP fallback |
| **风险 6** | — | **2 大二本科生工程能力**（30%）→ AI 辅助 + 导师 office hour（v2 已计入） |
| **风险 7** | — | **LIBERO-Plus 评测超 GPU-h 预算**（50%）→ 降 instance 500→300 |

**关键**：v2plus 风险 ~7 个 vs v2 ~5 个。新增 4 个风险都有明确缓解策略与 fallback（详见 [[05_风险与缓解]]）。所有 v2plus 风险都可通过 fallback 退回 v2 路径（**切换成本 < 1 周**）。

### 5. 投稿适配度

| | 版本二 v2 | 版本二plus |
|---|---|---|
| **第一目标** | CoRL/ICLR/NeurIPS workshop short | 同 + delight: CoRL main 短稿 |
| **方法学贡献量** | 1 个（InSpire 在多扰动下验证） | 3 个（InSpire+FSF 叠加、双源 focus、真机协议） |
| **数据集贡献** | 方向词标注 + failure 配对（2 个） | + VGGT 缓存（公开 326 GB）+ Focus mask 数据（共 4 个） |
| **代码贡献** | InSpire plugin + ablation 工具 | + FSF projector + Focus mask 管线 + 真机评测脚本 |
| **预期评审接受度（workshop）** | 中高（方法简单，叙事单一） | **高**（方法多层、叙事双轨、真机加分） |
| **预期评审接受度（main）** | 低（贡献偏单薄） | **中**（特别是 H4 真机成立时） |

### 6. 与师哥指引的对齐度

| 师哥语言 | v2 对应 | v2plus 对应 |
|---|---|---|
| "加 3D 是好想法" | ❌ 不加 3D | ✅ FSF 通道 |
| "按 InSpire 的会，特别是要在真机上验证的话" | ❌ 不做真机 | ✅ 3 任务 × 50 demo × 3 扰动 = 360 episode |
| "真机数据需要自己来标注了，不过我们可以接受" | ❌ | ✅ 实验室已有真机硬件 + 1 周采集 + 标注 |
| "参考 arXiv:2510.12276，隐式 3D 表征对齐" | ❌ | ✅ FSF loss 直接采用 SF 论文公式 |
| "FocusVLA 中也用了 VGGT，看做法是否有区别" | ❌ | ✅ §1.3 详述对比表（FocusVLA = 并行 encoder 劣等；SF = 中间层对齐优等） |
| "Spatial forcing 提供 3D 监督，但需要 focus 在重要位置；不相关背景监督可以更弱点" | ❌ | ✅ 双源融合 focus mask（前景 1.0 / 上下文 0.5 / 背景 0.1） |
| "VGGT 续作可以大致看下" | ❌ | ✅ §2.5 列入 FastVGGT（主选）、π³、StreamVGGT、HD-VGGT 等续作 |

**结论**：v2plus 全面响应师哥 7 条提示；v2 仅响应 0 条（因 v2 在师哥讨论之前定型）。

## 决策推荐：v2plus 作为最终方案投递

基于以上 6 维对比，我们建议 **v2plus 替换 v2 作为最终立项方案投递**（用户已确认 Q1）：

**支持理由**：
1. v2plus 全面响应师哥 7 条提示；v2 全部未响应（时序原因）
2. v2plus 增加 76% 工程量，但在 2 人团队 + ¥10K 预算约束下仍可行（实验室已有真机硬件）
3. v2plus 的 5 个 H 假设比 v2 的 3 个更丰富，论文 main 章节贡献量更高
4. v2plus 所有新增组件都有明确 fallback 退回 v2 路径，**切换成本 < 1 周**

**风险提示**：
1. v2plus 工程门槛比 v2 高（FastVGGT + 双源 mask + 真机），对 2 大二本科生有一定挑战
2. 真机数据采集 + 评测约占 1 个月人工，需要 P2-P3 阶段密集投入
3. FSF Loss 收敛性是 v2plus 的最大单点风险，**P1（M3）末 N1 节点必须达标，否则切 v2 only**
