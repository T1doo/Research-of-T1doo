# FSF-VLA 项目详情文档

## §1 项目摘要与立项依据

### 1.1 项目代号与一句话定义

项目代号 FSF-VLA（Focused Spatial Forcing for Vision-Language-Action Models）。在 π₀-FAST [1,23] 主 backbone 之上沿用 Spatial Forcing [2] 提出的 LLM 中间层（~Layer 13-14）VGGT 蒸馏范式，新增一个由方向词 ground truth 与 DINOv2 CLS attention 双源融合的 focus mask 调制 align loss，并叠加 InSpire [3] 风格的显式方向词提示。Teacher 锁 FastVGGT [4] 单一选项做离线缓存；评测同时覆盖 LIBERO-Plus [5] 7 维仿真扰动与小规模真机扰动两条轴。

### 1.2 问题与立项依据

视觉-语言-动作（VLA）模型在 LIBERO 等标准基准上已达 95% 以上的成功率，但 LIBERO-Plus [5] 在 7 个扰动维度（物体布局、相机视角、机器人初始状态、语言指令、光照、背景纹理、传感器噪声）上的系统评测显示，主流 VLA 模型成功率会从 95% 跌至 30% 以下。这一断崖式下跌是 VLA 走出实验室的关键障碍。

文献中针对扰动鲁棒性主要存在两条互补的路径。一条是 InSpire [3] 风格的"显式空间提示"——在原任务前置一个方向性问题，让模型先输出方向词再生成动作，从输出层引入与空间关系直接耦合的中间表示。另一条是 Spatial Forcing [2] 风格的"隐式 3D 表征对齐"——以冻结的 VGGT [6] 为 teacher，对 VLA 视觉编码器中间层做 per-token 余弦相似度对齐，让 VLA 在不引入额外 3D 输入的前提下隐式获得空间感知。这两条路径各自在原论文上得到验证，但它们的协同效应、以及在统一的多扰动评测与真机扰动两轴下的系统验证，至今仍是空白点。

本项目的科研价值由三个空白点界定：第一，InSpire 显式方向词提示与 Spatial Forcing 隐式 3D 对齐的叠加效应缺乏直接实证；第二，Spatial Forcing 对所有 visual token 采用均匀对齐权重，未充分利用任务相关性先验，而双源融合 focus mask（方向词 ground truth 倒推 2D Gaussian + DINOv2 CLS attention）调制 align loss 据我们调研范围未见；第三，仿真高分与真机扰动表现之间的差距在公开数据集上缺乏小规模本科生可执行的对照评测样本。

### 1.3 立项约束

复旦曦源项目 1 年期、2 人本科生、¥10000 预算。指导教师所在实验室具备 6 自由度以上机械臂、第三视角相机与工作台标定板，可支撑本项目真机数据采集与扰动评测。云端 GPU 算力按紧约束版规划约 230 GPU 小时，目标产出 4-6 页 workshop short paper 投稿至 CoRL 2027 / ICLR 2027 / NeurIPS 2027 Robot Learning workshop，并公开发布代码、模型权重与真机数据集。

## §2 文献综述与定位

### 2.1 VLA 主 backbone 与 π₀-FAST 选择

π₀-FAST [1] 以 PaliGemma [24] 为视觉-语言主干，叠加 FAST [23] 动作 tokenizer（在频域将连续动作 chunk 离散为进入语言模型词表的离散 token），是一个自回归 VLA，无独立 action expert，总参数约 3B（即 PaliGemma-3B 本身；区别于另含约 300M action expert、合计约 3.3B 的 flow-matching π₀）。PaliGemma 由 SigLIP-So400m 视觉编码器与 Gemma-2B 语言解码器组成；Gemma-2B 为 18 层 transformer、hidden 维度 2048。本项目选择 π₀-FAST 而非 7B 级主干（如 OpenVLA-OFT [25]）有两点理由：其一，参数规模降至约 3B，单卡微调门槛与迭代成本显著降低；其二，π₀-FAST 的自回归特性使 InSpire "先输出方向词、再生成动作"的链路天然成立——InSpire 即在 π₀-FAST 上验证其方法、并以 π₀-FAST 作为其真机评测的基线模型 [3]，故本项目五个假设 H1–H5（尤其依赖推理期方向词输出的 H3）可在 π₀-FAST 上原样保留。本项目以 π₀-FAST 为统一主 backbone，所有变体共享相同的视觉编码器（SigLIP-So400m）、相同的 Gemma-2B 语言模型、相同的 LoRA 配置（r=16，target_modules 为 q_proj/k_proj/v_proj/o_proj），仅在显式提示插件、focus mask 调制 align loss 与训练数据三个维度上做对照。需说明的是，π₀-FAST 的自回归解码在推理速度上明显慢于 flow-matching π₀ 与 OpenVLA-OFT 的并行解码（见 §4.1 评测算力与 §6 R6），但该特性属于 backbone 本身、对所有变体一致，且训练期 FSF 通道对推理零侵入这一结论不受影响。

### 2.2 隐式 3D 表征对齐：Spatial Forcing 范式

Spatial Forcing [2] 由 OpenHelix Team（Li 等，2025-10 投稿 ICLR 2026）提出。其核心做法是：以冻结的 VGGT [6] 为 teacher，对 VLA 主干 LLM 的中间层抽取 visual token hidden state（SF 原论文在其 32 层 Llama2-7B（OpenVLA）主干上取 Layer 24，约 75% 深度），过 BatchNorm + 2 层 MLP + 2D sinusoidal positional encoding 投影到 1024 维，然后与 VGGT aggregator Layer 23（倒数第二层）的 patch token 做 per-token 余弦相似度对齐：

$$\mathcal{L}_{\text{align}} = -\frac{1}{N}\sum_{i=1}^{N} \cos\Big(\mathrm{MLP}\!\cdot\!\mathrm{BN}(x_i^{V,L24}) + E_i,\; f_i^{3D,L23}\Big)$$

最终训练目标为 $\mathcal{L}_{\text{SF}} = \mathcal{L}_{\text{action}} + 0.5 \cdot \mathcal{L}_{\text{align}}$。Spatial Forcing 论文在 LIBERO 上取得 98.5% 平均成功率（vs OpenVLA-OFT [25] baseline 97.1%，长程任务 LIBERO-Long 96.0% vs 94.5% = +1.5pp，依据 arXiv:2510.12276 Table 1），训练加速 3.8 倍（论文 §3.3 "achieving the same success rates 3.8× more quickly"），训练数据降至 5%（约 5.9× 数据效率）即在 LIBERO-Long 上达到 66% 成功率（依据 spatial-forcing.github.io 项目页与论文 §3.3 Table 2），并在 RoboTwin 2.0 仿真 benchmark 上以 π₀ 为 base model 取得显著相对提升（依据 arXiv:2510.12276 §3 "Simulation Experiments" 与 Fig. 4 定性比较，论文中未提供具体 pp 数字）。该方法仅约 30 行核心代码改动，对原 VLA 推理路径零侵入。

Layer 24（在 32 层 Llama2-7B 中位于约 75% 深度）的选择是经过 Spatial Forcing 原论文 §4.4 完整层位扫描确定的最优值；本项目将这一"约 75% 深度"的原则等比例映射到 π₀-FAST 的 Gemma-2B（18 层）主干，对应 ~Layer 13-14，并由消融 A2 在 {Layer 6, 10, 13, 16} 中定档最优层位。过浅的层处于 language-vision token mixing 阶段尚未形成可对齐的空间语义，过深的层已被压缩到 action decoding，强行注入 3D 监督会破坏 task understanding。Teacher 取 VGGT aggregator 倒数第二层（Layer 23）而非最后一层，是因为最后一层 feature 已被各个 task-specific head 拉向特定输出，而倒数第二层保留了最丰富的通用 3D 语义。

### 2.3 显式方向词提示：InSpire 范式

InSpire [3] 由 Zhang 等（UESTC，2025-05，arXiv:2505.13888）提出。其做法是在原 VLA 任务的语言 instruction 前置一个方向性问题（"In which direction is the [object] relative to the robot?"），要求模型先输出方向词（right/left/up/down/front/back/grasped 共 7 类），再生成 action。方向词作为输出层的离散中间监督，强制模型显式建立"指令-空间关系"的耦合。InSpire 在原 LIBERO 上提升有限，但在视觉扰动维度上表现出鲁棒性增益。

### 2.4 VGGT 与 FastVGGT teacher 选择

VGGT [6] 是 CVPR 2025 论文，由 Meta FAIR 与 Oxford VGG 联合发布，作者 Wang/Chen/Karaev/Vedaldi/Rupprecht/Novotny。其核心创新是在单一 transformer + 单次 forward 中推理多种 3D 量（相机参数、深度图、点图、轨迹），参数量 1.2B。VGGT 在 16 个 3D 数据集（Co3Dv2, BlendMVS, MegaDepth, Kubric, ScanNet, HyperSim 等）上预训练，feature 同时编码深度、相机姿态、表面法向与点云对应关系。

FastVGGT [4]（You Shen 等，arXiv:2509.02560）通过 training-free 的 token merging 把 VGGT 推理加速 4 倍，深度 δ<1.25 精度损失约 0.4pp，训练显存降 29%。该实现与 VGGT 原版 100% 接口兼容，加载 VGGT 原 checkpoint 后运行时切换 merge ratio 即可。本项目锁 FastVGGT 作为唯一 teacher 选项，不再保留 StreamVGGT / HD-VGGT / π³ 等续作作为备选，以工程稳定性与本科生可执行性优先。

### 2.5 与 FocusVLA 的差异化定位

FocusVLA [7]（Zhang Yichi 等，arXiv:2603.28740）是同时期与本项目主题相近的工作，提出 Modality Cascaded Attention 与 Focus Attention 机制以缓解 VLA 视觉利用问题。FocusVLA 也使用了 VGGT，但其用法与 Spatial Forcing 存在结构性差异——FocusVLA 把 VGGT 视为与 SigLIP / DINOv2 并行的第三个视觉 encoder，在 policy 阶段（VLM 完成所有 self-attention 之后）由 Focus Attention 的 top-K + channel gate 间接利用其 token；VGGT 与 VLM 主干之间没有任何梯度交互。

本项目相对 FocusVLA 的差异化论证从结构性差异出发，不做跨主干 pp 横比——FocusVLA 基于 VLA-Adapter + Qwen2.5-0.5B（总可训参数约 343M，依据 arXiv:2603.28740 §3 主干配置），与 π₀-FAST（PaliGemma/Gemma-2B，约 3B）主干与参数量级不同，单看 LIBERO Avg pp 差距不具方法学可比性。一个可直接引证的结构性事实是：FocusVLA 最佳成绩 98.7%（multi-weights setting，依据 arXiv:2603.28740 Table 1）采用的是 VLM+DS+VLM 配置（DINOv2+SigLIP+VLM），主动舍弃 VGGT；Spatial Forcing 在同表的成绩为 98.5%。FocusVLA 论文 §4.3 Ablation Study "Visual Representation" 子节中作者明确写道："However, due to architectural constraints, to avoid disrupting the pretrained VLM, VGGT features can only be injected at the policy stage. As a result, their gradients are relatively weaker and less stable, which limits their performance upper bound."（论文原文，依据 arXiv:2603.28740 §4.3）。这是本项目选择继承 Spatial Forcing 中间层 teacher 对齐范式、不采纳 FocusVLA policy 阶段并联范式的方法学依据。

关键差异有两点。第一，VGGT 接入位置不同：本项目沿用 Spatial Forcing 在 LLM 中间层（~Layer 13-14）做中间层蒸馏（梯度直接反传至对齐层及其下游、即 Layer 1 至对齐层），而 FocusVLA 在 policy 阶段并联（VGGT 与 VLM 主干无梯度交互）。第二，"focus" 概念作用位置不同：FocusVLA 的 focus 作用于 attention 计算（top-K 视觉 token 筛选 + channel-wise gate），目的是减少推理时的 token 数；本项目的 focus 作用于 align loss 的 per-token 监督权重（双源融合 mask 调制三段量化），目的是让 3D 监督信号更专注于任务相关区域、不被背景稀释。

### 2.6 Focus mask 与 attention guidance 相关工作

近期已有若干工作探索用 saliency mask 引导 VLA 训练。AttentionVoxel [8]（Yurchyk 等，Humanoids 2025）把 DINOv2 CLS attention 提升到 3D voxel grid 作为操作 saliency 先验，与本项目的 Source C 同源但作用空间不同（3D voxel vs 2D patch loss 加权）。同期已发表与未发表工作的探索范围覆盖 VLM-based saliency 替代人类 gaze、人类眼动监督 attention 正则、以及历史视觉轨迹作为输入端 prompt 等多种思路。本项目相对这些工作的独特定位有三点：将 saliency mask 用于调制 VGGT-VLA 蒸馏对齐 loss 而非 attention 计算；方向词 ground truth 倒推 2D Gaussian 与 DINOv2 CLS attention 双源融合；三段量化（前景 1.0 / 上下文 0.5 / 背景 0.1）的显式权重设计。

### 2.7 鲁棒性评测与失败检测

LIBERO-Plus [5]（Fei 等，arXiv:2510.13626）在原 LIBERO [9] 4 个 task suite 与 130 个任务基础上，引入 7 个扰动维度的系统评测集，是当前最完整的 VLA 仿真鲁棒性 benchmark。SAFE [10]（Qiao Gu 等，NeurIPS 2025）提出基于 VLA 内部 feature 的多任务 failure detection 框架，是本项目 H3 假设（运行时失败检测）的直接 baseline。Averaging Trap [11]（Tang 等，arXiv:2603.18342）指出 entropy 平均聚合会稀释短促但安全关键的不确定性 spike，本项目用 InSpire 离散方向词预测的时序切换作为失败信号正是该问题的一种解药。

### 2.8 3D-aware VLA 工作分类

3D-aware VLA 工作大致分为五类：直接 RGB-D 输入路线（如 SpatialVLA [12]、3D-Mix [13]、AugVLA-3D [14]、DepthVLA [15]），显式点云路线（如 PointVLA），单目深度估计辅助路线（如 DepthVLA），多视图重建路线（基于 DUSt3R / MASt3R），以及隐式 3D 对齐路线（Spatial Forcing [2]、本项目 FSF-VLA）。本项目属于隐式对齐路线，与显式 3D 输入路线的差异是推理路径零侵入与零额外 latency。

## §3 方法学：FSF-VLA 详解

### 3.1 整体架构

FSF-VLA 在 π₀-FAST 主 backbone 之上沿用 Spatial Forcing 的训练时 VGGT teacher 蒸馏架构，新增三个组件：FastVGGT 离线缓存 pipeline、focus mask 双源生成 pipeline、focus mask 调制 align loss。推理时所有新增组件被丢弃，仅运行 student VLA，因此推理 latency 与原始 π₀-FAST 完全相同。需说明的是，Spatial Forcing 官方仅提供 OpenVLA 与 flow-matching π₀ 两种实现；本项目在 π₀-FAST 上的 FSF 由官方 π₀ 实现适配而来——对齐目标作用于 Gemma 中间层 hidden state、位于动作头上游，与下游是 FAST 自回归头还是 flow-matching 头无关，因此该适配在方法学上是干净的。

训练流程：输入 RGB（224×224）经 SigLIP-So400m patch embedding 后进入 Gemma-2B Layer 1-18 前向；在 ~Layer 13-14 抽取 visual token hidden state（256 个 patch token × 2048 维），过 BN + 2 层 MLP（2048 → 1024 → 1024）+ 2D sinusoidal PE 得到 student 表征。同帧 RGB 通过冻结的 FastVGGT 离线缓存得到 aggregator Layer 23 输出（256 × 1024 维）作为 teacher 表征。Focus mask 由方向词 ground truth 反向投影得到的 2D Gaussian（Source A）与 DINOv2 CLS attention（Source C）双源融合后三段量化得到；其中 DINOv2 为本项目新增的冻结离线编码器（详见 §3.3）。动作以 FAST token 形式由语言模型自回归解码，对应 7 自由度 chunked action（chunk size H=8）。

总损失：

$$\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{action}} + \alpha \cdot \mathcal{L}_{\text{direction}} + \beta \cdot \mathcal{L}_{\text{FSF}}$$

其中 $\alpha = 0.1$ 沿用 InSpire 风格平衡，$\beta = 0.5$ 沿用 Spatial Forcing 原论文最优 α=0.5 的位置。$\mathcal{L}_{\text{action}}$ 是 π₀-FAST 的 FAST 动作 token 自回归交叉熵 loss（FAST 将动作 chunk 离散为语言模型词表 token，动作与方向词共享同一自回归解码目标），$\mathcal{L}_{\text{direction}}$ 是 7-class 方向词交叉熵 loss。

### 3.2 FSF align loss 的精确形式

FSF align loss 在 Spatial Forcing 原版均匀监督基础上引入 per-token 权重 $m_i \in \{0.1, 0.5, 1.0\}$（式中 $x_i^{V,L13}$ 表示在 Gemma-2B 对齐层 ~Layer 13-14 抽取的第 $i$ 个 visual token 隐状态，$f_i^{3D,L23}$ 为 VGGT aggregator 倒数第二层对应 token）：

$$\mathcal{L}_{\text{FSF}} = -\frac{1}{\sum_{i=1}^{N} m_i}\sum_{i=1}^{N} m_i \cdot \cos\Big(\mathrm{MLP}\!\cdot\!\mathrm{BN}(x_i^{V,L13}) + E_i,\; f_i^{3D,L23}\Big)$$

分母 $\sum m_i$ 归一化保证不同 mask 分布下 loss 整体尺度一致。这是 Spatial Forcing 原版 $\frac{1}{N}$（均匀）的自然推广——当所有 $m_i = 1$ 时退化为 Spatial Forcing 原版。

### 3.3 Focus mask 双源融合

Focus mask 由两个互补源融合生成。

Source A 是方向词 ground truth 倒推 2D Gaussian。从 LIBERO demo 或真机数据的 robot end-effector 3D 位姿与目标物体 3D 位姿出发，通过相机内外参投影到 2D 图像坐标 $(u_{\text{eef}}, v_{\text{eef}})$ 与 $(u_{\text{obj}}, v_{\text{obj}})$，在两处分别构造 2D Gaussian（$\sigma_{\text{eef}}=30$ 像素，$\sigma_{\text{obj}}=40$ 像素），取像素级 max 后下采样到 14×14 patch grid，得到 $m^A \in [0, 1]^{256}$。Source A 提供任务相关性先验，定位精确但仅覆盖两个中心点，对非凸物体形状不敏感。

Source C 是 DINOv2[22] CLS attention。π₀-FAST 的视觉编码器仅含 SigLIP-So400m、不含 DINOv2，因此本项目将 DINOv2 作为一个新增的冻结离线编码器引入：与 FastVGGT 教师特征同批，对全部训练帧一次性离线提取 DINOv2 最后一层的 CLS-to-patch attention（多头平均）并缓存，训练与推理期均不在线运行 DINOv2（其预提取算力与存储计入 §3.5 缓存预算）。取出的 attention 经 min-max 归一化到 [0, 1] 即得 $m^C \in [0, 1]^{256}$。Source C 提供由独立自监督视觉模型（DINOv2）给出的视觉显著性先验，与 Source A 形状互补但不一定与任务对齐（可能突出装饰物或干扰物）。

融合采用 F1 加权平均策略：

$$m_i = \text{quantize}\big(\text{clip}(0.5 m_i^A + 0.5 m_i^C,\ 0.1,\ 1.0)\big)$$

其中量化阈值为：$m_{\text{raw}} \geq 0.7 \to 1.0$（前景），$0.3 \leq m_{\text{raw}} < 0.7 \to 0.5$（上下文），$m_{\text{raw}} < 0.3 \to 0.1$（背景）。三段量化的直接语义对应是"前景应被强监督、上下文应被中等监督、不相关背景不应完全忽略但应被弱监督"——bg=0.1 而非 bg=0 是因为硬 mask（bg=0）会让背景区域完全丢失梯度信号，反而损害 VLA 主干对背景的几何理解；软 mask（bg=0.1）保留 10% 信号，对 Source A/C 误判的鲁棒性更好。

### 3.4 三个变体与 ablation 设计

主对照实验设 3 个变体：V1 vanilla（π₀-FAST + LoRA r=16，原 LIBERO instruction，仅 $\mathcal{L}_{\text{action}}$）、V2 +InSpire（V1 + 前置方向词 prompt + 7-class direction head + $\alpha \cdot \mathcal{L}_{\text{direction}}$）、V3 +InSpire+FSF（V2 + FastVGGT teacher + 双源 focus mask + $\beta \cdot \mathcal{L}_{\text{FSF}}$）。V3 是本项目主创新点，V2 是 v2 版本的标杆，V1 是 π₀-FAST baseline 校准。

Ablation 设计共 6 组：

| 编号 | 内容 | 目的 | GPU 小时 |
|---|---|---|---|
| A1 | VGGT teacher 取层 {Layer 12, 18, 23} | 验证 Layer 23 最优 | 15 |
| A2 | Student 监督层位 {Layer 6, 10, 13, 16} | 验证 ~Layer 13-14 最优 | 20 |
| A3 | Mask 来源 {M-None, M-A, M-C, M-AC, M-Random} | H5 核心：双源 vs 单源 vs 均匀；M-Random 作为"focus 位置正确而非 mask 多样性"的兜底证据（M-AC > M-A/M-C 比较失败时仍可保留"focus 位置正确性"结论）| 35 |
| A4 | β 粗扫 {0.1, 0.3, 0.5, 0.8} | 验证 β=0.5 最优 | 10 |
| A5 | 背景权重粗扫 {0, 0.05, 0.1, 0.3} | 验证 bg=0.1 最优 | 10 |
| A6 | 视觉利用机制分析（token dropout + attention 可视化）| 机制层证据 | 25 |

合计 115 GPU 小时。设计原则是从 v2plus 设计稿原 8 组中砍掉融合策略扫描（F2/F3/F4）与方向词粒度扫描（6 类/12 类），以及 teacher 备选项扫描（StreamVGGT 等），匹配紧约束 GPU 预算。

### 3.5 FastVGGT 离线缓存

LIBERO 4 个 task suite 共约 26000 条 demo，平均每条 50 帧，合计约 1.3M frame。若每个 epoch 重新跑 FastVGGT 前向，仅 teacher 计算需消耗约 1100 GPU 小时（按 100 epoch 计），完全超出预算。采用一次性离线预提取：一次性消耗约 11 GPU 小时跑完所有 frame 的 FastVGGT aggregator Layer 23 feature，缓存到本地 SSD。

缓存格式采用 WebDataset（tar shards），256×1024 维 bf16 feature 单帧 0.5 MB，1.3M frame 合计约 650 GB。如学校共享存储不足，启用 fp8 量化将占用降到约 325 GB，精度损失小于 1%。如仍不足，可仅缓存 LIBERO-Spatial 单个 suite（约 81 GB）先验证 FSF 收敛性。备选方案是 streaming VGGT（训练时在线计算），代价是 GPU 小时增加 3-5 倍。

此外，由于 π₀-FAST 主干不含 DINOv2，focus mask 的 Source C 需新增一遍 DINOv2 离线预提取：与 FastVGGT 同批，对全部 1.3M frame 跑冻结 DINOv2 提取 CLS-to-patch attention（每帧仅 256 维，bf16 约 0.5 KB），合计约 0.5–1 GB [待确认]，相对 FastVGGT 特征缓存可忽略；一次性预提取算力约 [待确认] GPU 小时（DINOv2 参数量小于 VGGT，预计低于 FastVGGT 的 11 GPU 小时）。

### 3.6 LoRA 微调配置

target_modules 设为 q_proj / k_proj / v_proj / o_proj，r=16，alpha=32，dropout=0.05。单卡 A100 80GB 显存占用：VLA forward+backward 因 3B 主干显著低于原 7B 主干的约 32 GB（具体值 [待确认]），FastVGGT 缓存加载约 1 GB（vs 在线计算 11 GB），projector 与 optimizer state 约 5 GB，DINOv2 attention 已离线预提取、训练时仅读取缓存张量（<0.5 GB），合计低于原 7B 配置的约 40 GB，单卡 A100 80GB 余量更为充裕。

每个 variant × 每个 seed 的训练步数沿用 π₀-FAST 微调配置（步数与单 seed 时长 [待确认]；原 7B 主干约 50K step / 8-12 小时，3B 主干预计相当或更低）。

## §4 评测设计

### 4.1 仿真评测：LIBERO-Plus 7 维

主仿真评测采用 LIBERO-Plus 的 7 维扰动评测集（物体布局 / 相机视角 / 机器人初始状态 / 语言指令 / 光照 / 背景纹理 / 传感器噪声）。每维度对每个变体跑 300 instance / cell × 3 seed，共 21 cell（7 维 × 3 变体），合计 21 × 300 × 3 = 18900 episode。单 episode 平均推理时间因 π₀-FAST 自回归（FAST token 逐 token 解码）显著高于 OpenVLA-OFT 的并行解码——原基于 30 秒/episode 的 158 GPU 小时估算需上调，确切值以 P0 实测 π₀-FAST 在 LIBERO-Plus 上的吞吐定档：作为示意，若单 episode 推理升至约 60–90 秒，则该主评测约 316–474 GPU 小时（[待确认]）。

干净基线（clean LIBERO 4 task suite）作为校准，每变体跑 3 seed × 130 task × 50 instance ≈ 19500 episode，但与 LIBERO-Plus 维度评测共享部分计算，实际增量 GPU 小时约 20（同样随上调的单 episode 推理时间同比例增加，[待确认]）。

统计检验：每对变体差异用 paired t-test（同 seed × 同 instance 配对），7 维度上做 Bonferroni 多重比较修正，单维度有效显著性阈值 α = 0.05 / 7 ≈ 0.007。

### 4.2 真机评测：3 任务 × 12 cell

真机评测在实验室已有的 6 自由度以上机械臂平台上进行，第三视角相机固定。3 个真机任务覆盖不同难度：T1 block-to-bowl（基础抓放，预期 clean SR 80-95%）、T2 push-block-to-target-line（中等推动力控制，无 gripper 闭合，预期 60-80%）、T3 mug-to-plate-with-occluder（有干扰物的部分遮挡场景，预期 40-70%）。

训练采集：每任务 50 demo，共 150 demo。在 LIBERO 训好的 LoRA 基础上做小规模 calibration fine-tune（10-20 step），不全调，主要校准摄像头视角与物体外观分布差异。

评测设计 12 cell = 3 任务 ×（3 扰动 + 1 clean）。3 个扰动维度与 LIBERO-Plus 部分对应：D1 光照（暗光 / 侧射强光 / 频闪选 3 个子条件）、D2 背景纹理（木纹 / 杂乱 / 反光选 3 个子条件）、D3 初始位置（5×5 cm grid 9 个起点中选 3 个）。每 cell 跑 3 变体 × 30 episode，物理 reset 在 3 变体之间共享，合计物理执行约 12 × 30 = 360 episode 的 reset 工作量。

单 episode 时长：reset 20s + 推理（π₀-FAST 自回归解码较 30s 有所增加，[待确认]）+ 评估 10s，约 1–1.5 分钟。整 cell 30–45 分钟纯执行 + 10 分钟调试。考虑团队真机经验有限，真机阶段安排充足 buffer：12 cell 纯执行约 8–12 小时，连同调试、数据落地与可能的 SDK 适配，从容分布在延长后的真机窗口内完成（时间线与降级路径见 §5.1 与 §6 R4/R5）。

### 4.3 5 个 H 假设的形式化

**H1（仿真鲁棒性增益）**：在 LIBERO-Plus 7 维扰动评测中，V3 (+InSpire+FSF) 相对 V1 (vanilla) 在至少 2 个扰动维度上 SR drop 显著减小 ≥ 5pp（paired t-test, p < 0.05, Bonferroni 修正）。基于先验机制分析，最可能显著的维度是 viewpoint 与 robot initial state（与 3D 表征对齐对几何变化敏感的预期一致），最不可能显著的是 language instruction（LIBERO-Plus 论文已报该维度对所有 VLA 都不显著）。预期 V3 在 viewpoint 维度上相对 V1 提升 6-10pp（InSpire 贡献 3-5pp + FSF 贡献 3-5pp 叠加）。

**H2（叠加效应）**：V3 平均 SR 显著高于 V2 (+InSpire) 至少 1.5pp（paired t-test, p < 0.05）。这一阈值匹配 Spatial Forcing 原论文相对 OpenVLA-OFT baseline 的提升幅度（+1.4pp 平均、+1.5pp 长程 LIBERO-Long，依据 arXiv:2510.12276 Table 1；π₀-FAST 专属的 SF 增益数字尚未公开，本项目采 SF 在 OpenVLA-OFT 主干上的效应量作为先验阈值）。H2 与 H1 的关系是：H1 验证 V3 vs V1 的整体增益，H2 单独验证 FSF 在已有 InSpire 基础上的边际贡献。

**H3（Failure Predictor）**：用 V2 或 V3 的方向词预测置信度时序训练 LightGBM 二分类器，预测同 episode 最终成败 AUC ≥ 0.65（5-fold cross validation）。这是 v2 已验证的失败检测假设，本项目继承并预期 V3 因 FSF 训练让 hidden state 更 3D-aware 而进一步提升。

**H4（真机鲁棒性增益）**：V3 在真机 12 cell 平均扰动 SR drop 显著比 V1 小 ≥ 3pp（paired t-test, p < 0.10）。

**H4 阈值辩护**：3pp 阈值（与 H1 仿真 5pp 阈值的差异）在 12 cell × 30 episode/cell 的样本规模下对应统计 power 0.7-0.8，是与真机样本 power 匹配的合理阈值，非事后调整——本项目立项阶段即按此样本规模反推确定阈值。H1 仿真之所以用 5pp 是因为单 cell 有 300 instance × 3 seed = 900 episode 的样本规模，可以支撑更严苛的 effect size 阈值；真机 episode 数受人工预算约束（不可能像仿真一样跑 300 instance/cell），按相同 power 反推阈值需要降到 3pp。p 阈值放宽到 0.10 同样来自真机样本规模约束，并非降低显著性标准——同等 effect size 下真机的 p 值天然比仿真高。

**H5（双源 focus 优于单源/均匀）**：在 A3 mask 来源 ablation 中，M-AC（双源融合）平均 SR 满足两条不等式：M-AC > max(M-A, M-C) + 1pp，且 M-AC > M-None（Spatial Forcing 原版均匀监督）+ 1.5pp（p < 0.05）。同时 M-AC > M-Random + 3pp 验证"focus 位置正确"而非"mask 多样性"是有效来源。H5 是 v2plus 相对 Spatial Forcing 原论文最直接的方法学贡献假设。

### 4.4 多重比较修正与 power 估算

H1 与 H4 均涉及多维度检验，采用 Bonferroni 修正。H1 在 7 维度上单维度 α = 0.007；H4 在 4 扰动条件（3 扰动 + 1 clean baseline）上单条件 α = 0.025。H2 与 H3 单一比较，不修正。H5 在 5 种 mask 配置上的 4 对比较，修正后 α = 0.0125。

statistical power 估算：H1 单 cell 300 instance × 3 seed = 900 episode，paired-difference 5pp 阈值的 power 约 0.85；H4 单 cell 30 episode，paired-difference 3pp 阈值的 power 约 0.7-0.8。H2 与 H3 在 26000 LIBERO demo × 3 seed 的样本规模下 power 接近 1。

## §6 风险与缓解

### 6.1 R1 · FSF Loss 在 π₀-FAST 上不收敛或退化 SR

严重度高，概率约 30%。触发条件是 M3 末 V3 vs V1 clean SR 退化超过 10pp，或 FSF loss 在 1000 step 后未下降 50%，或训练 loss 出现 NaN。缓解措施按优先级：P0 末在 5 个 LIBERO-Spatial demo 上跑 100 step FSF 训练做预先验证（loss 应下降至少 30%）；β 系数从 0.5 起步，必要时按 2× 增量降到 0.1 再到 0.05；逐层冻结策略（FSF 训练时先冻结所有 LoRA 仅训 projector，再解冻 LoRA）；warmup 策略（前 1000 step β=0，后线性 ramp 到 β=0.5）；最坏 fallback 切到 V2 only。

### 6.2 R2 · VGGT 离线缓存超磁盘或缓存失败

严重度高，概率约 30%。触发条件是学校共享存储不足 650 GB，或 WebDataset 多进程 IO 瓶颈，或 FastVGGT 全量预提取 OOM。缓解措施：fp8 量化（650 GB 降到 325 GB，精度损失 < 1%，P0 末验证）；仅缓存 LIBERO-Spatial 1 个 suite（81 GB）先验证 FSF；学校大存储节点或阿里云 OSS 冷存储（约 ¥30/月/100GB，1 年 ¥360 在 ¥400 实验耗材预算内）；streaming VGGT 训练时在线推理（GPU 小时 ×3-5 但占用磁盘 0）。新增的 DINOv2 CLS attention 离线缓存仅约 0.5–1 GB（[待确认]），对本风险影响可忽略。

### 6.3 R3 · Focus mask 双源融合无显著差异（H5 失败）

严重度中，概率约 40%。触发条件是 A3 ablation 显示 M-AC 与最佳单源差异 < 0.5pp，或 M-AC 与 M-None 差异 < 0.5pp。缓解措施：扫描融合策略与阈值找 sweet spot；改叙事降级（"任意 focus 都比 uniform 好"仍是有意义结论）；主提交改单源（M-A 与 v2 工具复用度 100%）；如时间允许 stretch ablation 引入额外 mask 来源验证。H5 的 fallback 不影响 H1 / H3 / H4 的成立。

### 6.4 R4 · 真机数据不足以支撑 fine-tune

严重度中，概率约 45%。触发条件是 M8 真机 V1 zero-shot SR < 30%，或 V3 fine-tune 后 zero-shot SR < V1 - 10pp。缓解措施：调整为仅 calibration fine-tune（10-20 step，不全调）；扩到每任务 100 demo（增加约 5 天人工）；仅 zero-shot 真机评测不做 fine-tune（LIBERO LoRA 直接迁移）；真机降级为 qualitative case study（10 episode/任务）；极端情况减少任务数到 1-2 个。

### 6.5 R5 · 真机平台具体可用性

严重度中，概率约 25%。立项阶段实验室已确认具备 6 自由度以上机械臂、第三视角相机与工作台等基础硬件，但具体型号、可借时段、SDK 兼容性的细节需在 P0 第一周完成确认。如型号是 LeRobot 主线原生支持型号（SO-100 / Koch / Reachy2 等），按 12-13 天真机人工预算执行。如型号需要团队自实现 SDK adapter（如某些 Franka / xArm / AgileX 配置），P0 阶段额外 1-2 周工程量，P1 时段相应弹性扩展 0.5 个月。如 P0 末确认平台不可用，真机降级为 qualitative case study，论文叙事仍以仿真为主、真机作为 preliminary real-world results。

### 6.6 R6 · π₀-FAST 自回归推理使 LIBERO-Plus 评测超 GPU 预算

严重度中，概率较原估上调。根因是 π₀-FAST 自回归（FAST token 逐 token 解码）单 episode 推理时间显著高于 OpenVLA-OFT 的并行解码，使原基于 30s/episode 的 158 GPU 小时主评测估算需上调（见 §4.1/§5.3），P2 评测超 GPU 预算的概率随之上升。缓解措施：分批跑（每天 1-2 维度）；优先跑最可能验证 H1 的 viewpoint 与 state 维度；将 instance 数从 300 降到 200 甚至 150（power 仍 0.6+）；评估并引入 FASTer[26] 等自回归推理加速工作以提升解码吞吐；评测算力优先由学校共享 GPU 承担，将 stretch ablation 推到论文 stretch 章节不进主对比。需注意此风险源于 π₀-FAST backbone 本身的自回归解码、与 FSF 无关——FSF 仅在训练期生效，对推理零侵入这一结论不受影响。
