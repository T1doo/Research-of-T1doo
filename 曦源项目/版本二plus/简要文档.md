# FSF-VLA 项目简要文档

## §1 项目要解决什么问题

视觉-语言-动作（VLA）模型作为机器人通用控制的新兴范式，已在仿真任务上展现出强大能力——主流 VLA 在 LIBERO 等标准基准上的成功率普遍超过 95%。但 LIBERO-Plus 在 7 个扰动维度（物体布局、相机视角、机器人初始状态、语言指令、光照、背景纹理、传感器噪声）上的系统评测显示，同样的模型一遇到这些扰动，成功率会从 95% 断崖式跌至 30% 以下。这种"实验室高分、出门即跌"的现象是 VLA 走向实用的关键障碍。

学界针对扰动鲁棒性有两条互补思路：一条是在模型输出层引入"显式空间提示"，让模型先回答物体相对机器人的方向再生成动作（以 InSpire 为代表）；另一条是在视觉特征层引入"隐式 3D 监督"，用一个冻结的 3D 基础模型作 teacher 对视觉嵌入做表征对齐（以 Spatial Forcing 为代表）。这两条思路各自得到验证，但它们的协同效应、以及在统一的多维扰动评测与小规模真机扰动评测两轴下的对照实验，目前仍是空白。

本项目以一个开源 VLA 主干（π₀-FAST）为统一基础，系统验证"显式方向词提示与聚焦式 3D 表征对齐"两层互补机制在多扰动评测下的鲁棒性增益，并通过真机扰动评测补足仿真到真实的迁移验证。研究结果可为后续 VLA 鲁棒性研究提供可复用的对照实验框架与开源工具集。

## §2 方法核心创新

### 2.1 三个变体的最简对照

主对照实验设计 3 个变体，全部基于同一 π₀-FAST 主干、同一 LoRA 配置、同一训练数据：

- V1（vanilla）：原 π₀-FAST + LoRA，作为基线
- V2（+InSpire）：V1 之上前置方向词 prompt + 7 类方向词 head + 方向词交叉熵 loss
- V3（+InSpire+FSF，主创新）：V2 之上加入聚焦式 3D 表征对齐通道

V1/V2 对照 v2 版本的标杆，V3 是 v2plus 的核心增量。

### 2.2 v2plus 的核心增量：聚焦式 3D 表征对齐

在 V2 基础上引入三个组件构成 V3。第一是 FastVGGT 离线特征缓存——以 CVPR 2025 最佳论文 VGGT 的训练无关变体 FastVGGT（与 VGGT 原版 100% 接口兼容、可直接加载其 checkpoint；其 4× 加速仅在 ~1000 张长序列输入下出现，本项目对 LIBERO 单帧离线抽特征几乎用不到该加速，单帧吞吐以 P0 实测为准）作冻结 teacher，对全部训练数据一次性预提取空间表征。第二是聚焦式 3D 表征对齐 loss——在 VLA 主干 PaliGemma（Gemma-2B）约 75% 深度的中间层（~Layer 13-14，由 Spatial Forcing 原论文在其 32 层主干上确定的最优层 Layer 24 等比例映射到 Gemma-2B 的 18 层）抽取视觉 token 隐状态，与 FastVGGT 倒数第二层 token 做 per-token 余弦相似度对齐，无需任何显式 3D 输入。第三是双源融合焦点掩码——由方向词 ground truth 反向投影得到的 2D Gaussian（提供任务相关性先验，与 v2 工具复用）与 DINOv2 CLS attention（提供独立自监督视觉模型的视觉显著性先验；π₀-FAST 视觉编码器仅含 SigLIP-So400m、不含 DINOv2，故 DINOv2 作为新增的冻结离线编码器一次性预提取并缓存，与 FastVGGT 同样仅在训练期使用）双源加权融合后三段量化（前景 1.0 / 上下文 0.5 / 背景 0.1），作为对齐 loss 的 per-token 监督权重。

三段量化的直接语义对应是师哥提出的"任务相关位置应被强监督、不相关背景应被弱监督"。背景权重不设为 0 而设为 0.1 是因为硬遮蔽会让背景区域完全丢失梯度信号，反而损害 VLA 主干对背景的几何理解。

### 2.3 工程设计的关键友好性

聚焦式 3D 表征对齐仅在训练时使用，推理时所有新增组件被丢弃，仅运行 student VLA——因此推理 latency 与原始 π₀-FAST 完全相同，对部署 friendly。FastVGGT teacher 完全冻结，且通过离线缓存避免训练时反复推理，全 LIBERO 数据集一次性预提取的 GPU 小时按原版 VGGT 单帧吞吐重新估算、[P0 实测确认]（原"约 11 GPU 小时"基于不适用于单帧的长序列 4× 加速）。focus mask 中的 DINOv2 CLS attention 同样由一个新增的冻结 DINOv2 编码器离线一次性提取并缓存（与 FastVGGT 缓存同批），训练与推理期均不在线运行 DINOv2；其预提取算力与存储都很小（[待确认]，远小于 FastVGGT 特征缓存）。

## §3 怎么验证

### 3.1 5 个核心假设

H1（仿真鲁棒性增益）：主对照评测聚焦相机视角、机器人初始状态两个核心维度（由机制分析选定），V3 相对 V1 至少在其一上成功率下降幅度显著减小 ≥ 5pp（p < 0.05, Bonferroni over 2 维），并预期两维皆显著；其余 5 维作 stretch 评测提供广度。

H2（叠加效应）：V3 平均成功率显著高于 V2 至少 1.0pp（p < 0.05）——验证 FSF 在已有 InSpire 基础上的边际贡献。注意这是 FSF 叠加在 InSpire 之上的边际效应，可能小于 SF 相对 vanilla 基线的独立增益，故阈值取得比 SF 独立效应更保守。

H3（运行时失败检测）：用 V2 或 V3 的方向词预测置信度时序训练 LightGBM 二分类器预测最终成败，AUC ≥ 0.65（5-fold cross validation）。

H4（真机：preliminary real-world evidence）：不作强显著性假设，而作为初步真机证据——看 V3 相对 V1 在各扰动维度上成功率下降是否方向一致地更小；指示性参考量级为 12 cell 平均扰动 drop 减小约 ≥ 3pp（参考量级、非显著性门槛）。

**H4 样本规模说明**：真机受人工预算约束，12 cell × 30 episode/cell 只支撑初步证据而非确证性显著检验——该规模粗略对应数量级上 0.7-0.8 的 power，3pp 仅作指示性参考量级。H1 仿真单 cell 有 900 episode 可支撑更严苛阈值；真机跑不到同等样本量，故 H4 定位为方向一致性证据，结论以"V3 多数扰动维度 drop 方向一致地小于 V1"陈述，不以单一 p 值过线为准。

H5（双源 focus 优于单源/均匀）：在 mask 来源消融中，双源融合配置（M-AC）平均成功率优于任一单源 ≥ 1pp，且优于 Spatial Forcing 原版均匀监督（M-None）≥ 1.5pp（p < 0.05）。H5 是 v2plus 相对 Spatial Forcing 原论文最直接的方法学贡献假设。

### 3.2 评测矩阵

仿真主对照评测：聚焦相机视角 + 机器人初始状态 2 维 × 3 变体 × 3 seed × 200 instance/cell = 6 cell、3600 episode；因 π₀-FAST 自回归推理慢于并行解码，主对照收缩至 2 核心维度后约 60–90 GPU 小时（[待确认]，按 60–90 秒/episode 示意；较 7 维全量 316–474 小时大幅收缩）。其余 5 维与 300 instance 作 stretch 补跑。

真机评测：3 任务（block-to-bowl 基础抓放 / push-block-to-target-line 中等推动 / mug-to-plate-with-occluder 部分遮挡）×（3 扰动维度 + 1 clean）= 12 cell，每 cell 30 episode，共 360 物理执行 episode；训练采集 3 × 50 = 150 demo。3 个扰动维度与 LIBERO-Plus 部分对应：光照（暗光 / 侧射强光 / 频闪）、背景纹理（木纹 / 杂乱 / 反光）、初始位置（5×5 grid 选 3 点）。

6 组消融实验验证假设的不同方面：teacher 取层（A1）、student 监督层位（A2）、focus mask 来源（A3，H5 核心）、对齐 loss 权重（A4）、背景权重（A5）、视觉利用机制可视化（A6）。

## §4 风险与缓解

6 个主要风险按严重度排序：FSF loss 在 π₀-FAST 上不收敛或退化 SR（高，30%）通过 β 系数扫描与逐层冻结 warmup 缓解；π₀-FAST 自回归推理慢使评测算力超预算（中→低，已通过主对照矩阵收缩至 2 核心维度 × 200 instance 预先缓解，残余风险用分批跑、降 instance、FASTer 加速 arXiv:2512.04952 与学校 GPU 兜底）；VGGT 离线缓存超磁盘（高，30%）通过 fp8 量化降到 325 GB 与单 suite 验证缓解（新增 DINOv2 attention 缓存仅约 0.5–1 GB，可忽略）；focus mask 双源融合无显著差异即 H5 失败（中，40%）改叙事降级为"任意 focus 都比均匀好"仍是有意义结论；真机数据 50 demo/任务不足以 fine-tune（中，45%）通过 calibration fine-tune、扩 demo 量、真机阶段时间弹性向后顺延缓解；真机平台具体型号与 SDK 兼容性确认（中，25%）需在 P0 第一周完成。

最坏 fallback 是退到 v2 only（关闭 FSF 通道，仅运行 V1+V2），与 v2 共享全部代码框架，切换成本不超过 1 周。真机部分作为核心但时间弹性的工作流，若受团队经验或进度限制可降级为定性 case study、论文以仿真（H1/H2/H3/H5）为主仍成立。所有 fallback 路径均确保仍能产出 workshop short paper。

## §5 与现有工作的差异化

与本项目最相关的并行工作是 GLaD（Geometric Latent Distillation for VLA，arXiv:2512.09619，2025-12）：它同样用冻结 VGGT 作 teacher、在 LLM 中间层把视觉 token 隐状态与 VGGT 几何特征对齐（与 Spatial Forcing 同族的中间层蒸馏，LIBERO 94.1%），消融含"Layer 24/32 vs 最终层对齐"与"Weighted Fusion（VGGT 与 DinoSigLIP 特征级加权融合）"，并做了 LIBERO-PRO 鲁棒性评测与注意力分析。本项目相对 GLaD 的差异即新颖性核心：(a) 本项目叠加 InSpire 显式方向词提示，GLaD 没有；(b) GLaD 的"加权"是特征级融合（VGGT+DinoSigLIP 特征组合），本项目 focus mask 是按 per-token 任务相关性 × 视觉显著性调制对齐损失权重——作用于损失权重而非特征，机制不同；(c) 本项目含小规模真机扰动评测，GLaD 为纯仿真。此外，InSpire 显式方向词提示与 Spatial Forcing 隐式 3D 对齐据我们检索未见在同一框架内协同验证；Spatial Forcing 原版对所有 visual token 均匀对齐、未利用任务相关性先验。这些点共同构成本项目的核心新颖性。

与 FocusVLA（arXiv:2603.28740）的关系需要单独说明，因为 FocusVLA 也使用了 VGGT。两者的核心差异是 VGGT 接入位置与"focus"作用位置完全不同。本项目沿用 Spatial Forcing 在 VLA 中间层（~Layer 13-14）做 teacher 蒸馏的范式，对齐 loss 的梯度直接反传至 VLM 主干 Layer 1 至对齐层；FocusVLA 把 VGGT 作为 policy 阶段的并行编码器，与 VLM 主干无梯度交互。本项目的"focus"作用于 align loss 的 per-token 监督权重；FocusVLA 的"focus"作用于 attention 计算的 top-K 视觉 token 筛选。

FocusVLA 基于 VLA-Adapter + Qwen2.5-0.5B（总可训参数约 343M），与 π₀-FAST（PaliGemma/Gemma-2B，约 3B）主干和参数量级不同，单看 LIBERO Avg pp 差距不具方法学可比性。一个可直接引证的结构性事实是：FocusVLA 最佳成绩 98.7%（依据 arXiv:2603.28740 Table 1）采用的是 VLM+DS+VLM 配置（无 VGGT），主动舍弃 VGGT；Spatial Forcing 在同表的成绩为 98.5%。FocusVLA §4.3 Ablation Study "Visual Representation" 子节中作者明确指出，由于架构约束 VGGT features 只能在 policy 阶段注入以避免干扰预训练 VLM，其梯度相对较弱且不稳定，限制了性能上限。本项目选择继承 Spatial Forcing 中间层 teacher 对齐范式、不采纳 FocusVLA policy 阶段并联范式的方法学依据由此确立。

近期相关 saliency-guided VLA 工作（如 AttentionVoxel 的 DINOv2 CLS attention 提升至 3D voxel grid 用法）证明了 attention 引导的有效性，但作用空间不同。本项目相对这些工作的独特定位有三点：将 saliency mask 用于调制 VGGT-VLA 蒸馏对齐 loss 而非 attention 计算；方向词 ground truth 与 DINOv2 双源融合而非单源；三段量化（1.0 / 0.5 / 0.1）的显式权重设计直接对应"前景 / 上下文 / 背景"语义。

3D-aware VLA 工作大致分为五类（直接 RGB-D 输入、显式点云、单目深度估计辅助、多视图重建、隐式 3D 对齐）。本项目属于隐式对齐路线，与其他路线的差异是推理路径零侵入与零额外 latency——FastVGGT 与 DINOv2 均仅在训练时使用（离线缓存），部署时只需 π₀-FAST 主干。
