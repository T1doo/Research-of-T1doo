# 09 · 与版本二差异化说明

> **本文档专为与导师讨论时单独使用**——浓缩版本三 vs 版本二的所有关键差异，1-2 页内说清楚。便于在 30 分钟会议中协助导师快速判断。

## 一句话差异化

| | 版本二 v2 | 版本三 |
|---|---|---|
| **核心姿态** | 轻量训练 + 评测 | 纯评测 / 诊断（零训练） |
| **核心方法** | InSpire 显式提示插件 LoRA 微调 + 视觉利用 ablation | SAM mask 当外部视觉真值 + attention-mask IoU |
| **研究问题** | "显式提示能否提升 VLA 鲁棒性" | "VLA 视觉聚焦质量是否真的与鲁棒性有因果关系" |
| **与 FocusVLA 的紧密度** | 间接（InSpire 是 alternative path） | 直接（回应 FocusVLA 自述局限 #3 #4） |

## 6 维详细对比

### 1. 工程量

| | 版本二 v2 | 版本三 |
|---|---|---|
| 训练 VLA | ✅（LoRA + cls head + loss 联合） | ❌（零训练） |
| 写方向词 GT 标注脚本 | ✅（5 天） | ❌ |
| 写 cls head + loss 联合 | ✅（3 天） | ❌ |
| LoRA 微调调参 | ✅（7 天） | ❌ |
| attention hook | ✅（2 天，可选） | ✅（3 天，核心组件） |
| SAM pipeline | ❌ | ✅（5 天） |
| IoU + K\* 计算器 | ❌ | ✅（3 天） |
| 评测 + 统计 + 论文 | 25 天 | 31 天 |
| **总工程时间** | **~54 天** | **~55 天** |

**关键差异**：总工程时间持平，但版本二有 ~28 天是高风险训练工程（GT 噪声 / cls 收敛 / loss weight 调参），版本三 ~18 天是低风险"拼接现成工具"（SAM + attention hook）。**版本三训练侧风险为零**。

### 2. Novelty 来源

**版本二 v2**：
- "InSpire 在 LIBERO-Plus 多维扰动下的系统验证"——InSpire 原论文只在 clean 或自家 OOD 评测
- "视觉利用 ablation（token dropout / patch mask / attention 可视化）"作为机制证据

**版本三**：
- "用 SAM mask 作为外部视觉真值"——文献中**完全的空白**（5 个未被填补的研究问题，详见可行性 + novelty 报告 § B）
- "多维扰动下的 attention-mask IoU 量化"——FocusVLA / Oat-VLA / SlotVLA 等"视觉聚焦类 VLA" 完全未做
- "理论最优 K\* 分布画像"——对所有 token pruning 工作的实证检验
- "SAM-VLA 联动崩塌"——新的 failure mode 诊断维度

**版本二** novelty 集中在 **方法验证**（InSpire 真的鲁棒吗）；**版本三** novelty 集中在 **机制诊断**（视觉聚焦质量真的影响鲁棒性吗）。后者解决的问题更根本。

### 3. 与 FocusVLA 故事的紧密度

**版本二 v2**：
- InSpire 是 *alternative path*（FocusVLA 改架构 vs InSpire 显式提示），不直接回应 FocusVLA 自述局限
- 论证 framing 复杂："用 OpenVLA-OFT 作 Focus-style 代理" 曾被外部审阅指出不严谨

**版本三**：
- 直接回应 FocusVLA 自述局限 #3「VLM 内部的视觉利用没涉及」 → 本项目用 SAM mask 评估 VLA attention 质量
- 直接回应 FocusVLA 自述局限 #4「未在 robustness benchmark 上系统评测」 → 本项目在 LIBERO-Plus 7 维评测
- 论证 framing 直接：FocusVLA 留了空白 → 我们填

**版本三与导师"FocusVLA 方向深挖鲁棒性"指引的对齐度比版本二高**。

### 4. 主要风险

| | 版本二 v2 | 版本三 |
|---|---|---|
| 风险 1 | InSpire 代码未释（60%）→ 自己手写 200 行 | GroundedSAM 在 LIBERO 渲染图上失效（40%）→ 切轨道 B（仿真 GT mask） |
| 风险 2 | 方向词 GT 推导噪声 > 30%（40%）→ 滑动平均 + corner case | H1 否定（25%）→ 这本身是反直觉发现 |
| 风险 3 | cls head 不学（30%）→ 调 loss weight | 评测预算超支（35%）→ 分层抽样 |
| 风险 4 | LoRA 工程坑（30%） | 无 |
| 风险 5 | FocusVLA 代码未释（60%）→ OpenVLA-OFT 代理 | FocusVLA 代码未释（50%）→ 不依赖 |

**关键**：版本二 5 个风险中 3 个是训练侧风险（GT / cls / LoRA），版本三完全规避。版本三主要风险（GroundedSAM 失效）有**明确的 48h 切换 Plan**，风险可控性更强。

| ICLR Robot Learning workshop | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐（**主投**，2-3 月 deadline 落在 P5 开始处）|

### 6. 预期产出对照

| 产出物 | 版本二 v2 | 版本三 |
|---|---|---|
| workshop short paper | ✅（"InSpire 鲁棒性"故事）| ✅（"SAM-as-Probe 视觉聚焦质量与鲁棒性"故事）|
| 开源代码 | InSpire plugin（200 行）| `probe/` 工具包（attention hook + SAM + IoU + failure predictor，~800 行）|
| 数据集 release | 方向词标注数据 | attention-mask IoU 数据 + 失败模式画像 |
| 预训权重 | 2 变体 LoRA checkpoint | 无（不训练）|
| 反直觉发现可能性 | 中（H1 互补可能不成立）| 高（H1/H2/H3 任一否定都是有价值的负面 finding）|

**版本三的 `probe/` 工具包对社区的工程价值更大**——任何后续做 VLA 视觉聚焦评测的研究都可直接复用。

## 不应该说"版本三完全替代版本二"

虽然版本三有诸多优势，但版本二的工作仍有独立价值：

1. **版本二回答了"显式提示能否解 FocusVLA 局限 #2 初始状态敏感"** —— 这是 actionable 的方法贡献
2. **版本二的 InSpire 方向词 GT 标注脚本可作为社区贡献** —— 即使本项目不走这条路
3. **版本二的视觉利用 ablation 设计本身（token dropout / patch mask）** 可在版本三 P5 期间作为 future work 讨论

**最佳定位**：版本二是"方法学贡献"路线；版本三是"机制诊断"路线。两者长期互补。对一年期本科生项目而言，**选一条做扎实即可，版本三在 risk-adjusted novelty 上略胜**。

| "与 FocusVLA 故事的关系？" | "FocusVLA 自述局限 #3 #4 是本项目核心切入点。直接回应 FocusVLA，不需要 framing 折衷" |
| "如果 H1 否定怎么办？" | "这本身是反直觉的有价值发现——否定 FocusVLA / Oat-VLA / SlotVLA 等视觉聚焦类 VLA 工作的根本假设。可投 negative-finding workshop" |
| "如果 SAM 在 LIBERO 上失效怎么办？" | "Plan B 是切换到仿真 GT mask（robosuite camera API），48h 内完成切换，研究问题与方法学几乎不变" |
