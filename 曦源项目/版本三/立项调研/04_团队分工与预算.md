# 04 · 团队分工与预算

## 1. 团队成员与角色

| 同学 | 角色 | 主要承担 |
|---|---|---|
| **A 同学**（项目负责人） | **工程主责** | 探针管线工程（attention hook / SAM 推理 / IoU 计算器）；OpenVLA-OFT + SmolVLA 推理；failure predictor 训练 |
| **B 同学** | **评测主责 + 论文主笔** | 大规模评测调度（24K episode）；统计分析（配对 t-test / Pearson / K\* 分布画像）；论文撰写 |

两人在 W2 末 Go/No-Go 决策、P2 末 H1 验证、P4 末 failure predictor 收尾、P5 论文投稿 4 个关键节点共同决策。

| 月份 | A 同学（工程主责） | B 同学（评测 + 论文主责） |

## 4. GPU 算力分项

| 阶段 | GPU 时长（h） | 占总算力 | 用途 |
|---|---|---|---|
| P1 探针管线 | 25 | 12.5% | smoke test + sanity check + mIoU 校准 |
| P2 大规模 IoU 评测 | 120 | 60% | 24K episode × 30s（含 SAM 推理） |
| P3 K\* 画像 | 10 | 5% | 离线 K\* 计算（SAM mask 已 dump） |
| P4 failure predictor | 15 | 7.5% | attention 重新 dump + LightGBM 训练 + AUC 测试 |
| P5 写作（重跑实验） | 25 | 12.5% | rebuttal 期可能重跑关键 cell |
| **小计** | **195** | **97.5%** | 留 5h ≈ 2.5% 应急 |

**关键缓解**：若 P2 评测超 120h，启动**分层抽样**：clean + 主战场 4 维（layout / viewpoint / state / texture）全量 500 instance；次要 3 维（instruction / light / noise）取 200 instance（1/2.5）。可压到 80h，留余量给 P5 重跑。
