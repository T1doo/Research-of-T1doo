## W1 · Day 3-5（06-03 ~ 06-05）：探针管线核心组件

### A 同学任务

- [ ] **D3**：写 `probe/attention_hook.py` —— hook OpenVLA-OFT last-layer cross-attention
  - 关键路径：`openvla-oft/prismatic/models/vlms/prismatic.py`
  - 注入点：在 forward 中找 cross-attention 模块（如 `self.llm.layers[-1].cross_attn`），注册 forward hook 把 attention weight 存到 dict
  - 输出：每 forward step 一个 (B, head, query_len, key_len) tensor
  - 测试：跑 1 个 episode，确认 attention weight shape 与 LIBERO 输入对齐
- [ ] **D4**：写 `probe/sam_pipeline.py` —— GroundedSAM + SAM2 端到端
  - 输入：(image, instruction)
  - 步骤：
    1. 从 instruction 抽 noun phrase（用 spaCy 或简单字符串规则）
    2. GroundingDINO 找 box（threshold 0.3 起步）
    3. SAM 从 box 出 mask
    4. SAM2 在后续帧做视频传播
  - 输出：每帧的 mask（H × W 二值图）
- [ ] **D5**：写 `probe/iou.py` —— mask → patch grid 投影 + IoU
  - mask → patch grid（14×14）的投影逻辑：每 patch 内 mask 像素占比 ≥ 0.3 则 patch_mask=1
  - IoU 计算：`|attn_topK ∩ patch_mask| / |attn_topK ∪ patch_mask|`
  - 同时输出：mask coverage @K、K\*（覆盖 mask 90% 所需的最小 K）

### B 同学任务

- [ ] **D3-D4**：写评测脚本骨架 `eval/eval_libero_iou.py`
  - 输入：backbone path + LIBERO suite name + condition (clean / 7 维)
  - 输出：每任务 SR + 每 episode 的 attention map dump + SAM mask dump + IoU
  - 这个脚本 P2 评测会反复用，提前打磨
- [ ] **D5**：写 mIoU 校准实验脚本 `eval/calibrate_groundedsam.py`
  - 输入：5 张 LIBERO 任务图 + 任务指令 + LIBERO 自带 object pose（仿真 GT mask 来源）
  - 输出：GroundedSAM mIoU vs 仿真 GT mask（10-50 张图的均值）

### 同步检查点

- [ ] **D5·末**：4 个组件是否跑通？
  - 是 → 进入 W2
  - 否 → 局部 debug，最迟 D7 必须通

### W2 末 Go/No-Go

- [ ] **D5·晚**：召开团队会议，回答 3 个问题：
  1. **GroundedSAM 在 LIBERO 渲染图上 mIoU 多少**？≥ 0.6 → 走轨道 A；0.4-0.6 → 调参或切；< 0.4 → 直接切轨道 B
  2. **50 episode 上的 IoU 与成败有相关迹象吗**？（不要求显著，只要看到趋势）—— 成功 episode IoU 是否 > 失败 episode 至少 0.05？
  3. **K\* 直方图中位数大概多少**？< 80 → H2 有戏；100-200 → H2 边缘；> 200 → 反直觉发现

| 三问回答 | 决策 |
|---|---|
| (1) ≥ 0.6 + (2) 趋势对 + (3) K\* < 80 → | ✅ **Plan A 全力推进**，进入 P1 末扫尾 |
| (1) 0.4-0.6 但 (2)(3) 趋势对 → | ⚠️ 给自己 2 周调 GroundedSAM 参数（box threshold、文本提示工程），不行则切 B |
| (1) < 0.4 → | 🛑 直接切 [[02_备选方案_仿真GT_mask兜底]] |
| (2) 趋势完全反 → | ⚠️ 检查 attention hook 层选择，确认不是 hook bug |
