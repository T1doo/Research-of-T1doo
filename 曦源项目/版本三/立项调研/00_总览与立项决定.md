# 00 · 总览与立项决定

## 1. 为什么要切换到版本三

T1doo 团队的立项方向已经历两次迭代：

- **版本一**（Efficient VLA · AAC + RobustVLA + RC-NF 三方向耦合）→ 导师反馈"耦合实现难度高、一年期产出风险大"，已舍弃。
- **版本二**（FocusVLA × InSpire 互补研究）→ 已完成 v2 scope 降级版（2 变体 + 视觉利用 ablation），见 `版本二/立项调研/08_最终方案_FocusVLA_InSpire互补研究.md`。但存在一处隐性弱点：v2 仍依赖 **方向词 GT 自动标注 + LoRA 微调 + cls head 联合训练**，这些训练侧风险未被根除。

导师在 2026-05 末的沟通中给出更明确的方向指引：

> "**继续沿 FocusVLA 方向深挖鲁棒性**，可以考虑更系统、更触及'视觉利用机制本身'的研究方式。"

团队据此提出版本三新思路：

1. **把 visual token 做成变长**（不再固定 K=256）
2. **接入物体分割模型**（SAM2 / GroundedSAM），让模型有"明确的物体级 grounding"

经过完整的调研期论文检索与可行性分析，**最终把这两条原始想法收敛到一个统一姿态**：

> **不重训 VLA，把 SAM 当探针**——用 SAM mask 作为"外部视觉真值"，系统诊断 VLA 注意力区域与扰动鲁棒性的对齐关系。"变长 token" 在评测中体现为 "理论最优 K\*" 的统计画像，而非真的去训练一个变长 K 的 VLA。

这个收敛符合用户对工程复杂度的明确约束（**纯评测/分析**，不改架构、不重训）。

## 2. FocusVLA 4 大局限与本项目的对应

FocusVLA（arXiv:2603.28740）自述 4 大局限：

| FocusVLA 自述局限 | 本项目（版本三）如何回应 |
|---|---|
| **#1**「鲁棒性对背景 + 纹理同时变化仍有限」 | 在 LIBERO-Plus 7 维评测中包含 texture / light / noise 维度，正面量化 |
| **#2**「对初始状态敏感」 | LIBERO-Plus 的 state 维度直接覆盖，IoU 退化分析可定位"是否因 attention 与物体真实位置偏移" |
| **#3**「VLM 内部的视觉利用没涉及」 | **本项目核心切入点**——用 SAM mask 作为外部真值评估 VLA attention 是否真的看任务物体 |
| **#4**「未在 robustness benchmark 上系统评测」 | **本项目核心切入点**——LIBERO-Plus 7 维 × 2 backbone × 3 seed 系统评测 |

本项目直接回应 FocusVLA 留下的 #3 和 #4 两个空白，与导师建议高度对齐。

## 3. 现实约束（与版本二一致）

| 维度 | 数值 |
|---|---|
| 团队 | **2 名大二本科生**（综述报告原假设 4-6 人） |
| 预算 | **¥10,000**（约 $1400，能租 100-200 GPU-hours 云端 A100） |
| 时间 | 1 年（2026-06 → 2027-06） |
| GPU | 学校共享 / 云端租用，单卡 ≤ 80GB |
| 目标 | **能投个 workshop 论文**（不强求顶会 main） |
| 工程上限 | **纯评测/分析**（不改 VLA 架构、不重训 VLA） |

版本三对工程上限的约束比版本二更严——版本二允许 LoRA + cls head 联合训练，版本三**完全不训练 VLA**。

## 4. 选定方案（双轨）

### 主方案 · SAM-as-Probe 纯诊断式实证研究

**核心姿态**：用 SAM2/GroundedSAM 物体 mask 作为"外部视觉真值"，把 VLA 当被测对象，离线计算 attention-mask IoU 与各种衍生指标。整个项目周期内**零 VLA 训练**。

**研究问题**：在 LIBERO clean + LIBERO-Plus 7 维扰动下，**VLA 的视觉聚焦质量（attention-mask IoU）是否真的与扰动鲁棒性（SR）有因果关系**？固定 K=256（FocusVLA 等工作的默认）是否在多场景下过度/不足？SAM 在扰动下也认错时（共同失败）vs SAM 认对但 VLA 仍崩（看错了）的失败模式比例如何？

**详细设计**：[[01_主方案_SAM探针式诊断]]

### 备选方案 · 仿真 GT mask 兜底

仅在 **P1 末（M2，2026-07 末）GroundedSAM 在 LIBERO 渲染图上 mIoU < 0.5** 时启动。技术上把 SAM mask 换成 **robosuite 仿真器原生支持的物体级 GT mask**（通过 camera API + URDF 渲染），H1 / H2 假设保留，H3 假设从 "SAM-fragility" 改为 "GT-mask drift"。

**关键**：与主方案**共用 attention dump + IoU 计算框架**，切换成本 < 2 周。

**详细设计**：[[02_备选方案_仿真GT_mask兜底]]

### 排除方案

- 真改 FocusVLA 架构（重写 Cascaded Attention + Focus Attention）→ 工程超纲，与"纯评测"约束直接冲突
- 复现 Oat-VLA / SlotVLA / TokenFLEX 等变长 / object-centric VLA → 需大量训练，novelty 已被占
- 把 SAM mask 作为弱监督训练 VLA → 重训 VLA，与约束冲突；且评审会问"为何不端到端 SAM-VLA 联合训"

详见 [[07_为什么不选其他方案]]。

## 5. 项目命名（双轨）

老师建议在 FocusVLA 方向深挖鲁棒性，本项目实际定位是"用外部视觉真值（SAM mask）来系统诊断 VLA 视觉聚焦质量"。命名兼顾两种受众：

### 主题名（推荐）

> **「基于物体分割引导的视觉-语言-动作模型视觉聚焦质量与扰动鲁棒性诊断研究」**

适合所有材料（申请书、立项书、组会汇报）作为主标题。突出：
- **「物体分割引导」** ——明确技术手段（SAM2/GroundedSAM）
- **「视觉聚焦质量」** ——明确核心研究对象（VLA 的 attention 是不是真的看任务区域）
- **「扰动鲁棒性」** ——明确应用场景与切入点（LIBERO-Plus 7 维）
- **「诊断研究」** ——明确研究姿态（评测/分析、零训练）

### 内部代号

**SAM-as-Probe**——一句话 41 字：用 SAM2 物体 mask 作为"外部视觉真值"，系统诊断 VLA 注意力区域与扰动鲁棒性的对齐关系。

### 备选名（学术风格变体）

- A：「面向视觉-语言-动作模型的分割-注意力对齐度评测框架研究」
- B：「视觉-语言-动作模型在多维扰动下的视觉聚焦机制实证诊断研究」

## 6. 核心研究问题（H 假设速览）

| 假设 | 命题 | 证伪条件 | 验证方式 |
|---|---|---|---|
| **H1（对齐假设）** | 失败 episode 的 VLA 注意力 × SAM 物体 mask IoU 显著低于成功 episode，ΔIoU ≥ 0.10，配对 t-test p < 0.05 | ΔIoU < 0.05 或 p ≥ 0.1 | 收集 ~2k 成功/失败 episode 对，配对 t-test |
| **H2（理论最优 K）** | 每场景 "任务物体 mask 覆盖所需的最小 patch 数 K\*" 中位数 ≤ 80，K\* 在 LIBERO-Plus 7 维扰动下分布漂移 ≥ 30% | K\* 中位数 ≥ 200 或漂移 < 10% | 对所有 keyframe 算 K\*，分维度直方图与 boxplot |
| **H3（联动崩塌）** | SAM mIoU 在某扰动维度上 drop ≥ 15pp 时，VLA SR drop 与 SAM mIoU drop 显著正相关 r ≥ 0.5（n=7 维 × 2 backbone）；同步失败 episode ≥ 30% | r < 0.3 或同步失败率 < 15% | 14 个数据点的 Pearson 相关 + 失败 episode 聚类 |

**关键工程同构**：三个假设的验证都只需 (i) VLA 前向 + attention dump，(ii) SAM2/GroundedSAM 前向，(iii) 离线对齐计算。**零 VLA 训练**。

详见 [[01_主方案_SAM探针式诊断]] § 2 与 [[08_最终方案_SAM_as_Probe]] § 3。
