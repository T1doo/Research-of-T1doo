# 05 · 风险与缓解

## 1. Top 3 风险（按严重度排序）

### 风险 1 · GroundedSAM 在 LIBERO 渲染图上失效

**严重度**：🔴 高 · **概率**：40%（LIBERO 渲染风格 ≠ COCO 真实图像分布）

**影响**：
- 轨道 A（GroundedSAM 自动管线）无法用，需切换到轨道 B（仿真 GT mask）
- 真机泛化叙事减弱（仿真 GT mask 不可移植）

**缓解策略**：
1. **P1 末 mIoU 校准实验**（50 张 LIBERO 图 × 4 任务类型 × GroundingDINO box threshold 4 个值 = 800 数据点）
2. **GroundingDINO box 阈值与 prompt engineering**：
   - 默认 threshold 0.3 → 尝试 0.2 / 0.25 / 0.35
   - 改 prompt 模板（"a [object]" → "the [object] on the table" → "the [object] near the robot"）
3. **不达标即切轨道 B**：实现成本 < 2 周（[[02_备选方案_仿真GT_mask兜底]] § 5）
4. **SlotVLA 备用方案**：SlotVLA 论文（ICRA 2026, arXiv:2511.06754）发布的 LIBERO+ mask 标注，可作为第三方 mask GT 来源（避免自己写仿真渲染脚本）

**触发条件**：W2 末校准实验 mIoU < 0.5 → 切轨道 B；mIoU 0.4-0.5 给自己 2 周调参；mIoU < 0.4 直接切

### 风险 2 · H1 完全否定（IoU 与 SR 无关）

**严重度**：🟡 中 · **概率**：25%

**影响**：
- 项目核心假设 "VLA 视觉聚焦质量与扰动鲁棒性有因果关系" 不成立
- 主叙事 framing 需要重写

**缓解策略**：
1. **接受反直觉发现作为有价值的 negative finding**：这本身否定了 FocusVLA / Oat-VLA / TokenFLEX 等"视觉聚焦类 VLA"的方法学根本假设——是 workshop 接受度很高的题材
2. **三层备选叙事**：
   - 第一层（理想）：H1 验证 → 写 "VLA 视觉聚焦质量决定扰动鲁棒性" 故事
   - 第二层（部分否定）：仅 1-2 维度成立 → 写 "哪些维度上视觉聚焦质量解释失败" 故事
   - 第三层（完全否定）：改写为 "VLA 失败不是因为没看对，是看对了但仍做错动作" → workshop short paper "Reconsidering Visual Utilization Hypothesis in VLA Robustness"
3. **引入第二参照**：除整个物体 mask，加入**任务关键 keypoint mask**（如手 / 物体接触点），IoU 可能更敏感
4. **错位分析**：即使 IoU 不预测成败，可能存在 "IoU 高但 attention 在错位 keypoint" 这种细粒度失败模式——纳入 P4 失败模式画像

**触发条件**：P2 末 7 维全部不显著 → 启动备选叙事第三层

### 风险 3 · 评测预算超支（实际 ≈ 350 GPU-h 超 200h 上限）

**严重度**：🟡 中 · **概率**：35%

**影响**：
- 完成全量 24K episode 评测需 350h，超 GPU 预算 ¥5,500 / ~190h
- 可能被迫减少 instance 数 / seed 数 / 维度

**缓解策略**：
1. **分层抽样**（已设计）：
   - clean + 主战场 4 维（layout / viewpoint / state / texture）每维 500 instance × 3 seed
   - 次要 3 维（instruction / light / noise）每维 200 instance × 2 seed
   - 总量降到 ~10K episode × 30s ≈ 80 GPU-h
2. **SmolVLA 主跑 + OpenVLA-OFT 子集复现**：SmolVLA 推理速度 5×，主战场 7 维全跑 SmolVLA；OpenVLA-OFT 只在 4 主战场维度跑（验证趋势一致即可）
3. **attention dump 一次性**：每 episode 跑一次推理，attention 离线存盘后 IoU 计算用 CPU
4. **SAM 推理批量化**：SAM2 视频传播一次性处理整个 episode 而非逐帧调用
5. **学校共享 GPU 优先**：FVL Lab 算力资源充足，可减少云端开销

**触发条件**：M4 末（2026-09 末）累计 GPU 用量 > ¥3,500（60% 预算）→ 启动分层抽样

## 2. 次要风险（影响有限但需注意）

### 风险 4 · attention hook 与 OpenVLA-OFT 内部结构不兼容

**严重度**：🟢 低 · **概率**：15%

**缓解**：
- OpenVLA-OFT 是开源仓库，PyTorch hook 是标准 API
- 如果 hook 失败，降级到从 model output 直接取 attention map（HF transformers 多数模型支持 `output_attentions=True`）
- 学习版本一二次调研中的 OpenVLA-OFT 注入点分析

### 风险 5 · SAM2 视频传播在 LIBERO 长 episode 上失稳

**严重度**：🟢 低 · **概率**：20%

**缓解**：
- LIBERO 单 episode 通常 30-200 steps，远低于 SAM2 在通用视频上的能力上限
- 物体被抓取后形态变化是主要风险点 → 在抓取瞬间重新 prompt SAM2（增加触发条件）
- 退化方案：每 N=10 step 重新调用 GroundedSAM 重做 mask（成本略增，但稳定）

### 风险 6 · H2 完全否定（K\* 中位数 > 200）

**严重度**：🟢 低 · **概率**：15%

**缓解**：
- 这本身是有价值的发现："固定 K=256 是合理的"——反过来支持现有 token pruning 工作
- 改 P3 叙事为"为什么固定 K=256 合理：mask 覆盖统计的反证"
- 不影响 H1 / H3 验证

### 风险 7 · FocusVLA 代码至 P4 末仍未 release

**严重度**：🟢 低（仅是 stretch goal） · **概率**：50%

**缓解**：
- 不依赖 FocusVLA 复现，本项目 backbone 是 OpenVLA-OFT + SmolVLA
- FocusVLA 在版本三的角色仅是"故事 anchor"（直接回应其自述局限），不在主路径里
- 若 release，可作为第 3 backbone 在 P4 末插入

### 风险 8 · VLA-Risk 代码 / 数据始终不释

**严重度**：🟢 低（仅 stretch） · **概率**：50%

**缓解**：
- 用 LIBERO-Para（已开源，专做语言改写鲁棒性）作为 instruction 维度的替代
- 主战场是 LIBERO-Plus 7 维，足够支撑 H1 / H2 / H3 验证

### 风险 9 · 团队工程能力跟不上

**严重度**：🟢 低 · **概率**：20%

**缓解**：
- 版本三的工程量比版本二低 ~50%（零训练）
- 主要技术栈（PyTorch hook / SAM 调用 / IoU 计算）都有公开教程
- AI 辅助开发预算 ¥800（Claude/GPT API）
- 学校共享 GPU + FVL Lab 资源
- 每月 1 次 office hour 请导师过工程关键决策

## 3. Plan A → Plan B 切换决策树

### 主决策点（P1 末，M2，2026-07 末）

```
W2 末 (M0.5) GroundedSAM mIoU 校准实验
├── A. mIoU ≥ 0.6
│   └── ✅ Plan A 全力推进，进入 P1 末扫尾
│       └── M2 末再确认 50 episode IoU 配对趋势
│           ├── 趋势对 → 进 P2
│           └── 趋势完全反 → 检查 attention hook 层选择（不是切 B 的理由）
│
├── B. mIoU 0.5-0.6
│   └── ⚠️ 给 2 周调参（GroundingDINO box threshold / prompt engineering）
│       └── M2 中（W4 末）再决策
│           ├── 升至 ≥ 0.6 → 进 A
│           └── 仍 0.4-0.6 → 切 [[02_备选方案_仿真GT_mask兜底]]
│
└── C. mIoU < 0.4
    └── 🛑 直接切 [[02_备选方案_仿真GT_mask兜底]]
        └── 切换 48h 内完成
```

### 次决策点（P2 末，M5，2026-10 末）

```
M5 末 (P2 end) H1 验证
├── A. H1 强成立（≥ 4 维 ΔIoU ≥ 0.10，p < 0.05）
│   └── ✅ 继续 Plan A P3-P5，写"VLA 视觉聚焦质量决定扰动鲁棒性"故事
│
├── B. H1 部分成立（仅 1-3 维显著）
│   └── ⚠️ 改叙事为"主要差异在哪些维度" + 维度归因画像
│
└── C. H1 完全否定
    └── 🛑 启动备选叙事第三层："Reconsidering Visual Utilization Hypothesis"
        └── 加入第二参照（keypoint mask）+ 失败模式画像作为补强
```

## 5. 重要：不要做的事

- ❌ **不要在 W2 末 mIoU 校准未做之前就大跑 P2 评测**——24K episode 跑完才发现 SAM 不准等于浪费 60% 预算
- ❌ **不要追求 100% 完美的 SAM mIoU**——0.6 - 0.8 在工程上已经够 H1 验证（信噪比足够）
- ❌ **不要为了"工程美感"重写 OpenVLA-OFT 的 fork**——只做必要 hook patch
- ❌ **不要省略 baseline**——所有 cell 必须有 3-seed 均值（或分层抽样 2-seed）
- ❌ **不要在 P1 期间研究 P3 的 K\* 计算**——按 phase 顺序推进，避免 context switch
- ❌ **不要在 P2 中途切 backbone**——确定用 OpenVLA-OFT 还是 SmolVLA 作主跑要 P1 末就敲定
