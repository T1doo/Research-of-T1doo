# Obsidian × Zotero 配置指南

> 这是**一次性配置指南**。设置完之后日常怎么用 → 看 [[02 日常使用]]
>
> 涉及插件：
> - Obsidian 端：**Zotero Integration**（已安装 ✅）
> - Zotero 端：**Better BibTeX**（已安装 ✅）

---

## 🎯 当前已配置状态（备忘）

如果哪天重装/换电脑，按下面步骤复原即可。

- **Citekey 公式**：`auth.lower + shorttitle(3,3) + year`（例：`liangAdaptiveActionChunking2026`）
- **文献笔记存放**：`References/{{citekey}}.md`
- **图片存放**：`References/images/`
- **模板文件**：`_templates/zotero-literature.md`

## Step 1：Zotero 端配置

### 1.1 开启 HTTP Server（默认就是开的，验证一下）

Zotero → **编辑 → 设置 → 高级 → 常规 → 配置编辑器**（点"我接受风险"）
→ 搜索 `extensions.zotero.httpServer.enabled` → 确认是 `true`
→ 端口默认 `23119`，不用改

> ⚠️ **使用插件时 Zotero 必须保持运行**，否则 Obsidian 这边搜不到东西。

### 1.2 设置 Quick Copy 引用样式（必需，否则 Insert Citation 报错）

Zotero → **编辑 → 设置 → 导出**：
- **默认输出格式**：选一个引用样式
  - 英文常用：`Chicago Manual of Style 17th edition (author-date)` / `APA 7th edition`
  - 中文：`GB/T 7714—2015 (numeric, 中文)`
- 没有想要的样式 → 点"**获取更多样式**"下载

### 1.3 Better BibTeX：Citekey 公式

Zotero → **编辑 → 设置 → Better BibTeX → Citation keys**：

- **Citation key formula**：
  ```
  auth.lower + shorttitle(3,3) + year
  ```
  效果：`liangAdaptiveActionChunking2026`（作者姓 + 标题前3词各取3字母 + 年份）

- 勾选 **On item change → Keep keys**（防止 key 漂移）
- 新增文献后如果 key 没生成，右键文献 → **Better BibTeX → Refresh BibTeX key**

> ⚠️ **不要随便改公式**。Citekey 是 Obsidian 笔记和 Zotero 文献之间的"身份证"，改公式会让旧笔记里的 `[@key]` 全部失效。

## Step 2：Obsidian 端配置

打开 **设置 → 第三方插件 → Zotero Integration**：

### 2.1 General Settings

| 选项 | 值 |
|---|---|
| **PDF Utility** | 点 Install 安装（必需，否则不能导高亮） |
| **Database** | `Zotero`（Zotero 7 也选这个） |
| **Note Import Location** | `References` |
| **Open the created or updated note(s) after import** | 开 ✅ |
| **Which notes to open after import** | `First imported note` |

### 2.2 Citation Format（引用插入格式）

点 **+ Add Citation Format**：

| 字段 | 值 |
|---|---|
| **Name** | `Markdown` |
| **Output Format** | `Pandoc` |
| **Include Brackets** | 开 ✅ |

效果：`Insert Citation` 命令会插入 `[@liangAdaptiveActionChunking2026]`。

### 2.3 Import Format（文献笔记导入格式 ⭐ 最关键）

点 **+ Add Import Format**：

| 字段 | 值 | 说明 |
|---|---|---|
| **Name** | `文献笔记` | 命令面板显示的名字 |
| **Output Path** | `References/{{citekey}}.md` | ⚠️ **必须含文件名和 .md 扩展名**，否则会报 EISDIR 错误 |
| **Image Output Path** | `References/images` | 图片存放**文件夹** |
| **Image Base Name** | `{{citekey}}` | 图片文件名前缀 |
| **Template File** | `_templates/zotero-literature.md` | 选 vault 里的模板文件 |

### 2.4 Image Settings

| 选项 | 值 |
|---|---|
| **Image Format** | `jpg` |
| **Image Quality** | `90` |
| **Image DPI** | `120` |

## Step 3：模板文件

模板已经放在 `_templates/zotero-literature.md`，**不要删**。

如果误删了，可以从这份配置目录里的 [[zotero-literature-模板备份]] 恢复（见同目录下的备份文件）。

模板会渲染：
- Frontmatter（标题/作者/年份/期刊/DOI/citekey/Zotero链接/itemType/tags）
- 元信息 callout 框
- Abstract
- **🧠 我的思考**（含 4 个子区块，被 `{% persist %}` 保护，重导不覆盖）
- **✏️ PDF 高亮与注释**（按页码渲染，含原文/颜色/批注/框选图片）

## 🚨 配置过的坑（避免重蹈覆辙）

### 坑 1：Output Path 填成文件夹而不是文件
**症状**：导入命令运行后没反应，Console 报 `EISDIR: illegal operation on a directory`
**原因**：把 `References` 填进 Output Path 了，应该是 `References/{{citekey}}.md`
**修复**：插件设置里改，或直接改 `.obsidian/plugins/obsidian-zotero-desktop-connector/data.json` 里的 `outputPathTemplate`

### 坑 2：手动改了 data.json 后插件没生效
**症状**：改完配置文件后行为没变
**原因**：Obsidian 运行时不重读 data.json
**修复**：`Ctrl+P` → `Reload app without saving`，或关闭再启用插件

### 坑 3：Citekey 是空的
**症状**：笔记没生成或文件名是 `.md`
**原因**：Better BibTeX 没给该文献生成 key
**修复**：Zotero 里右键文献 → Better BibTeX → Refresh BibTeX key

### 坑 4：PDF 高亮没导入
**原因 A**：高亮是在 Zotero 自带 PDF 阅读器**之外**画的（如 Adobe / WPS）→ 必须用 Zotero 内置阅读器
**原因 B**：PDF Utility 没装 → 设置页面顶部点 Install

### 坑 5：Insert Citation 报错
**原因**：Zotero 没设 Quick Copy 引用样式（Step 1.2）

### 坑 6：persist 块内的 fallback 文字会被重复保留
**症状**：首次没高亮显示"暂无注释"，后来加了高亮重导后"暂无注释"那行还在
**原因**：`{% persist %}` 块会保护块内所有内容，包括 `{% else %}` 渲染出的兜底文字
**修复**：persist 块里**不要写 fallback 文字**，空就空着（模板已按此修正）
