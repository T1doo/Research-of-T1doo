# Zotero × Obsidian 日常使用手册

> 配置怎么设的 → 看 [[01 配置指南]]
> 这份文档讲**配置完之后每天怎么用**。

---

## 🎬 标准工作流（从读文献到写笔记）

```
1. 在 Zotero 里收藏文献（拖 PDF / DOI 抓取 / 网页插件）
        ↓
2. 在 Zotero 内置 PDF 阅读器里读 PDF + 画高亮 + 写批注
        ↓
3. Obsidian 里跑命令导入 → 生成文献笔记（含元数据 + 高亮 + 批注）
        ↓
4. 在笔记的「🧠 我的思考」区写自己的总结与关联
        ↓
5. 写综述/论文时，用 [@citekey] 引用
```

---

## 📥 操作 1：把文献导入成 Obsidian 笔记

### 前提
- ✅ Zotero 已打开
- ✅ 想导入的文献已在 Zotero 库里

### 步骤
1. Obsidian 按 `Ctrl + P` 打开命令面板
2. 输入 `zotero`
3. 选 **Zotero Integration: 文献笔记** → Enter
4. 弹出搜索框 → 输入文献关键词（作者姓 / 标题词）
5. ↑↓ 选中 → Enter
6. 笔记自动生成在 `References/<citekey>.md` 并自动打开

### 生成的笔记长什么样
- **Frontmatter**：标题、作者、年份、期刊、DOI、citekey、Zotero链接、tags
- **元信息 callout**：一眼看清关键信息
- **Abstract**：原文摘要
- **🧠 我的思考**：4 个空区块（核心观点 / 方法论 / T1D 关联 / 待解问题）等你填
- **✏️ PDF 高亮与注释**：按页码自动渲染你在 Zotero 里画的高亮和批注

---

## 🔄 操作 2：重新导入（更新笔记）

读 PDF 时加了新高亮 → 想同步到 Obsidian？

### 步骤
**和首次导入一模一样**：`Ctrl+P` → `Zotero Integration: 文献笔记` → 搜索 → 选中 → Enter。

### 重要：哪些内容会被保留
| 区块 | 重导后 |
|---|---|
| Frontmatter | ⚠️ 完全覆盖（手动加的字段会丢） |
| Abstract | ⚠️ 完全覆盖 |
| **🧠 我的思考**（在 `%% begin my-thoughts %%` 之间） | ✅ **保留** |
| **✏️ PDF 高亮**（在 `%% begin annotations %%` 之间） | ✅ **保留**已有内容 + 追加新高亮 |

> 💡 **想加的自定义内容**，写在 `%% begin ... %%` 和 `%% end ... %%` 之间就不会丢。

## 📝 操作 3：在正文里引用文献（⭐ 日常用这个）

平时记笔记、写综述时引用某篇文献。

### 步骤（最常用，可点击跳转）
1. 编辑模式下，光标放在要插入的位置
2. **英文输入法**下打一个 `@`
3. 弹出文献搜索框 → 输关键词 → ↑↓ 选中 → Enter
4. 插入 `[[liangAdaptiveActionChunking2026]]`
5. 按住 `Ctrl` 点击 → 跳到对应文献笔记

### 两种引用语法的分工 🔑

| 写法 | 怎么生成 | Obsidian 里可点击 | 用途 |
|---|---|---|---|
| `[[citekey]]` | 打 `@` → 选文献 | ✅ | **日常笔记、内部关联** |
| `[@citekey]` | 命令 `Zotero Integration: Markdown` | ❌ | 写论文准备 Pandoc 导出时 |

**结论**：日常 99% 用 `@` 触发的 wikilink 就够了。`Markdown` 命令几乎用不到，等真要导出正式论文时再用。

### 多篇一起引
重复打 `@` 选多次，或手动写：
```markdown
最新研究表明 [[wang2024]]、[[liu2025]]、[[zhao2026]] 均观察到该现象。
```

### 将来要写正式论文怎么办
那时候改用 `[@citekey]` 格式（命令面板里的 `Zotero Integration: Markdown`），用 Pandoc 导出 docx/PDF 时会自动展开：

- markdown 原文：`研究表明 [@liangAdaptive2026]`
- 导出后：`研究表明 (Liang et al., 2026)`，并自动生成参考文献列表

届时再写专门的 Pandoc 导出指南。

### 如果 `@` 没反应
设置 → Zotero Integration → 滚到底部 **Cite Suggest** 区 → 确认 **Enable cite suggest** 开关是开的。

## 🔍 操作 4：在 Obsidian 里找文献

### 方法 A：按 citekey
- 打字 `[[liangAdaptive...` → Obsidian 双链自动补全跳转

### 方法 B：浏览
- 文件树打开 `References/` 文件夹，按时间/字母排序浏览

### 方法 C：搜索（最常用）
- `Ctrl + Shift + F` 全文搜索 → 输入作者/关键词
- 或 `Ctrl + O` 按文件名搜

### 方法 D：通过 tag
- frontmatter 里 `tags: [literature, T1D]` → 点 tag 看所有 T1D 文献

## 🔗 操作 5：从 Obsidian 跳回 Zotero

打开任一文献笔记 → 元信息 callout 框里点 **「在 Zotero 中打开」** 链接
→ 系统自动唤起 Zotero 并定位到那条文献。

适合：想看原 PDF / 想看 Zotero 里的笔记 / 想改文献元数据。

## ✏️ 操作 6：在 Zotero 里画高亮的正确姿势

**必须用 Zotero 自带的 PDF 阅读器**（外部软件画的高亮不能导入）。

### 双击 PDF 打开后
- **高亮文字**：选中文本 → 工具栏选颜色 → 自动保存
- **加批注**：点高亮 → 右侧出现评论框 → 输入
- **框选图片**：工具栏第二个工具（矩形）→ 在 PDF 上拖框
- **加便签**：工具栏便签工具 → 点 PDF 任意位置

### 颜色建议（自己定一套约定）
| 颜色 | 含义 |
|---|---|
| 🟡 黄色 | 核心观点 |
| 🔴 红色 | 存疑 / 反对 |
| 🟢 绿色 | 方法 / 技术 |
| 🔵 蓝色 | 与我研究的关联 |
| 🟣 紫色 | 值得追读的引用 |

模板里 `{{a.colorCategory}}` 会渲染颜色名（Yellow / Red / Green...）。

## 🆘 出问题怎么办

### 命令运行后没反应
1. 检查 Zotero 是否打开
2. `Ctrl+Shift+I` 打开 DevTools → Console 看红色错误
3. 对照 [[01 配置指南]] 里的"配置过的坑"小节

### Citekey 是 `undefined` 或显示不对
→ Zotero 里右键文献 → Better BibTeX → Refresh BibTeX key

### 中文文献作者顺序奇怪
→ 改模板里 `{{authors}}` 的渲染方式（拆 firstName/lastName 自己拼）

### 高亮没导入
→ 确认是 Zotero 内置阅读器画的，不是 Adobe/WPS

## 🚀 命令速查表

### 平时只用这两个 ⭐

| 想干啥 | 怎么触发 | 输出 |
|---|---|---|
| **导入/更新文献笔记** | 命令面板 → `Zotero Integration: 文献笔记` | 新文件 `References/<citekey>.md` |
| **插入可点击引用** | 编辑模式下打 `@` → 选文献 | 当前光标 `[[citekey]]` |

建议给"文献笔记"命令绑快捷键 `Ctrl+Shift+Z`：设置 → 快捷键 → 搜 `文献笔记` → 点 `+`。

### 偶尔/将来才用

| 命令 | 何时用 |
|---|---|
| `Zotero Integration: Markdown` | 写论文准备 Pandoc 导出时，插入 `[@citekey]` |
| `Zotero Integration: Data explorer` | 改模板或排查 bug 时，查 Zotero 返回的原始数据 |
| `Zotero Integration: Import notes` | 把 Zotero 笔记导入为独立 md 文件（基本用不到） |
| `Zotero Integration: Insert notes into current document` | 把 Zotero 笔记内容塞进当前文档（基本用不到） |
| `Reload app without saving` | 手动改了 `data.json` 后让插件重新加载 |
