# §1 导言

## 1.1 立项背景

T1doo 团队最初设想以「**Efficient VLA (EfVLA)**」为旗舰立项，三个技术方向组合：

1. **Adaptive Action Chunking (AAC)** — 推理时根据场景熵动态调整 action chunk 长度
2. **RobustVLA** — 多模态对抗训练 + worst-case δ 优化
3. **SmolVLA backbone** — 0.45 B 参数、46 GB 显存可全参微调

导师审阅后给出关键反馈：**AAC × 鲁棒性的耦合实现难度高、一年期产出风险大**，建议**收窄到鲁棒性单一主线**深耕。同时抛出导航星论文 **FocusVLA**（[arXiv:2603.28740](https://arxiv.org/abs/2603.28740)，2026-03-30，HIT-Shenzhen × DaiMon Robotics × NJU × RUC），指向「**视觉利用率提升 → 隐性鲁棒性**」这条相对工程可控的路线——证据是 0.5 B 参数 FocusVLA 在 LIBERO 多权重上 98.7%，打败 7 B 的 OpenVLA-OFT (98.5%)。

## 1.2 本报告要解答的三个问题

1. **「具身智能鲁棒性」当下的全景如何？** FocusVLA 在其中占据什么位置？
2. **三条主线（视觉利用 / 数据训练 / 补盲）各自的最前沿是什么？** 有哪些方法学差异、可组合点、互相冲突点？
3. **一年期本科生课题，最值得做什么？** 在「不降低难度」的前提下，给出 3-5 个候选课题排序。

# §2 概念地图：什么是「具身智能鲁棒性」

## 2.1 与传统机器学习鲁棒性的差异

具身智能（Embodied AI / VLA）的鲁棒性研究**继承了传统 ML 鲁棒性的工具箱**，但**问题结构有 3 个本质差异**：

| 维度 | 传统 ML 鲁棒性 | 具身智能鲁棒性 |
|---|---|---|
| **失败成本** | 错分一张图（虚拟代价） | 撞坏物体 / 伤人（物理代价） |
| **决策结构** | 单步预测 | 闭环长序列（每步影响下一步状态） |
| **输入模态** | 单一（图像 / 文本） | 多模态（视觉 + 指令 + proprioception + 触觉） |
| **数据分布** | 静态 i.i.d. | 动态 + 部分可观测 + 任务条件化 |
| **评估** | 准确率 / AUC | 任务成功率 + 长程稳定 + 安全约束满足 |

这些差异决定了：**单纯把 CV 的对抗训练搬到 VLA 上是不够的**——一个鲁棒的分类器 + 不鲁棒的决策器 = 仍不鲁棒的系统。

## 2.2 鲁棒性的三层 taxonomy

根据本调研对约 230 篇相关论文的归纳，我们提出三层 taxonomy：

```
                      具身智能鲁棒性
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
   层1: 输入扰动鲁棒        层2: 决策内禀鲁棒     层3: 系统级安全
   ─ Visual perturbation   ─ Action 一致性       ─ Failure detection
   ─ Instruction perturb   ─ Worst-case 优化     ─ Safety filter
   ─ State estimation noise─ Out-of-distribution ─ Replanning / recovery
   ─ Cross-embodiment      ─ Distribution shift  ─ Formal verification
```

- **层 1（输入扰动鲁棒）**：模型对**输入侧**变化的容忍。覆盖视觉扰动（光照、纹理、遮挡）、指令扰动（同义改写、错别字、不可执行指令）、状态扰动（proprioception 噪声）。**FocusVLA 主要在层 1**（隐性方式提升视觉利用率）。
- **层 2（决策内禀鲁棒）**：模型决策本身的**鲁棒性质**——action 平滑、worst-case 行为有界、对未见分布有保守预测。**RobustVLA 主要在层 2**（worst-case δ 显式优化）。
- **层 3（系统级安全）**：当前两层都失效时，系统能否检测、降级、恢复？**RC-NF (References 已读)、Safety Filter、Replanning 在这一层**。

**重要观察**：三层是**正交且可叠加**的。但目前 VLA 鲁棒性主流工作多集中在层 1+2，**层 3 是相对空白的研究空间**。

## 2.3 六个核心扰动维度（来自 VLA-Risk ICLR 2026）

VLA-Risk [[VLA-Risk_ICLR2026]] 提出的 6 维分类体系是目前最系统的评估框架：

```
              Vision (V)              Instruction (I)
Object        V_obj                    I_obj
              (Object Mislabeling)    (Attribute Contradiction)
Action        V_act                    I_act
              (Action Contradiction)  (Action Impossibility)
Space         V_spa                    I_spa
              (Space Distraction)     (Space Inconsistency)
```

**关键实证**：指令侧攻击平均 ASR **63.99%**，视觉侧 38.91%——这说明**当前 VLA 的语言-视觉对齐机制更易被语言侧击穿**，对 FocusVLA 主线是个警告（视觉聚焦改善不了指令侧脆弱）。

## 2.4 鲁棒性 ≠ 数据增广

一个**广泛误区**：「加更多数据增广就能解决鲁棒性」。RobustVLA [[2026_RobustVLA_Guo]] 的实证彻底打破这个假设：
- BYOVLA 在高斯噪声 +7.3%、死像素 +22.3%
- 但在 action / instruction / environment 模态改进 **0.0%**

**意味着**：单模态视觉增广是一条 dead-end road。真正有效的鲁棒性提升需要：
- **多模态联合优化**（RobustVLA worst-case δ）
- **架构层面的视觉利用提升**（FocusVLA Cascaded Attention）
- **运行时的失败检测与恢复**（RC-NF + Replanning）

这三条路径分别对应了我们的 **主线 A、B、C**。

# §3 评估基础设施：benchmark 全景

## 3.1 benchmark 演化时间线

```
2018-2020: MetaWorld(50 tasks) → RLBench(100) → D4RL → RoboMimic
2021-2022: CALVIN(语言条件 long-horizon)
2023: LIBERO(Original, 4 suites × 130 tasks) → CompoSuite
2024: BEHAVIOR-1K → SimplerEnv → VLABench → Colosseum(14-axis perturb) 
      → Open X-Embodiment(1M+ traj)
2025: LIBERO-Plus(7D perturb) → LIBERO-PRO(generalization) → RoboTwin 2.0(5-embod)
      → RoboCasa365 → ManiSkill-ViTac → RoboChallenge(real-robot online)
2026: LIBERO-X(hierarchical) → LIBERO-Para(paraphrase robustness)
      → VLA-Arena(170 tasks × L0-L2) → VLA-Risk(6维扰动 ICLR 2026)
```

## 3.3 曦源 TOP-3 推荐 benchmark

基于「**评测维度全面性 × 复现成本 × baseline 丰富度 × 与立项相关性**」四维评分：

### TOP-1：LIBERO-Plus（必选）
- **理由**：7 维扰动覆盖最全（layout/viewpoint/state/instruction/light/texture/noise），10K perturbed episodes 规模适中；现有 baseline（OpenVLA, π0, Octo 等）全部已在原 LIBERO 上跑过，迁移成本低；46 GB 单卡 4-6 小时可跑完。
- **用途**：核心鲁棒性评测平台

### TOP-2：VLA-Arena（强烈推荐）
- **理由**：170 任务 × L0-L2 难度分级，V0-V4 / W0-W4 视觉/语言扰动级别正交可控，**特别适合做消融实验**（不同模型在哪个扰动级别开始崩塌）；任务量适中可快速迭代。
- **用途**：快速消融与课题方案对比

### TOP-3：VLA-Risk（实验后期必须）
- **理由**：作为 ICLR 2026 的最新 benchmark，是「最权威 + 最难」的鲁棒性试金石；6 维扰动结构清晰；包含自动驾驶域（OpenEMMA），可拓展报告价值。
- **风险**：需等代码 release；扰动生成依赖 GPT-5（成本可能较高）；可作为最终评测而非迭代评测。

### 备选：SimplerEnv（用于 sim-to-real 桥接）+ RoboChallenge（如有真机预算）

## 3.4 「评估单一 benchmark 就够吗？」—— 一个常见的研究陷阱

LIBERO-PRO 的关键实证 [[2025-10_LIBERO-PRO]]：**主流 VLA 在原 LIBERO 上 90% 成功率，在 LIBERO-PRO 上掉到 0%**。

这意味着：**单一 benchmark 上的高分本质上是「记忆」而非「鲁棒泛化」**。曦源课题立项时必须在**至少 3 个不同 benchmark** 上交叉验证（推荐 LIBERO-Plus + VLA-Arena + VLA-Risk 组合）。

# §4 主线 A · 视觉利用鲁棒性

> 锚点：[[2026-03_FocusVLA_Zhang]]（HIT 深圳 + DaiMon Robotics + NJU + RUC，2026-03-30）。覆盖 A1-A7 共 28 篇精读笔记。

## 4.1 主线定位与基础假设

主线 A 的核心假设：**改进 VLA 对视觉信息的利用机制，而非增加视觉编码器质量本身，是获得隐性鲁棒性的最优工程路径**。

这是 FocusVLA 的反主流观察：尽管已经有 OpenVLA（DINOv2+SigLIP）、SmolVLA（SmolVLM-2）等强 encoder backbone，VLA 在精细操作上仍然脆弱。FocusVLA 把症结定位在三个**架构层面**的瓶颈：

1. **Mixed Attention 的 shortcut**：VLA-Adapter 等主流架构把视觉、动作 query、action latent 拼接后做单次 attention，让 action query 偷偷绕过视觉细节；
2. **Visual Token 过载**：512+ token 的视觉输入稀释 attention 焦点；
3. **任务无关噪声**：背景 / 干扰物 / 纹理变化以 spatial bias 形式污染 action 生成。

FocusVLA 用 **Modality Cascaded Attention**（顺序独立 attention 替代并行混合）+ **Focus Attention**（patch-level top-K 选 + channel-level element-wise gate）解决这三个问题，实现 0.5 B 参数在 LIBERO 多权重 98.7%、单权重 97.0%——**打败了 7 B 参数的 Spatial Forcing 与 OpenVLA-OFT**。这是「小模型 + 巧架构 > 大力出奇迹」的旗舰证据。

## 4.2 七个子主题的演化趋势

### A1 Visual Token Pruning：三条分化路线 + 共同空白
- **路线 1 · Dual-Objective**：VLA-Pruner (2511.16449) 同时优化语义对齐 + 动作相关性，1.99× 加速
- **路线 2 · Task-Phase Aware**：Compressor-VLA (2511.18950) 用指令条件化 + 时序感知动态调度
- **路线 3 · RL-Learned**：TPRL (2603.13394) 把 pruning 形式化为 MDP，66.7% token / 54.2% FLOPs 削减
- **训练-free 极简路线**：VLA-IAP (2603.22991) 用几何先验完全免训练
- **极致压缩**：LUVC (2512.09010) 渐进 + 谱剪枝，号称 lossless 2× 加速

**关键空白**：上述工作**全部没有在 LIBERO-Plus / VLA-Risk 等鲁棒性 benchmark 上系统评测**——剪枝对鲁棒性的影响是个空白课题。这是 FocusVLA 路线可以差异化切入的方向：**「剪枝得多 ≠ 鲁棒性差，但激进剪枝可能让 attention 更聚焦反而提升鲁棒？」** 是个值得实证的研究假设。

### A2 Attention Guidance / Saliency 注入：oracle 信号源加速降本
2024-2026 的演化轨迹：
- **2024 之前**：人工 saliency（成本高、不可扩展）
- **2025**：人眼 Gaze 数据（Gaze-Regularized VLA 2603.23202 用 KL gaze-heatmap regularization，4-12% 提升、无推理开销）
- **2025-11**：VLM saliency（AutoFocus-IL 2511.18617，Gong et al. claim **VLM saliency > 人类 gaze**）
- **2026**：DINOv2 自监督 attention（Attention-Guided Voxel 2509.20579，+8.2% RLBench bimanual）

趋势：**oracle 信号越来越便宜 → 接近免费**。AutoFocus-IL 的 "VLM > 人眼" 强 claim 若复现成立，将是 attention supervision 的范式级利好（彻底解放标注）。

### A3 Task-Relevant Patch Selection：2D → 3D → 4D
最重要的子主题——与 FocusVLA 同代：
- **2D 经典基础**：Perceiver IO (2107.14795)、TraceVLA (2412.10345 用 visual trace prompting，**4 B 打败 7 B**)
- **3D 表示**：SpatialVLA (2501.15830 Ego3D PE)、SG-VLA (2603.22760 spatial grounding 6-DoF)
- **4D 时空**：ST-VLA (2603.13788 4D spatiotemporal mask，44.6% / 30.3% zero-shot 提升)
- **同代架构改进**：EF-VLA (CLIP 早融合)、GuidedVLA (2605.12369 plug-and-play attention head 专化)

ST-VLA 的 44.6% / 30.3% zero-shot 提升是**惊人数字**，暗示 4D 表示可能 dominate 2D 路线。TraceVLA 是反方向的优雅解——用 visual prompting 让 2D VLM 获得时空感知，**3.5× 真机提升**。

### A4 Visual Grounding for Manipulation：从研究问题到上游工具
2023-2026 的角色转变：
- **2023 显式符号**：VoxPoser (2307.05973 3D value maps + LLM)、ReKep (2409.01652 关键点 + LLM 约束)
- **2024 混合监督**：SG-VLA (auxiliary head)、Hierarchical Lang-Action Alignment (2604.05614 contrastive)
- **2024-2026 工具基础设施**：**Grounded SAM** (2401.14159) 已成为标准工具，open-vocab grasp 全套工具链就绪

**关键观察**：Foundation model 把 grounding 从"研究问题"变成"上游已解决工具"。曦源课题不必再做 grounding，可以**消费它作为基础设施**——这降低了立项的实施风险。

### A5 多视角 / 多分辨率融合：2026 大趋势 — 视频化策略
2026 年的代表工作：
- **MV-VDP** (2604.03181 multi-view video diffusion policy)
- **Action Images** (2604.06168 把 action 像素化为多视角 image)
- **GP3** (2509.15733 多视角 RGB → dense 3D，不依赖深度)
- **VIPA-VLA** (2512.13080 visual-physical alignment from human videos)

**核心趋势**：action 表示从「关节空间」转为「视频/heatmap/像素轨迹」，让 video backbone 直接做 policy。这是与传统 VLA 范式的重大差异点。

### A6 Shortcut Learning / Spurious Correlation 诊断：三段演化
最具理论价值的子方向：
- **诊断阶段**（2024-2025）：[[2025-08_ShortcutLearning_Xing]] (CoRL 2025, 2508.06426) 系统识别 VLA dataset fragmentation 与 spurious roots，**这是给"为什么需要数据增广"提供理论依据的关键论文**
- **轻量抑制**（2025）：InSpire (2505.13888) 用 spatial reasoning prompt，+6.2% seen / +10% unseen
- **自反思**（2025-2026）：CF-VLA (2512.24426) 自驾场景的 counterfactual reasoning；CAST 用 VLM-queried 反事实指令做轨迹增广

**核心叙事候选**：**「诊断（A6）—架构（A FocusVLA）—数据（B RobustVLA）」三方组合** 是 BC 申请书 / CoRL 论文的理想叙事支点。

### A7 Encoder 选择：2025-2026 共识与隐忧
- **共识**：SigLIP + DINOv2 双流已成标配（OpenVLA、Cobra、FocusVLA 都用）
- **轻量化代表**：SmolVLA (SmolVLM-2 单流)、SemanticVLA (2511.10518 encoder-agnostic pruning，97.7% LIBERO)
- **基础重提**：DINOBot (2402.13181) 提醒「foundation feature 自带 grounding」，**质疑端到端 policy 学习的必要性**

潜在课题：**Encoder 选择对鲁棒性的影响系统对比**（DINOv2 vs SigLIP vs CLIP），目前还没有专门工作。

## 4.3 FocusVLA 自己留下的研究空白

FocusVLA 论文显式列出 5 个限制——**每一个都能成为曦源课题方向**：

| 限制 | 课题机会 | 估计难度 |
|---|---|---|
| 「鲁棒性对背景+纹理同时变化仍有限」 | 多扰动叠加鲁棒训练 | 中 |
| 「对初始状态敏感」 | Initial state randomization training | 低 |
| 「VLM 内部的视觉利用没涉及」 | LoRA 改造 VLM 内部 attention | 高 |
| 「视觉表示需要大量数据 / 模型规模」 | Few-shot 视觉鲁棒微调 | 中 |
| 「未在 robustness benchmark 上系统评测」 | **VLA-Risk + LIBERO-Plus 鲁棒性评测系统**（方案 1 核心） | 低 |

## 4.4 主线 A 内部的方法学冲突

并非所有 A 子方向都互相兼容：
- **A1 token pruning** 与 **A3 patch selection** 都减少视觉 token，但目标不同（前者是效率，后者是聚焦）——能否同时叠加是开放问题
- **A2 attention guidance**（外注入 saliency）与 **A6 shortcut diagnosis**（内诊断捷径）逻辑相反——前者教模型「该看哪」，后者让模型「自查为啥不该看哪」；两者结合可能 yield "guided + diagnosed" 双层 attention
- **A4 grounding** 与 **A7 encoder 选择**——是用工具（Grounded SAM）还是用 encoder 内置（DINOv2）？前者灵活但延迟高，后者集成但不可控

## 4.5 与主线 B/C 的接口

**A × B 组合候选**：
- FocusVLA（架构）+ RobustVLA（训练损失）= 双重防御（方案 1+2 候选）
- FocusVLA + RoboAug（B2 region-contrastive 数据增广）+ RobustVLA = 三模块组合（**方案候选**）

**A × C-iii 组合候选**：
- FocusVLA + RC-NF + RaC = 「鲁棒+检测+恢复」闭环（方案 2 核心）

**A × C-ix 组合候选**：
- FocusVLA backbone + Test-Time Adaptation = 训练时架构 + 推理时学习（方案 3 核心）

## 4.6 推荐曦源团队第一个 4-6 周实验

基于本主线 28 篇笔记的综合判断，**InSpire (2505.13888)** 是 ROI 最高的"warmup 实验"：
- Plugin 形式，**零额外训练数据**
- 可叠加到 FocusVLA / OpenVLA 上验证「显式 grounding 是否互补隐式 attention」
- 4-6 周内即可拿到 ablation 结果，作为第一个可发表（workshop 级）的工作
- 同时给团队建立工作流（环境配置、benchmark 跑通、ablation 流程）

# §5 主线 B · 数据 & 训练侧鲁棒性

> 锚点：[[2026_RobustVLA_Guo]]（References 已读）。覆盖 B1-B7 共 25+ 篇精读笔记。

## 5.1 主线定位

主线 B 的核心方法学：**通过数据生成 / 增广 / 对抗训练 / worst-case 优化获得显式鲁棒性**。所有干预都发生在「训练时」，与主线 A 的「架构时」、主线 C 的「推理时」形成完整的鲁棒性策略空间。

## 5.2 RobustVLA 的三个划时代发现（References 已读笔记摘要）

1. **Action 是最脆弱模态**（不是视觉！）—— π0 在 0.05 幅度 action 噪声下 SR 从 96% 跌到 52.4%，0.1 幅度下完全归零
2. **视觉鲁棒不迁移**：BYOVLA 在 action / instruction / environment 模态改进 **0.0%**
3. **多模态 worst-case 优化几乎零代价**：clean 95.5% vs 96.0%

这三个发现重塑了主线 B 的研究范式——必须做**多模态联合**鲁棒训练。

## 5.3 七个子主题的演化趋势

### B1 多模态对抗训练：三脉络
- **路线 a 攻击新维度扩展**：EDPA 嵌入空间攻击 (2024-11) → DPA 多步策略攻击 (2024-12) → **VLA-Fool** (2025-11) **cross-modal amplification 1.3× 直接打 RobustVLA 独立扰动假设**
- **路线 b 评估范式统一化**：Eva-VLA (2025-09) 提供连续物理参数搜索
- **路线 c 防御从训练时走向推理时**：Model-Agnostic AdvDef (2025-10) diffusion purification

**关键发现**：**VLA-Fool 揭露 RobustVLA 多臂 bandit 假设的盲点**——扰动并非独立，cross-modal 攻击有放大效应。**这是曦源差异化创新的 catchy 故事**：「扩展 RobustVLA 到 joint cross-modal worst-case」。

### B2 视觉数据增广：annotation 效率竞争
代表演化线：ROSIE (2023-02 开山) → GreenAug (2024-07) → BYOVLA (2024-10) → RoboEngine (2025-03 SAM2+SD) → RESample (2025-10 OOD 状态) → **RoboVIP** (2026-01 多视角) → **RoboAug** (2026-02 region-contrastive，per-task 1 标注)

**核心趋势**：从「换背景」→「换物体」→「加干扰物」→「换视角」→「换状态」；标注成本竞争主导。

**最佳枢纽**：**RoboAug 可作为 RobustVLA input-robust 模块的 ω*(o) 生成器**——离线增广 + 训练时对抗一致性损失是天然组合。

### B3 物理域随机化
- **RoboCat** (2023-06) 视觉 DR + zero-shot 迁移
- **DR Survey 2024** (ICLR) 流动 DR、动态合规性
- 趋势：从静态 DR 走向动态 / 自适应 DR

### B4 合成数据 / Sim-to-Real（B 主线最重要子方向）
代表工作：
- **2023 LLM 生成式**：RoboGen (2311.01455)、Gen2Sim (2310.18308)
- **2024 物理统一仿真**：Genesis（统一刚体/软体/流体），81× 加速；UniSim (ICLR 2024) 学 real-world simulator
- **2024 LLM 自动环境**：Holodeck (CVPR 2024) LLM + 游戏资源
- **2024 数据生成 pipeline**：MimicGen (2310.17596) SE(3) 10→200 demo；DexMimicGen 双臂
- **2024-2025 大规模仿真**：RoboCasa（厨房 65 atomic tasks）、Habitat 3.0

**轻量级可访问性梯度**（46 GB 显存约束）：
- ✅ 单卡可用：MimicGen (30 GB), RoboCasa (40 GB), GreenAug, BYOVLA, RoboEngine, Holodeck (+ API)
- ⚠️ 简化版可行：Gen2Sim（跳过 Zero-1-to-3）、Genesis（生态稳定性需 6-12 月验证）
- ❌ 不可复现：UniSim（100×A100 训练）、RoboCat（TPU pod）

**B3+B4 agent 首选推荐组合**：**MimicGen（数据合成基石）+ RoboCasa（场景闭环）**，4-8 周可产出完整 pipeline；第 1 月跑通 MimicGen 后解锁 DexMimicGen / DYNAMIMICGEN 全家桶。

**B4 子方向最大研究空白**：12 篇 pipeline **全部只生成 clean instruction**，**「Adversarial RoboGen / Adversarial MimicGen」是巨大空白**（用 LLM 生成对抗指令配合合成场景生成）——可成为方案 4 或新方案 6 的核心创新。

### B5 噪声 demo 鲁棒训练
- **Robust IL from Noisy Demos** (2010.10181) 对称损失 + 伪标记
- **Robust IL Env Dynamics** (2206.09314) DRO 环境变化

子方向相对冷清——LLM 时代对 noisy demo 的处理还欠缺新方法。

### B6 Worst-case / DRO
- **DRO Library** (2505.23565) Stanford 工程库，**可一行代码替换 RobustVLA 多臂 bandit 为理论保证的 DRO**
- **ReMix DRO** (2408.14037) DRO 数据混合
- **DRO Theory Survey** (2411.02549) 理论奠基

**关键工程结论**：DRO Library 是最快落地的改进——可立即装上替换 RobustVLA 的启发式调度。

### B7 Cross-Embodiment Generalization（B 主线最热子方向）
时间线（共识演化）：

**HPT 最 elegant**：模块化「shared trunk + per-embodiment tokenizer」，可借鉴用于 LeRobot 多机体扩展。

## 5.4 RobustVLA 留下的研究空白

主线 B 的最大锚点 RobustVLA 留下的空白：

## 6.2 C-ix Test-Time Adaptation（首推 · 方案 3 核心）

### 代表工作时间线
- **2025-01 RoboMonkey**：大规模测试时采样验证，3.85% relative SR 增益 across 12 OOD
- **2025-06 VLA-RL**：TT sampling + verifier
- **2026-01 TT-VLA** (2601.06748)：**首个端到端 RL adaptation 框架**

### 核心方法
$$\mathcal{L}_{TT} = \mathbb{E}_{t} \left[ -r_t^{\text{proxy}}(o_t, a_t) + \beta \cdot D_{KL}(\pi_\theta \| \pi_{\theta_0}) \right]$$

其中 $r_t^{\text{proxy}}$ 是任务进度信号，$\beta$ 是 KL 正则化项防止策略漂移。

### 与曦源对接
| 接口 | 机制 |
|---|---|
| C-ix × FocusVLA | TT-VLA 在 FocusVLA backbone 上做在线适应 |
| C-ix × RobustVLA | 静态鲁棒训练 + 动态适应 = 完整双侧 |
| C-ix × RC-NF | RC-NF 异常分数作为 TT-VLA 的辅助 reward signal |
| C-ix × VLA-Risk | 6 维扰动是 TT-VLA 的完美评测场 |

### 核心研究空白（C agent 识别）
> **「VLM Verifier 替代 TT-VLA 手工 reward + RC-NF safety filter 防止 TT 学坏 + 在 VLA-Risk 6 维扰动上系统评测在线适应能否补救训练分布外」**

# §7 横向对比与组合空间

## 7.1 三主线的方法学定位

| 主线 | 干预层 | 干预时刻 | 计算开销 | 代表 | 与 baseline 的兼容性 |
|---|---|---|---|---|---|
| **A 视觉利用** | 架构 | 训练时定型 | 训练 + 推理减小 | FocusVLA | 需重训（架构修改） |
| **B 数据训练** | 损失 / 数据 | 训练时 | 训练 ↑ 推理 = baseline | RobustVLA | 重训但保留架构 |
| **C-iii 失败恢复** | 系统 | 推理时反应 | 推理 + 监控开销 | RC-NF + RaC | 即插即用 / 不改 backbone |
| **C-ix Test-Time Adapt** | 策略 | 推理时学习 | 推理 ↑（采样+更新） | TT-VLA | 不重训 backbone |
| **C-x 指令鲁棒** | 数据 / 训练 | 训练时 | 训练 ↑ | LIBERO-Para + adv | 重训 |

## 7.2 三主线如何叠加：四种组合空间

### 组合 1 · 「架构 + 训练」双重防御（A × B）
**方案**：FocusVLA Cascaded Attention + RobustVLA worst-case δ 训练
**机制**：架构让模型必须看视觉，训练让模型对扰动 δ 鲁棒；两者干预不同环节，理论上正交可叠加。
**已知证据**：FocusVLA 在 LIBERO clean 98.7%，RobustVLA 在 LIBERO+17 扰动平均 +12.6%；两者 baseline 不同但都基于 transformer，可融合训练。
**风险**：FocusVLA 的 cascaded attention 改变了 attention 分布，可能影响 RobustVLA 的 worst-case δ 计算（δ 是基于原 attention 的 saliency）。

### 组合 2 · 「训练 + 推理」双侧鲁棒（B × C-ix）
**方案**：RobustVLA 训练 + TT-VLA 测试时适应
**机制**：训练时把已知扰动 family 学进策略；推理时遇到未见扰动用 TT-VLA reward proxy 微调。
**优势**：覆盖「分布内 + 分布外」完整谱系，理论上最完整的鲁棒系统。
**新颖性**：目前尚无工作系统结合这两者，是**首推方案 3 的核心创新点**。

### 组合 3 · 「检测 + 恢复」闭环（C-iii × A/B）
**方案**：RC-NF（检测）+ Replanning（C-iii 恢复策略）挂在 FocusVLA/RobustVLA backbone
**机制**：主推策略尽力鲁棒，failure detection 兜底，replanning 主动恢复。
**优势**：层 3（系统级安全）的完整体系；与导师推的 FocusVLA 不冲突。
**复杂度**：实现门槛高（需协调检测灵敏度 + 恢复策略）。这是**方案 2 的核心**。

### 组合 4 · 「视觉 + 指令」双模态防御（A × C-x）
**方案**：FocusVLA（视觉聚焦）+ Instruction-Robust 微调（C-x）
**机制**：视觉侧用架构改进，指令侧用 paraphrase invariance + infeasibility detection 训练。
**驱动证据**：VLA-Risk 实证指令侧 ASR 高于视觉侧（63.99% vs 38.91%），单边防御不够。
**新颖性**：是**方案 4 的核心**——把 VLA 鲁棒性从视觉拓展到指令。

## 7.3 「最优组合」不一定是最好课题

理论上「A × B × C-iii × C-ix × C-x」全部叠加是最强鲁棒系统，但**一年期本科生课题不该追求这种"大锅烩"**：
- 工作量爆炸
- 创新点稀释（很难分清每个组件的贡献）
- 投稿时讲不清核心 contribution

**最佳策略**：选 2 主线深度耦合（如方案 3 = B × C-ix），第三个作为消融对照。

## 7.4 轻量 backbone（SmolVLA）下的可行组合

46 GB 显存 + 0.45 B 参数约束下，**以下组合工程可行**：
1. ✅ SmolVLA + Cascaded Attention（直接改 action expert，参数增量 < 50M）
2. ✅ SmolVLA + RobustVLA worst-case δ（训练时 +50% 显存，仍在 46 GB 内 batch=4）
3. ✅ SmolVLA + RC-NF（plug-and-play，推理时 +80 ms 延迟）
4. ✅ SmolVLA + TT-VLA（推理时 +N 采样开销，可调）
5. ⚠️ SmolVLA + 4 主线全叠加（训练显存可能溢出，需 LoRA 替代全参微调）

## 7.5 时间窗内组合方案的可行性矩阵

| 组合 | 一年期可达性 | Novelty | 推荐课题 |
|---|---|---|---|
| A only (FocusVLA 评测) | ✅✅✅ | ⭐⭐ | 方案 1（保守） |
| C-iii + A/B (Recovery) | ✅✅ | ⭐⭐⭐⭐ | 方案 2 |
| C-ix only (TT-VLA) | ✅✅✅ | ⭐⭐⭐⭐ | **方案 3（首推）** |
| C-x + A (Instruction) | ✅✅ | ⭐⭐⭐ | 方案 4 |
| A6 + B6 (Shortcut Diagnosis) | ✅ | ⭐⭐⭐⭐⭐ | 方案 5（高赌注） |
| A × B × C 全叠加 | ❌ | — | 不推荐 |

# §9 风险与不确定性

## 9.1 一年期最大风险（已识别 5 个）

### R1 · FocusVLA 代码未释 → 复现成本
**风险**：导航星论文目前 abstract 只承诺 "code will be public"，未给时间表。如果 6-8 月仍未释代码，团队需自己从 VLA-Adapter codebase 重写 Cascaded Attention + Focus Attention（预估 2-4 周纯工程时间）。
**缓解**：
- 立项后第 1 周向作者邮件确认 release 时间
- 平行启动：基于 OpenVLA-OFT codebase 实现 baseline，FocusVLA 模块作为 "drop-in" attention 替换
- Plan B：如果到 9 月仍无代码，改用更易复现的 SemanticVLA / Cobra 作锚点

### R2 · LIBERO 仿真 vs 真机 gap
**风险**：LIBERO-PRO 实证显示主流 VLA 仿真 90% → 真实 0%。如果只在仿真做实验，论文可能被质疑「sim-to-real 没有保证」。
**缓解**：
- 在 SimplerEnv（real-to-sim visual matching）上做 secondary 评测
- 申请实验室真机时间（若团队有 UR5/Franka 可用）
- 在 RoboChallenge 提交（real-robot online platform）

### R3 · 计算资源争用
**风险**：46 GB 显存假设是单卡 A100/H100 80G 半卡。如果实验室卡数紧张，多人共享会拖慢迭代。
**缓解**：
- 优先用 LoRA（rank 8-16）而非全参微调
- 用 SmolVLA-级 backbone（0.45B 全参微调只需 24 GB）
- 异步推理（SmolVLA async stack）可让多模型共享 GPU

### R4 · 方案 3（TT-VLA）的奖励代理设计
**风险**：TT-VLA 的核心 secret sauce 是奖励代理设计（task progress estimation）。如果设计不当，测试时适应反而损害性能。
**缓解**：
- 先复现 TT-VLA 原论文，确认 baseline 行为
- 用 RC-NF 的异常分数作为辅助奖励信号（与 References RC-NF 笔记衔接）
- 设计多种 reward proxy 做消融

### R5 · 论文投稿时间窗
**风险**：CoRL 2027 ddl 大约在 2027-06（一年期末尾），如果实验半路出问题没足够时间写作。
**缓解**：
- 投稿时间表反向规划：12 月有完整 baseline，3 月有主结果，4-5 月写作
- workshop 投稿（CoRL workshop / ICRA workshop）作为备份
- 早期发 arXiv 占坑

## 9.2 不确定性来源（无法完全消除）

1. **新论文涌现**：VLA 领域每月新论文 50+ 篇。综述写完后 3-6 个月可能就有更强 baseline，需持续追读。
2. **GPT-5 API 成本变动**：方案 4（Instruction-Robust）依赖 LLM 生成扰动指令，API 价格上涨会影响实验规模。
3. **导师方向调整**：导师反馈是动态的，方案 3 进展不利时可能再被建议方向收窄。

## 9.3 备选 Plan B

如果方案 3（首推）进展受阻：
- **降级到方案 1**：FocusVLA 评测系统化（保守但稳产）
- **横切到方案 4**：Instruction-Robust（VLA-Risk 实证驱动，工作量适中）
- **延伸到方案 2**：复用 RC-NF 基础做 Recovery-Aware VLA（与团队已有积累衔接）

# §10 附录

## 10.1 baseline 模型推荐表（基于 Classic agent 综合）

| 优先级 | 模型 | 参数 | 显存 | 选择理由 |
|---|---|---|---|---|
| **#1 必选** | OpenVLA-7B | 7B | bf16 14GB / LoRA 24GB | 社区共识 baseline，所有鲁棒性论文都对比它 |
| **#2 强推** | OpenVLA-FAST | 7B | 同上 | 与 OpenVLA 对照验证「离散化是否是鲁棒性瓶颈」 |
| **#3 推荐** | Octo-93M | 93M | 几乎无压力 | diffusion action 路线代表，学术血统纯正 |
| **#4 推荐** | ECoT (OpenVLA+CoT) | 7B | 同 OpenVLA | reasoning-enhanced VLA 对照 |
| **#5 备选** | NORA-3B | 3B | 完全够用 | 2025 轻量化趋势代表，多个 ablation 可同时跑 |
| **轻量首选** | SmolVLA | 0.45B | LoRA 单卡 24GB | 曦源 EfVLA 项目天然起点 |

## 10.2 评测 benchmark 推荐表（基于 Bench agent 综合）

| 用途 | 推荐 benchmark | 理由 |
|---|---|---|
| 主测试场 | **LIBERO-Plus** (2510.13626) | 7 维扰动 + 10K perturbed + baseline 最丰富 |
| 快速消融 | **VLA-Arena** (2512.22539) | 170 任务 × L0-L2，扰动级别正交可控 |
| 最终评测 | **VLA-Risk** (ICLR 2026, 31EjDFwFEe) | 6 维最权威 + 含自动驾驶域 |
| 真机门面 | **RoboChallenge** (2510.17950) | 远程 API，本科生无需硬件 |
| 语言鲁棒（备选） | **LIBERO-Para** (2603.28301) | paraphrase 专测，22-52pp 退化 |
| 桥接验证 | **SimplerEnv** (2405.05941) | real-to-sim visual matching |

**不推荐**：LIBERO-PRO（与 Plus 重叠 + 老 PyTorch）、Colosseum（baseline 落伍）、RoboTwin 2.0（双臂与单臂主线不对接）
