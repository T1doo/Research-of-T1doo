# GP3 — 3D Geometry-Aware Policy 精读笔记

> [!info] 元信息
> - **作者**：Quanhao Qian et al.（Alibaba / Damo Academy）
> - **机构**：Alibaba DAMO Academy
> - **日期**：2025-09-19 (arXiv)
> - **arXiv**：[2509.15733](https://arxiv.org/abs/2509.15733)
> - **定位**：A5 多视角融合 — *无深度传感器* 的 RGB-only 3D 几何感知 policy

## 📄 TL;DR

GP3 解决 "manipulation 需要 3D 几何但深度传感器不可靠" 的问题。核心：**多视角 RGB → spatial encoder → dense 3D features**（隐式深度 + 相机参数）→ 紧凑 3D 场景表示 → 与语言融合 → lightweight policy head → 连续 action。整套系统**不需要深度传感器、不需要预建图、不需要标定**。Sim-to-real 仅需少量微调。本质是「**纯 RGB 学 3D 几何 → 几何特征喂给 manipulation policy**」的端到端方案。

## 🧠 我的思考

%% begin my-thoughts %%

### 核心观点

1. **「RGB 学到的 3D 比深度相机给的 3D 更鲁棒」** — 这个反直觉论断如果成立，对 VLA 工程意义巨大。深度传感器对光照 / 透明物体 / 反光物体常失效；RGB-only 3D 反而更稳。
2. **「3D 几何特征 = 视觉鲁棒性的天然先验」**：3D 表示与相机视角无关，对视角变化天然鲁棒；这是 2D RGB 模型做不到的。
3. **与 VGGT / DUSt3R 同代工作**：spatial encoder 这一层应该用了类似 dense 3D vision foundation model 思路，可能用 VGGT / DUSt3R 类预训练。论文是 "应用层" 工程组合。

### 方法论

- **多视角 RGB 输入**：N 个视角 → spatial encoder（推断 depth + camera intrinsic）
- **3D scene representation**：dense 几何特征，可能是 3D feature field / voxel grid
- **语言融合**：3D feature × text embedding
- **policy head**：lightweight MLP → 连续 action
- **训练**：仿真预训练 + 少量真机 fine-tune

### 与曦源关联（FocusVLA / RobustVLA）

- **与 FocusVLA 方法学差异**：FocusVLA 在 2D image patch 上做 attention；GP3 在 3D feature field 上做 grounding。如果 3D field 已隐含 grounding（不同物体在 field 中分离），FocusVLA 的 patch attention 收益可能变小。**但反过来**：3D field 上做 attention 是个新研究空间。
- **与 RobustVLA 组合**：扰动训练时，*物理场景未变 → 3D field 应不变*；可设计 3D field consistency loss，作为比 2D 像素 consistency 更强的约束。
- **暴露 FocusVLA 未涉及的子问题**：FocusVLA 没回答"2D vs 3D 表示哪个更鲁棒"。GP3 直接挑战 2D-only VLA 的天花板。**这是个值得做的对照实验**：同样 backbone，2D patch vs 3D feature，看 OOD 鲁棒性差距。
- **本科生子模块可行性**：⭐⭐ — 3D spatial encoder 依赖 VGGT 类预训练模型，复现门槛较高；但**评估 GP3 在 LIBERO-Plus 上的鲁棒性**是可行子任务（5-6 周）。

### 待解问题

1. **3D 表示的 sim2real gap**：仿真 3D 几何与真机 3D 几何分布差异（材质 / 光照）如何处理？
2. **多视角依赖**：单视角下退化吗？大部分真机部署只有 1-2 路相机。
3. **算力开销**：spatial encoder 比 2D encoder 重，推理速度？
