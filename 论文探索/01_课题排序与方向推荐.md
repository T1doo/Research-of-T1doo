## 方案 1 · FocusVLA 鲁棒性显式评测与改进（保守 Plan B）

### 问题陈述
导航星论文 FocusVLA 自身**完全没有在 robustness benchmark 上系统评测**（论文只报 LIBERO clean 98.7%）。其作者明确列出 5 个局限：
1. 「鲁棒性对背景 + 纹理同时变化仍有限」
2. 「对初始状态敏感」
3. 「VLM 内部的视觉利用没涉及」
4. 「视觉表示需要大量数据 / 模型规模」
5. **「未在 robustness benchmark 上系统评测」** ← 本方案核心切入点

### 研究假设
- H1：FocusVLA 的 Cascaded Attention + Focus Attention 在视觉扰动（LIBERO-Plus）上显著优于 OpenVLA / VLA-Adapter baseline
- H2：FocusVLA 在指令扰动（VLA-Risk I_obj/I_act/I_spa）上**没有**显著优势——验证「视觉聚焦保护不了指令侧」
- H3：在 FocusVLA 基础上叠加 RobustVLA worst-case δ 训练可进一步提升 +5pp 平均成功率

### 与现有工作差异
- **FocusVLA 没做**：robustness benchmark 评测
- **RobustVLA 没做**：在 FocusVLA backbone 上验证
- **VLA-Risk 没评测**：FocusVLA / 小模型 VLA
- 本方案在三者交集填空：**FocusVLA + RobustVLA + VLA-Risk 三模块联合实验**

### 实验设计
- Phase 1（1-2 月）：复现 FocusVLA 在 LIBERO-Plus + VLA-Arena + VLA-Risk 上的 clean baseline
- Phase 2（3-4 月）：扰动评测，定位 FocusVLA 弱点
- Phase 3（5-7 月）：FocusVLA + RobustVLA worst-case δ 联合训练
- Phase 4（8-10 月）：与 OpenVLA / VLA-Adapter baseline 对比，绘制 robustness scaling curves
- Phase 5（11-12 月）：写作 + 投稿 ICRA/CoRL workshop

### Go/No-Go 准则
- Phase 1 末：FocusVLA 在 LIBERO clean 上复现至少 95%（vs 论文 98.7%）
- Phase 2 末：暴露至少 2 个统计显著的弱点维度
- Phase 3 末：联合训练 +3 pp 平均改进（否则评测论文为主）

### 风险
- FocusVLA 代码未释（abstract 仅承诺 will be public）
- 如果只是「评测 + 小改」novelty 可能不足撑顶会 main

### 第一步 1-2 周任务清单
- [ ] 联系 FocusVLA 作者获取代码 / checkpoint release 时间
- [ ] 准备 LIBERO-Plus 评测环境（HuggingFace 上下载 LIBERO + OpenVLA checkpoint）
- [ ] 跑 OpenVLA-7B 在 LIBERO-Plus 上的 baseline（建立评测 pipeline）
- [ ] 准备 FocusVLA 重现的最小可行版本（基于 VLA-Adapter codebase，先实现 Cascaded Attention）

## 方案 2 · Recovery-Aware VLA（Plan A 候选）

### 问题陈述
当 VLA 主推策略失败时（碰撞、卡住、识别错误），现有系统大多只能"硬撞墙"。Recovery-Aware VLA 把鲁棒主推策略 + 失败检测 + 主动恢复整合为闭环系统。

### 研究假设
- H1：在 RobustVLA 训练的 VLA 上加 RC-NF 失败检测，可降低端到端任务失败率 +15pp
- H2：RaC 风格的 recovery 数据混合比例存在最优值（不在 0%/30% 极端）
- H3：用 RC-NF 异常分数代替人工介入，可自动化触发 recovery，节约 90% 标注成本

### C agent 识别的核心研究空白
> **「Recovery 数据最优混合比例 scaling curve（0/5/10/20/30%）+ RC-NF/FAIL-Detect 替代人工介入」** — 用 LeRobot 1-2 人月可采 ~10k recovery trajectory

### 实验设计
- Phase 1（1-2 月）：复现 RC-NF on π0 (References 已有笔记)，建立异常检测 baseline
- Phase 2（3-5 月）：采集 ~10k recovery trajectory（用 LeRobot + 人在回路）
- Phase 3（6-9 月）：扫 recovery 数据混合比例 0/5/10/20/30%，画 scaling curve
- Phase 4（10-12 月）：用 RC-NF 自动化触发 recovery 替代人工介入实验
- Phase 5（5-6 月 2027）：写作 + 投 ICRA 2027 / CoRL workshop

### 第一步 1-2 周任务清单
- [ ] Week 1：配置 LeRobot + RC-NF 仓库（References 已有 RC-NF 笔记可作为入口）
- [ ] Week 1：复现 RC-NF 在 LIBERO-Anomaly-10 上的 baseline AUC
- [ ] Week 2：搭建 recovery 数据采集 pipeline（SO-100 真机 + LeRobot teleop）
- [ ] Week 2：定义 recovery 触发条件（异常分数阈值 + 时序判定）

## 方案 3 · Test-Time Adaptation for VLA Robustness（首推 Plan A）

### 问题陈述
VLA 在测试时遇到分布偏移（新光照、新物体、新背景），能否在线适应而无需重训？TT-VLA (2601.06748) 提供了基础框架，但**手工设计的 task progress reward 是 secret sauce**，难以推广。

### 研究假设
- H1：VLM perception verifier（RePLan 风格）可作为通用的 task progress reward，替代手工设计
- H2：加入 RC-NF safety constraint 后，TT-VLA 的"测试时学坏"概率降低 50%+
- H3：组合方法在 VLA-Risk 6 维扰动上可补救 ≥ 10pp 的 OOD 性能

### C agent 识别的核心研究空白
> **「VLM Verifier 替代 TT-VLA 手工 reward + RC-NF safety filter 防止 TT 学坏 + 在 VLA-Risk 6 维扰动上系统评测在线适应能否补救训练分布外」**

### 数学框架
$$\mathcal{L}_{TT}^{\text{ours}} = -\mathbb{E}_t \left[ r_t^{\text{VLM}}(o_t, a_t) - \lambda \cdot p_{\text{anomaly}}^{\text{RC-NF}}(o_t) \right] + \beta \cdot D_{KL}(\pi_\theta \| \pi_{\theta_0})$$

- $r_t^{\text{VLM}}$：VLM（InternVL3 / GPT-4V）估计的 task progress reward
- $p_{\text{anomaly}}^{\text{RC-NF}}$：RC-NF 异常分数作为「学坏」惩罚
- $D_{KL}$：保留预训练知识的 KL 正则

### 实验设计

| Phase | 时间 | 目标 |
|---|---|---|
| Phase 1 | 8-9 月 | 复现 TT-VLA on OpenVLA + LIBERO-Plus，建立 baseline |
| Phase 2 | 10-12 月 | 替换 reward proxy 为 VLM verifier (InternVL3 / GPT-4V) |
| Phase 3 | 1-2 月 | 加入 RC-NF safety constraint 做消融 |
| Phase 4 | 3-4 月 | 在 VLA-Risk 6 维扰动上完整评测 |
| Phase 5 | 5-6 月 | 投稿 CoRL 2027 main + workshop |

### Go/No-Go 准则

| 节点 | Go 条件 | No-Go 触发 |
|---|---|---|
| Phase 1 末 | OpenVLA + LIBERO-Plus baseline SR ≥ 65% | < 50% → 检查环境配置 |
| Phase 2 末 | VLM verifier reward 与原 reward 相关性 r > 0.5 | < 0.3 → 换 VLM 或换设计 |
| Phase 3 末 | RC-NF safety 后 TT 稳定性提升 ≥ 20% | < 5% → 简化为 KL only |
| Phase 4 末 | VLA-Risk 6 维平均 ASR 改进 ≥ 10pp | < 5pp → 转 workshop 投稿 |

### 第一步 1-2 周任务清单
- [ ] Week 1：环境配置 OpenVLA + LeRobot；下载 LIBERO-Plus benchmark
- [ ] Week 1：运行 OpenVLA-7B baseline evaluation on LIBERO-Spatial
- [ ] Week 2：阅读 TT-VLA 代码（如有）或基于论文重现核心 reward proxy 模块
- [ ] Week 2：第一个 sanity check on LIBERO-Spatial（无 VLM、无 RC-NF 的最小 TT 版本）
- [ ] 同步：联系 RC-NF 作者获取 LIBERO-Anomaly-10 access；申请 VLA-Risk 代码 release

## 方案 4 · Instruction-Robust VLA（补盲备选）

### 问题陈述
VLA-Risk 实证指令侧 ASR **63.99%** > 视觉侧 38.91%（高 25 pp）。FocusVLA 的视觉聚焦保护不了指令侧，这是 VLA 安全部署的最大隐患。

### 研究假设
- H1：把 RobustVLA worst-case δ 框架从 visual / action 维度扩展到 instruction 维度可显著提升指令鲁棒性
- H2：用 PAIR / GCG 生成对抗指令 + 正确 action 配对做 LoRA 微调，可在 LIBERO-Para + VLA-Risk I_* 上提升 ≥ 20pp

### C agent 识别的核心研究空白
> **「Instruction-Adversarial-Trained VLA」** — RobustVLA worst-case δ 从视觉延伸到指令侧；PAIR/GCG 生成 adv instruction + 正确 action 配对做 LoRA 微调；**与 RobustVLA 作者天然连接（同所），潜在合作可能**

### 实验设计
- Phase 1（1-2 月）：复现 RobustVLA + 阅读 VLA-Fool / Annie / AdvVLA Jones
- Phase 2（3-5 月）：用 PAIR / GCG 在 LIBERO 上生成对抗指令配对数据
- Phase 3（6-9 月）：LoRA 微调 OpenVLA / SmolVLA on adv instruction pairs
- Phase 4（10-12 月）：在 LIBERO-Para + VLA-Risk I_obj/I_act/I_spa 上评测
- Phase 5（5-6 月 2027）：投稿（CoRL workshop / EMNLP cross-listing）

### 第一步 1-2 周任务清单
- [ ] Week 1：阅读 PAIR (Chao et al. 2023) + GCG (Zou et al. 2023) jailbreak 论文
- [ ] Week 1：复现 OpenVLA on LIBERO-Para（评测指令侧 baseline）
- [ ] Week 2：用 GCG 在 LIBERO 上生成 100 条对抗指令样本，人工核对质量
- [ ] Week 2：评估 OpenVLA 在对抗指令上的 ASR

## 方案 5 · Shortcut Learning 诊断与防御（高赌注）

### 问题陈述
[[2025-08_ShortcutLearning_Xing]] (CoRL 2025) 系统诊断了 VLA dataset fragmentation 与 spurious correlation 根源，**这是给"为什么需要数据增广 / 架构改进"提供理论依据的关键论文**。但目前没有系统的「诊断 + 防御」端到端工作。

### 研究假设
- H1：在 OpenVLA / SmolVLA 中存在可量化的 shortcut（如「左手拿杯 = 蓝色背景」的伪相关）
- H2：用 worst-case 优化（DRO 谱系）+ counterfactual augmentation（CAST / CF-VLA）的组合可显著降低 shortcut 依赖
- H3：诊断指标可作为评估 VLA 鲁棒性的新维度（区别于扰动评测）

### 核心叙事
**「诊断（A6 ShortcutLearning）—架构（A FocusVLA）—数据（B RobustVLA）」三方组合** 是 BC 申请书的理想叙事。

### 第一步 1-2 周任务清单
- [ ] Week 1：精读 ShortcutLearning in Generalist Robot Policies (2508.06426) 论文 + 代码
- [ ] Week 2：在 OpenVLA 上跑 shortcut 诊断 toolkit，识别 2-3 个具体 spurious correlations

### 风险
- 诊断阶段实验设计复杂
- 如果没找到强 shortcut 故事拍不响

## 方案 6 · Adversarial Synthetic Data Generation（备选，新发现）

### 问题陈述（来自 B3+B4 agent）
所有 12 篇合成数据 pipeline（RoboGen / MimicGen / RoboCasa / Holodeck / Genesis 等）**全部只生成 clean instruction**。但 VLA-Risk 实证显示指令侧 ASR > 视觉侧——**合成数据的「instruction adversarial 版本」是巨大空白**。

### 研究假设
- H1：用 LLM 在 RoboGen / MimicGen 的 pipeline 上生成对抗指令 + 配对 action，作为训练增广
- H2：「Adversarial RoboGen」生成的数据可提升 VLA 在 VLA-Risk I_* 维度上的 ASR ≥ 15pp
- H3：合成对抗数据比真实对抗扰动便宜 10×，效果可比

### 数据 pipeline
1. RoboGen 自动生成 clean task → LLM 注入对抗扰动（attribute contradiction / action impossibility）→ 用主推 VLA 生成 correct action 作为标签
2. 训练时把 (adv instruction, correct action) 对加入 RobustVLA 训练 loop

### 第一步 1-2 周任务清单
- [ ] Week 1：跑 RoboGen 在 LIBERO 上的最小可行版本（参考 README）
- [ ] Week 2：用 GPT-5 在 RoboGen 任务 prompt 上生成 5 类对抗扰动，验证合成结果质量

### 风险
- 完全新方向，没有先例可对照
- 需要协调 RoboGen 的 task generation + adversarial prompting + RobustVLA 训练三个 pipeline

## 关于"不降低难度"的承诺

本排序在选择中**坚决保留了野心**：
- 首推方案 3 的 contribution 是 **CoRL/RSS main paper 级别**（VLM verifier + safety constraint + 6 维评测 = 完整故事）
- 保留方案 5（高赌注）作为团队成员能力突出时的延伸方向
- 不推荐过度保守的"评测加小改"路线作为主推（除非作为备选）

但同时 acknowledges 一年期本科课题的现实约束：
- 提供方案 1 作为保守备选
- 决策树明确每阶段的降级路径
- 不要求团队同时推进 ≥ 2 个深度方案
